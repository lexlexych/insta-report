# InstaReply — инструкция для Codex

Ты работаешь над этим проектом попеременно с Claude Code: любая сессия может быть начата одним
агентом и продолжена другим. Единственный источник истины — файлы репозитория.
Полный регламент — `docs/orchestration/workflow.md`, читай его в начале каждой сессии,
затем `docs/tickets/OVERRIDES.md` и `state/progress.json`.

## Специфика Codex (у тебя нет сабагентов — играешь роли последовательно)
Выполняя тикет, пройди роли по очереди, не смешивая:
1. РОЛЬ implementer: реализуй тикет строго по его файлу `docs/tickets/T-0XX.md`
   и соглашениям `docs/plan.md §5` (с учётом OVERRIDES). Только заявленные файлы.
2. РОЛЬ reviewer: отложи «шляпу автора», пройди дифф по чек-листу ревью из workflow.md
   как чужой код. Найденные блокеры исправь и проревьюй повторно.
3. РОЛЬ scribe: создай `docs/manual-tests/T-0XX.md` по шаблону из
   `docs/orchestration/manual-testing.md`; обнови `state/progress.json` и `docs/build-log.md`.
4. Ветка/PR — по правилам GIT из workflow.md.

## Начало сессии
Выполни процедуру RESUME из workflow.md (синхронизация статусов по влитым веткам),
сообщи человеку состояние и следующий доступный тикет, жди подтверждения.
