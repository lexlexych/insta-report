# Ручной тест T-045 · Zernio: env-группа и API-клиент

## Что сделано

Добавлена необязательная конфигурация Zernio и типизированный клиент для profiles, Instagram-аккаунтов, inbox и настроек webhook. При незаполненной паре переменных Zernio приложение не включает интеграцию и продолжает работать как раньше. Demo-скрипт безопасно показывает доступные данные, не печатая ключ или секрет.

## Подготовка

- ветка: `git fetch && git checkout zernio && pnpm install`
- зарегистрируйтесь на [zernio.com](https://zernio.com), в dashboard создайте API-ключ и скопируйте `.env.example` в `.env.local`.
- env: заполните вместе `ZERNIO_API_KEY` ключом из Zernio Dashboard и `ZERNIO_WEBHOOK_SECRET` случайной строкой (например, результатом `pnpm tsx scripts/gen-key.ts`). `ZERNIO_API_BASE` оставьте `https://zernio.com/api`.
- сервисы: внешний сервис запускать не требуется.

## Проверки (каждая: Действие → Ожидаемое)

1. Оставьте `ZERNIO_API_KEY` и `ZERNIO_WEBHOOK_SECRET` пустыми, затем выполните `pnpm tsx scripts/manual/T-045-demo.ts` → выводится `isZernioEnabled() === false`, команда завершается с кодом 0.
2. Заполните обе переменные действительными значениями и выполните `pnpm tsx scripts/manual/T-045-demo.ts` → JSON содержит массивы `profiles`, `accounts` и `webhookSettings`; в выводе нет API-ключа и webhook-секрета.
3. В Zernio Dashboard создайте profile, затем повторите demo → новый profile отображается с `profileId` и `name`.
4. Подключите Instagram Business-аккаунт к созданному profile через Dashboard, затем повторите demo → в `accounts` есть `accountId`, `profileId`, `platform`, `username`, `displayName` и `isActive`.
5. Выполните `pnpm lint` и `pnpm typecheck` → обе команды завершаются успешно.

## Негативные проверки

1. Заполните только одну из пары `ZERNIO_API_KEY` / `ZERNIO_WEBHOOK_SECRET` и запустите любую команду приложения → валидация env сообщает, что переменные должны быть заданы вместе.
2. Укажите заведомо неверный `ZERNIO_API_KEY`, оставив webhook-секрет заполненным, и выполните demo → команда завершается с `ZernioAuthError`; ни ключ, ни секрет не появляются в выводе.

## Как понять, что тикет НЕ работает

- При пустых Zernio-переменных demo пытается обратиться к сети или завершается с ошибкой вместо сообщения `isZernioEnabled() === false`.
- При корректном ключе profile, account или настройки webhook отсутствуют в выводе demo; проверьте ключ и содержимое соответствующих разделов Zernio Dashboard.
- В ошибке или выводе появляется значение API-ключа либо webhook-секрета.
