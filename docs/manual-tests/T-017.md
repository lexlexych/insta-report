# Ручной тест T-017 · Сбор контекста беседы

## Что сделано
- Добавлен модуль `src/lib/pipeline/context.ts`, который строит контекст диалога: username клиента, хронологичную историю, неотвеченный хвост клиента и timestamp последнего ответа бизнеса.
- `deriveContext` — чистая функция без сети/БД; `buildContext` параллельно получает username и последние сообщения через Instagram Graph client.
- В `src/lib/ig/client.ts` `created_time` из Graph нормализуется из ISO-строки в epoch milliseconds, чтобы downstream-логика работала с числами.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-017 && pnpm install`.
- Env: для demo-проверки не нужен `.env` и не нужны реальные Meta/Supabase credentials.
- Сервисы: ничего поднимать не нужно; проверка выполняется локальным скриптом без сети и БД.

## Проверки
1. Выполните `pnpm lint` → ожидаемо: команда завершается без ESLint-ошибок.
2. Выполните `pnpm typecheck` → ожидаемо: TypeScript завершается без ошибок.
3. Выполните `pnpm tsx scripts/manual/T-017-demo.ts` → ожидаемо: все строки начинаются с `OK`, в конце есть `OK T-017 demo completed`.
4. AC «deriveContext — чистая функция c исчерпывающими проверками»: в выводе demo должны быть `OK` для сценариев `[К, Б, К, К]`, «бизнес не писал», «последним писал бизнес», «история хронологична», «пустой текст → [вложение]`.
5. AC «Fallback-режимы покрыты»: в выводе demo должны быть `OK последним писал бизнес -> pendingText fallbackText`, `OK пустая беседа -> single-message fallback` и `OK пустая беседа -> pendingText из webhook`.
6. Проверка нормализации Graph `created_time`: выполните `rg "createdTime: number|normalizeCreatedTime|Date.parse" src/lib/ig/client.ts` → ожидаемо: видны тип `createdTime: number`, функция `normalizeCreatedTime` и `Date.parse`.

## Негативные проверки
1. Откройте `scripts/manual/T-017-demo.ts`, временно замените ожидаемую строку `Нужна запись\nНа завтра` на любое другое значение и выполните `pnpm tsx scripts/manual/T-017-demo.ts` → ожидаемо: хотя бы одна строка начинается с `FAIL`, процесс завершается ошибкой `T-017 manual demo failed`. Верните файл обратно после проверки.
2. Временно закомментируйте сортировку в `deriveContext` и повторите demo → ожидаемо: сценарии хронологичной истории/неотвеченного хвоста могут упасть, что подтверждает чувствительность проверки к порядку сообщений. Верните файл обратно после проверки.

## Как понять, что тикет НЕ работает
- `pnpm tsx scripts/manual/T-017-demo.ts` падает на `[К, Б, К, К]` — неверно собирается хвост клиента после последнего сообщения бизнеса.
- Demo падает на `последним писал бизнес -> pendingText fallbackText` — не обработана гонка, когда Graph уже содержит ответ бизнеса, а webhook-текст всё ещё надо сохранить как fallback.
- Demo падает на `пустая беседа -> single-message fallback` — `buildContext` не подставляет webhook-сообщение при пустом ответе Graph.
- `pnpm typecheck` ругается на `createdTime` — проверьте, что Graph client возвращает epoch milliseconds, а не ISO-строку.
