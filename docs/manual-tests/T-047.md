# Ручной тест T-047 · Zernio: бэкенд подключения Instagram

## Что сделано

Mini App теперь получает OAuth-ссылку Zernio для Instagram и безопасно принимает возврат из
авторизации. Подключённый аккаунт хранится у нужного тенанта, его можно проверить и отключить.
Подпись возврата и серверная сверка аккаунта не позволяют подменить параметры ссылки.

## Подготовка

- ветка: `git fetch && git checkout ticket/T-047 && pnpm install`
- env: заполните обязательные переменные приложения из `.env.example`, а также `ZERNIO_API_KEY` и
  `ZERNIO_WEBHOOK_SECRET` из дашборда Zernio; `APP_BASE_URL` должен быть публичным HTTPS-адресом
  туннеля или dev-развёртывания; `TELEGRAM_BOT_USERNAME` — username бота из BotFather.
- сервисы: примените миграции T-046 командой `pnpm db:push`; запустите `pnpm dev` и направьте
  публичный туннель (например, cloudflared) на порт 3000. Укажите публичный адрес туннеля в
  `APP_BASE_URL`, затем перезапустите dev-сервер.

## Проверки

1. Войдите в Mini App как тенант и выполните `POST /api/miniapp/zernio/connect` с телом
   `{"embedded":false}` и cookie сессии → ответ `200`, `ok: true` и `url` ведёт на домен
   авторизации Zernio; в `zernio_accounts` появляется одна строка со статусом `pending` и профилем.
2. Откройте полученный `url`, войдите в Business-аккаунт Instagram и завершите OAuth → callback
   показывает страницу «Instagram подключён», а запись тенанта в `zernio_accounts` имеет статус
   `active`, username и время подключения.
3. Повторите переход по тому же callback URL → снова показана страница успеха, в БД остаётся одна
   запись и один `zernio_account_id` для тенанта.
4. Выполните `GET /api/miniapp/zernio/status` с cookie тенанта → ответ содержит `ok: true`,
   `enabled: true`, статус `active`, username и connectedAt; запрос не создаёт сетевых вызовов в
   дашборде Zernio.
5. Выполните `POST /api/miniapp/zernio/disconnect` → ответ `ok: true`, запись становится
   `disconnected`, а аккаунт исчезает из соответствующего профиля в дашборде Zernio.
6. После отключения ещё раз вызовите connect → используется прежний `zernio_profile_id`, а не
   создаётся новый профиль; OAuth снова можно пройти до статуса `active`.
7. Откройте callback через Telegram deep-link с параметром `startapp=zernio` → после авторизации
   Mini App открывает `/app` (дашборд), а не экран прямого Meta-подключения.

## Негативные проверки

1. Измените хотя бы один символ `state` в callback URL и откройте его → `403 Forbidden`; в
   дашборде Zernio нет нового запроса, в БД нет изменений.
2. Для валидного callback URL замените `profileId` на другой → `403 Forbidden`; аккаунт не
   активируется.
3. Откройте callback без `accountId` → отображается страница «Не получилось», запись не получает
   статус `active`.
4. Временно удалите оба значения `ZERNIO_API_KEY` и `ZERNIO_WEBHOOK_SECRET`, перезапустите
   сервер и вызовите connect, status или disconnect → каждый маршрут отвечает `404` с
   `error: "disabled"`.

## Как понять, что тикет НЕ работает

Если URL connect не открывает Zernio, проверьте вкладку Network и серверный лог: ошибки Zernio
должны возвращаться как `zernio_error` без ключа API. Если callback не активирует запись, проверьте
совпадение profileId в `zernio_accounts` с Zernio и наличие accountId среди аккаунтов профиля.
При ошибке возврата в Telegram убедитесь, что публичный `APP_BASE_URL` и username бота доступны
из браузера телефона.
