# Ручной тест T-035 · Реестр сфер деятельности и шаблоны баз знаний

## Что сделано
- Добавлен типобезопасный реестр четырёх сфер деятельности с иконками и ключами локализации.
- Markdown-шаблоны для русского и немецкого языков встраиваются в приложение на этапе генерации, без чтения файлов в serverless-окружении.
- Добавлены команда проверки синхронизации и demo-скрипт, который выводит все восемь шаблонов.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-035 && pnpm install`.
- Env: для этой проверки переменные из `.env.example` не нужны.
- Сервисы: не требуются.

## Проверки
1. Выполните `pnpm gen:kb-templates` → создаётся или детерминированно обновляется `src/lib/kb-templates/templates.gen.ts`.
2. Выполните `pnpm gen:kb-templates --check` → команда завершается успешно, подтверждая синхронизацию с Markdown-шаблонами.
3. Выполните `pnpm tsx scripts/manual/T-035-demo.ts` → выводятся восемь строк: по одной для `events`, `photographer`, `travel` и `massage` на языках `ru` и `de`; каждая строка начинается с идентификатора языка и сферы, после чего выводится первый заголовок шаблона. Завершение: `T-035 demo OK`.
4. Выполните `pnpm lint && pnpm typecheck` → обе команды завершаются без ошибок.

## Негативные проверки
1. Временно измените один символ в `src/lib/kb-templates/templates.gen.ts`, затем выполните `pnpm gen:kb-templates --check` → команда завершается с ошибкой о рассинхронизации и предлагает запустить `pnpm gen:kb-templates`. Сразу после проверки восстановите файл командой `pnpm gen:kb-templates`.
2. Временно переименуйте любой файл, например `src/lib/kb-templates/ru/events.md`, затем выполните `pnpm gen:kb-templates` → команда завершается с понятной ошибкой `Missing KB template for ru/events`. Верните исходное имя файла и снова запустите генерацию.

## Как понять, что тикет НЕ работает
- `pnpm gen:kb-templates --check` сообщает о рассинхронизации сразу после обычной генерации.
- Demo-скрипт выводит меньше восьми строк, не завершается `T-035 demo OK` или сообщает об отсутствующем шаблоне.
- `pnpm lint` или `pnpm typecheck` сообщают об ошибке типов при обращении к реестру или к шаблонам.
