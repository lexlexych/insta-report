# Ручной тест T-024 · Топики в личном чате бота по категориям

## Что сделано

Черновики Telegram уходят в топики личного чата владельца: топик создаётся для категории и переиспользуется при следующих доставках.

Доработка (исправление после первого прогона T-024):

- **Топики теперь реально появляются.** Раньше создание топика было заблокировано кэшем `tenants.tg_topics_enabled`, который наполнялся только из `has_topics_enabled` на `/start`/initData; Telegram это поле отдаёт ненадёжно (в приватных чатах его нет в `getChat`/`ChatFullInfo`), поэтому кэш оставался `false` и топики не создавались никогда. Теперь источник истины — сам вызов `createForumTopic`: доставка/создание категории пытается создать топик напрямую, при успехе кэш `tg_topics_enabled` становится `true`, при неудаче (Threaded Mode выключен) — бесшовный fallback в плоский чат. Включённый в BotFather Threaded Mode подхватывается сразу, без повторного `/start`.
- **Кнопка «Отправить» редактирует карточку на месте.** Отдельного топика «Архив» больше нет. После успешной отправки карточка остаётся в своём топике категории и редактируется в лаконичный вид: шапка «✅ отправлено в HH:MM \<ссылка на получателя\>», без кнопок, без строки категории и без нижней строки статуса.

Если темы выключены или Telegram возвращает ошибку по топикам, доставка деградирует в обычный General/плоский чат без потери черновика; отправка в любом случае редактирует карточку на месте.

## Подготовка

- ветка: `git fetch && git checkout ticket/T-024 && pnpm install`
- env: заполните стандартные переменные для Telegram, Supabase и Instagram из `.env.example`; секреты не вставляйте в отчёт о проверке.
- сервисы: примените миграции в dev-проект Supabase командой `pnpm db:push`, затем запустите приложение `pnpm dev`.
- Telegram: в `@BotFather` выберите бота и включите **Threaded Mode** для личных чатов. Если этот режим недоступен/выключен, негативная проверка плоского режима всё равно должна проходить.
- данные: нужен tenant с завершённым онбордингом, активным Instagram-подключением и открытым личным чатом владельца с ботом.

## Проверки

1. **Статус в настройках**: откройте Mini App → «Настройки». → Ожидаемо: виден блок «Доставка в Telegram» со строкой «Темы в чате: включены/выключены» и подсказкой про BotFather Threaded Mode.
2. **Создание категории создаёт один топик**: при включённых темах откройте «Категории» и создайте категорию «Запись». → Ожидаемо: категория создана, в личном чате бота появляется топик «Запись», в таблице `labels.tg_thread_id` у этой категории заполнен `message_thread_id`.
3. **Повторная доставка не создаёт дубль**: отправьте два входящих Instagram-сообщения, которые классифицируются в категорию «Запись». → Ожидаемо: обе карточки приходят в один и тот же топик «Запись», новых топиков с тем же именем не появляется.
4. **Переименование категории синхронизирует топик**: переименуйте «Запись» в «Бронирование». → Ожидаемо: имя топика в Telegram обновилось, категория в Mini App тоже переименована.
5. **Доставка черновика по топику**: отправьте новое входящее Instagram-сообщение для категории «Бронирование». → Ожидаемо: Telegram-карточка черновика приходит в топик «Бронирование», кнопка «Отправить» работает как раньше.
6. **Отправка редактирует карточку на месте**: нажмите «✅ Отправить» на карточке в топике «Бронирование». → Ожидаемо: ответ отправлен в Instagram; карточка **остаётся в топике «Бронирование»** и редактируется в шапку «✅ отправлено в HH:MM @получатель» — без кнопок, без строки категории, черновик в цитате. Отдельного топика «Архив» не появляется. Повторный/дубль-клик по кнопке не роняет бота и показывает «⌛️ Карточка устарела».
7. **Удаление категории**: удалите категорию «Бронирование». → Ожидаемо: приложение не падает; если Telegram разрешил удалить топик — топик исчез, если нет — ошибка только в логах, категория удалена.

## Негативные проверки

1. **Темы выключены**: выключите Threaded Mode в `@BotFather` или проверьте tenant с `tg_topics_enabled=false`, затем создайте категорию и доставьте черновик. → Ожидаемо: категория создаётся, топик не обязателен, черновик приходит в обычный личный чат без `message_thread_id`.
2. **Топик удалён вручную**: при включённых темах удалите/закройте топик категории в Telegram и отправьте новое входящее сообщение этой категории. → Ожидаемо: доставка не теряется; карточка приходит в General/плоский чат или топик создаётся заново, владелец не получает отдельный спам об ошибке.
3. **Плоский режим — карточка редактируется на месте**: на tenant с `tg_topics_enabled=false` (Threaded Mode выключен) нажмите «Отправить». → Ожидаемо: Instagram-ответ отправлен, статус черновика `sent`, карточка в личном чате редактируется в «✅ отправлено в HH:MM …» без кнопок (поведение одинаково с режимом тем).

## Как понять, что тикет НЕ работает

- Threaded Mode включён в BotFather, но топики так и не появляются, а `tenants.tg_topics_enabled` остаётся `false` — смотрите лог `[tg] label topic fallback`: там будет ответ Telegram на `createForumTopic` (например, что режим тем всё-таки выключен для этого чата).
- Для одной категории появляется несколько одинаковых топиков — проверьте, что после первого создания заполнен `labels.tg_thread_id` и последующие доставки его переиспользуют.
- После удаления топика новая доставка падает вместо fallback — смотрите логи `[pipeline] draft topic send fallback` и факт появления карточки в General.
- После «Отправить» карточка не отредактировалась в «✅ отправлено в …» — смотрите лог `[pipeline] sent card edit failed`. Сбой редактирования best-effort и не должен влиять на `drafts.status='sent'`.
