# Ручной тест T-006 · Каркас Mini App

## Что сделано
Добавлена оболочка Telegram Mini App: SDK инициализируется на клиенте, применяет тему Telegram и показывает нижнюю навигацию.
Созданы страницы-заглушки для всех вкладок, онбординга и подключения Instagram; видимые строки берутся из i18n на русском и немецком.

## Подготовка
- ветка: `git fetch && git checkout ticket/T-006 && pnpm install`
- env: для этой проверки специальные переменные не нужны; если `pnpm install` в вашей сети впервые подтягивает пакет, проверьте, что установлен `@telegram-apps/sdk-react`.
- сервисы: запустить приложение командой `pnpm dev` и открыть локальный URL из вывода Next.js.
- Telegram: для проверки внутри Telegram нужен бот с кнопкой Web App на локальный HTTPS-туннель, например `npx localtunnel --port 3000`, либо другой туннель до dev-сервера.

## Проверки
1. В браузере вне Telegram откройте `http://localhost:3000/app` → ожидается заглушка «Öffne die App über den Telegram-Bot @InstaReplyBot» и пояснение, что панель работает только внутри Telegram Mini App.
2. Откройте `/app` через Telegram Mini App → ожидается страница «Дашборд» или «Dashboard» с текстом «в разработке» и нижней навигацией из 4 пунктов.
3. По очереди нажмите нижние вкладки `/app`, `/app/simulator`, `/app/labels`, `/app/settings` → ожидается переход на соответствующую заглушку; активная вкладка подсвечена цветом ссылки Telegram.
4. Откройте `/app/onboarding` и `/app/connect-instagram` через Mini App → ожидаются заголовки онбординга и подключения Instagram, текст «в разработке» берётся из текущей локали.
5. В Telegram переключите тему клиента dark/light и переоткройте или измените тему во время открытой Mini App → ожидается, что фон, текст, hint и link-цвета меняются без правки CSS/пересборки.
6. Проверьте локаль Telegram: с русским интерфейсом Telegram видны русские строки; с немецким или любым другим интерфейсом — немецкие строки по fallback.

## Негативные проверки
1. Откройте `/app/simulator` напрямую в обычном браузере → ожидается та же Telegram-only заглушка, без падения страницы и без навигации.
2. В DevTools Console вручную вызовите отсутствующий i18n-ключ через компонентную проверку или временный breakpoint в `t(locale, 'missingKey')` → ожидается возврат строки `missingKey` и warning в консоли, UI не падает.

## Как понять, что тикет НЕ работает
- Вместо Telegram-only заглушки вне Telegram виден белый экран: смотрите Console на ошибки инициализации SDK.
- Внутри Telegram нет нижней навигации или переходы ведут на 404: проверьте файлы маршрутов в `app/(miniapp)/app/**/page.tsx`.
- Тема не меняется: проверьте CSS variables `--bg`, `--text`, `--hint`, `--link` на `document.documentElement` во вкладке Elements.
- Часть строк на русском/немецком захардкожена или отсутствует: проверьте словарь `src/lib/i18n.tsx`.
