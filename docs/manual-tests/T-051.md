# Ручной тест T-051 · Подключение Instagram через Zernio на дашборде

## Что сделано

На дашборд Mini App добавлена независимая область подключения Instagram через Zernio. Она скрыта,
пока интеграция Zernio не настроена, запускает OAuth в подходящем для Telegram окружении и обновляет
статус после возврата. Прямое Meta-подключение остаётся отдельной, неизменённой областью.

## Подготовка

- ветка: `git fetch origin && git checkout zernio && pnpm install`
- env: заполните обязательные переменные приложения из `.env.example`, а также пару
  `ZERNIO_API_KEY` и `ZERNIO_WEBHOOK_SECRET` из кабинета Zernio. Задайте публичный HTTPS-адрес
  dev-развёртывания или туннеля в `APP_BASE_URL`; не публикуйте значения ключей в терминале,
  скриншотах или отчёте.
- сервисы: примените миграции к отдельному dev-проекту командой `pnpm db:push`, запустите
  `pnpm dev`, откройте приложение через Telegram-бота как тестовый тенант. Подготовьте доступный
  бизнес-аккаунт Instagram для OAuth. Для проверки на телефоне `APP_BASE_URL` должен быть доступен
  с этого телефона по HTTPS.

## Проверки

1. Откройте `/app` в Telegram Desktop при заполненной паре переменных Zernio. → Под существующей
   областью прямого подключения Instagram видна отдельная область «Подключить Instagram через
   Zernio» с описанием подключения в один клик; прежняя область Meta не изменилась.
2. На Desktop нажмите «Подключить». → На время запроса кнопка недоступна, затем OAuth открывается
   через ссылку Telegram (либо в отдельной вкладке). В Network запрос
   `POST /api/miniapp/zernio/connect` содержит `{"embedded":false}`.
3. Завершите OAuth для тестового Instagram-аккаунта и вернитесь в Mini App. → При возврате по
   фокусу дашборд обновляет данные без перезапуска; область Zernio показывает `@username`, бейдж
   «активно» и кнопку «Отключить».
4. Откройте тот же `/app` внутри Telegram на Android и нажмите «Подключить». → OAuth открывается
   переходом текущего окна, а запрос connect содержит `{"embedded":true,"ios":false}`. После
   успешного callback Mini App возвращается на `/app?zernio=success`, показывает активный статус
   и убирает параметр `zernio` из адресной строки.
5. Откройте `/app` внутри Telegram на iPhone и нажмите «Подключить». → Страница Instagram OAuth
   остаётся во встроенном окне Telegram; нативное приложение Instagram не открывается и не
   показывает alert с ошибкой. В Network запрос connect содержит `{"embedded":true,"ios":true}`.
   После успешного callback Mini App возвращается на `/app?zernio=success` и показывает активный
   статус. Повторите с выходом из Instagram в этом окне, чтобы убедиться, что можно ввести логин
   и пароль.
6. При активном подключении нажмите «Отключить», сначала отмените системный диалог. → Статус не
   меняется. Повторите действие и подтвердите. → После обновления дашборда область показывает
   кнопку «Переподключить», а прямое Meta-подключение сохраняет свой прежний статус.
7. Нажмите «Переподключить» и снова завершите OAuth. → Используется тот же поток подключения,
   после возврата снова отображается активный `@username`.
8. Если для одного тестового тенанта активны и прямое Meta-подключение, и Zernio, откройте
   дашборд. → Обе области отображают свои статусы одновременно и независимо друг от друга.

## Негативные проверки

1. Начните OAuth, но закройте браузер или отмените авторизацию до callback. Вернитесь на
   дашборд. → Область остаётся видимой со статусом «Подключение не завершено», а повторный клик
   по «Подключить» доступен и запускает OAuth заново.
2. В отдельной локальной конфигурации временно удалите обе переменные `ZERNIO_API_KEY` и
   `ZERNIO_WEBHOOK_SECRET`, перезапустите сервер и обновите `/app`. → Области Zernio нет вовсе,
   а весь остальной дашборд загружается без ошибки. Верните обе переменные после проверки.
3. В отдельной dev-конфигурации укажите для `ZERNIO_API_BASE` заведомо недоступный HTTPS-адрес,
   перезапустите сервер и нажмите «Подключить». → Появляется сообщение «Попробуйте позже», кнопка
   снова становится активной. Восстановите обычный адрес API после проверки.

## Как понять, что тикет НЕ работает

- область Zernio видна при отключённых обеих переменных либо пропадает при заполненных — проверьте
  перезапуск сервера и корректность пары переменных без вывода их значений;
- после OAuth остаётся старый статус — во вкладке Network проверьте повторный
  `GET /api/miniapp/dashboard`, а в `zernio_accounts` для тенанта — `status` и `username`;
- на телефоне OAuth открылся во внешней вкладке вместо текущего окна или callback не вернул в Mini
  App — проверьте `embedded:true` в запросе connect и публичную HTTPS-доступность `APP_BASE_URL`;
- на iPhone вместо формы OAuth открылось нативное приложение Instagram — проверьте, что запрос
  connect содержит `ios:true`; если URL от Zernio уже не использует известную обёртку
  `/accounts/login/?force_authentication=1&next=/oauth/authorize...`, зафиксируйте только host и
  pathname стартового URL и обратитесь к Zernio за iOS/web-вариантом OAuth (query/state не
  записывайте);
- при ошибке API кнопка остаётся заблокированной или нет сообщения для пользователя — проверьте
  ответ `POST /api/miniapp/zernio/connect`: ошибки Zernio должны приходить как HTTP 502 без ключей
  API в ответе или логах.
