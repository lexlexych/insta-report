# Ручной тест T-009 · Визард онбординга Mini App

## Что сделано
Добавлен пошаговый онбординг Mini App: приветствие, форма организации, генерация базы знаний, просмотр/редактирование и финальный выбор следующего действия. Прогресс сохраняется в `tenants.onboarding_step`, а Mini App перенаправляет незавершивших онбординг пользователей обратно на `/app/onboarding`.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-009 && pnpm install`.
- Env: заполнить переменные, нужные для Mini App auth и Supabase: `TELEGRAM_BOT_TOKEN`, `MINIAPP_JWT_SECRET`, `SUPABASE_URL`, `SUPABASE_SECRET_KEY`; значения взять из BotFather, локального секрета сессий и dev-проекта Supabase. Секреты в логах не печатать.
- БД: применить актуальные миграции на dev-проекте Supabase командой `pnpm db:push`.
- Сервис: запустить `pnpm dev`, открыть Mini App через Telegram-бота с настроенным Web App URL на локальный туннель или preview-домен.

## Проверки
1. Откройте Mini App новым Telegram-пользователем или очистите в БД у существующего тенанта `onboarding_step`, `org_name`, `org_description`, `knowledge_base`.
   → Ожидаемо: открывается `/app/onboarding`, виден welcome-шаг с 3 тезисами о продукте, основная CTA находится в Telegram MainButton с текстом «Начать»/`Starten`.
2. Нажмите MainButton «Начать».
   → Ожидаемо: в таблице `tenants` у тенанта `onboarding_step = org_form`, показана форма с полями «Название организации» и «Расскажите о бизнесе».
3. Закройте Mini App и откройте снова.
   → Ожидаемо: пользователь возвращается на незавершённый шаг `org_form`, а не на дашборд.
4. Заполните название минимум 2 символа и описание минимум 80 символов, затем нажмите MainButton «Сгенерировать базу знаний».
   → Ожидаемо: показывается шаг `generating` со скелетон-анимацией и подписями этапов, затем `review_kb`; в БД заполнены `org_name`, `org_description`, `knowledge_base`, `onboarding_step = review_kb`.
5. На шаге `review_kb` нажмите «Редактировать», измените текст базы знаний, нажмите «Предпросмотр», затем MainButton «Всё верно».
   → Ожидаемо: текст отображается простым markdown-рендером (заголовки, списки, абзацы), изменения сохраняются через `PUT /api/miniapp/knowledge`, появляется финальный шаг.
6. На шаге `done` нажмите «Подключить Instagram».
   → Ожидаемо: `onboarding_step = done`, Mini App переходит на `/app/connect-instagram`; повторное открытие `/app/onboarding` редиректит на `/app`.
7. Повторите финальный шаг для второго пользователя или сбросьте `onboarding_step` на `done`, затем нажмите «Попробовать тест-чат».
   → Ожидаемо: Mini App переходит на `/app/simulator`.

## Негативные проверки
1. На шаге `org_form` оставьте название короче 2 символов и/или описание короче 80 символов, нажмите MainButton генерации.
   → Ожидаемо: переход не выполняется, под полями показаны inline-ошибки на языке интерфейса, запрос генерации не отправляется.
2. На шаге `generating` временно сломайте запрос в DevTools Network (например, заблокируйте `/api/miniapp/onboarding/generate`) и повторите генерацию.
   → Ожидаемо: появляется сообщение/тост об ошибке и MainButton «Повторить», повторный успешный запрос переводит на `review_kb`.
3. Попробуйте открыть `/app` при `onboarding_step != done`.
   → Ожидаемо: роутинг-гард возвращает пользователя на `/app/onboarding`.

## Покрытие критериев приёмки
- Возврат на незавершённый шаг после закрытия Mini App: проверки 2–3.
- Telegram MainButton как основная CTA на каждом шаге: проверки 1, 2, 4, 5 и негативная проверка 2.
- Happy-path вместо автотеста по OVERRIDES: проверки 1–7 проходят полный сценарий вручную.

## Как понять, что тикет НЕ работает
- Если открывается дашборд при `onboarding_step != done`, проверить guard в Mini App и значение `tenants.onboarding_step` в Supabase.
- Если MainButton не появляется или не меняет текст, проверить консоль WebView Telegram и совместимость `@telegram-apps/sdk-react`.
- Если генерация не доходит до `review_kb`, смотреть Network-запрос `/api/miniapp/onboarding/generate` и серверные логи Next.js.
- Если редактирование базы знаний не сохраняется, проверить Network-запрос `PUT /api/miniapp/knowledge` и поле `tenants.knowledge_base`.
