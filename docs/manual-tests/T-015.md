# Ручной тест T-015 · Cron: рефреш долгоживущих токенов

## Что сделано

Добавлен ежедневный Vercel Cron `/api/cron/refresh-tokens`, который выбирает активные Instagram-подключения с устаревшим или пустым `token_refreshed_at`, обновляет long-lived token через Instagram Graph API и сохраняет новый токен только в зашифрованном виде. Ошибка одного тенанта не останавливает цикл: подключение помечается `error`, владелец получает Telegram-сообщение с кнопкой Mini App, администраторы получают алерт.

## Подготовка

- Ветка: `git fetch && git checkout ticket/T-015 && pnpm install`.
- Env в `.env.local`:
  - `CRON_SECRET` — случайная строка для защиты cron-роута.
  - `ENCRYPTION_KEY` — base64-ключ на 32 байта (`pnpm gen:key`).
  - `APP_BASE_URL` — публичный URL приложения; для локальной проверки можно указать `http://localhost:3000`.
  - `TELEGRAM_BOT_TOKEN` — токен бота из BotFather для проверки уведомлений.
  - `ADMIN_TELEGRAM_IDS` — один или несколько Telegram user/chat id через запятую для проверки админ-алерта; можно временно указать свой id.
  - `SUPABASE_URL`, `SUPABASE_SECRET_KEY` — dev-проект Supabase с применёнными миграциями.
- Сервисы: `pnpm db:push`, затем `pnpm dev`.
- Данные в БД: минимум 3 строки `tenants` с `tg_chat_id`/`telegram_user_id` и 3 строки `ig_connections` со `status='active'`; у первой и третьей `access_token_enc` должен содержать валидный зашифрованный long-lived token, у второй можно указать заведомо невалидный токен. Для попадания в выборку поставьте `token_refreshed_at = null` или дату старше 7 дней.

## Проверки

1. Откройте `vercel.json`.
   Ожидаемо: есть cron `{ "path": "/api/cron/refresh-tokens", "schedule": "0 4 * * *" }`.
2. Выполните ручной запуск с query-секретом: `curl "http://localhost:3000/api/cron/refresh-tokens?key=$CRON_SECRET"`.
   Ожидаемо: JSON содержит `processed: 3`, `refreshed: 2`, `failed` с tenant id второй записи; первая и третья записи обработаны несмотря на падение второй.
3. Проверьте первую и третью строки `ig_connections` в Supabase.
   Ожидаемо: `token_refreshed_at` обновлён, `access_token_enc` начинается с формата шифрования `v1:` и не равен plaintext-токену из ответа Instagram.
4. Проверьте вторую строку `ig_connections`.
   Ожидаемо: `status='error'`; остальные активные подключения не переведены в `error`.
5. Проверьте Telegram владельца второй строки и админ-чаты из `ADMIN_TELEGRAM_IDS`.
   Ожидаемо: владелец получил сообщение `⚠️ Подключение Instagram требует внимания: токен не удалось обновить. Откройте панель → Подключение` с кнопкой открытия Mini App; каждый админ получил один алерт по этому tenant id.
6. Повторите запуск через заголовок: `curl -H "Authorization: Bearer $CRON_SECRET" "http://localhost:3000/api/cron/refresh-tokens"`.
   Ожидаемо: запрос авторизован; если подходящих записей уже нет, ответ содержит `processed: 0`.

## Негативные проверки

1. Выполните `curl -i "http://localhost:3000/api/cron/refresh-tokens?key=wrong"`.
   Ожидаемо: HTTP 401 и тело `{"error":"Unauthorized"}`; в БД нет изменений, Telegram-сообщения не отправляются.
2. Временно очистите `ADMIN_TELEGRAM_IDS` и запустите сценарий с невалидным IG-токеном.
   Ожидаемо: ошибка tenant обрабатывается, владелец уведомляется, отсутствие админов не ломает cron.

## Как понять, что тикет НЕ работает

- Неверный секрет даёт 200 — смотреть проверку авторизации в `app/api/cron/refresh-tokens/route.ts`.
- В ответе нет tenant id упавшего подключения или цикл остановился после первой ошибки — смотреть обработку `try/catch` внутри последовательного цикла cron.
- В БД появился plaintext Instagram token — запись должна идти через `encrypt()` перед `markTokenRefreshed(...)`.
- `token_refreshed_at = null` не попадает в выборку — смотреть `igConnections.listActiveForRefresh(7)`.
- Нет Telegram-уведомлений — проверить `TELEGRAM_BOT_TOKEN`, `tg_chat_id`/`telegram_user_id` tenant и `ADMIN_TELEGRAM_IDS`.

## Покрытие критериев приёмки

- Провал одного тенанта не влияет на остальных: шаги 2 и 4.
- Новый токен шифруется перед записью: шаг 3.
- `vercel.json` c расписанием и docs про `CRON_SECRET`: шаги 1, 2, 6 и раздел README «Плановые задачи Vercel Cron».
