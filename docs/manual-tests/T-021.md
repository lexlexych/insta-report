# Ручной тест T-021 · Обработчик кнопки «✅ Отправить»

## Что сделано
- Telegram callback `send:{draftId}` теперь сразу закрывает спиннер, атомарно переводит pending-черновик в `sending`, повторно читает Instagram Direct и отправляет ответ только если владелец ещё не ответил вручную.
- Карточка черновика редактируется в финальный статус: отправлено, устарела, ручной ответ победил или ошибка отправки с кнопкой «🔁 Повторить».
- Callback `retry:{draftId}` разрешён только для черновиков в `error` и повторяет ту же безопасную отправку.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-021 && pnpm install`.
- Env: заполните `.env.local` по `.env.example`: `TELEGRAM_BOT_TOKEN` из BotFather, `TELEGRAM_WEBHOOK_SECRET`, `APP_BASE_URL` с HTTPS-туннелем, `SUPABASE_URL`/`SUPABASE_SECRET_KEY` dev-проекта, `ENCRYPTION_KEY`, `MINIAPP_JWT_SECRET`, LLM-переменные из предыдущих тикетов. Не публикуйте секреты в отчёте.
- БД: примените миграции к dev-проекту Supabase командой `pnpm db:push`.
- Сервисы: запустите приложение `pnpm dev`, поднимите HTTPS-туннель на порт 3000 и обновите Telegram webhook командой `pnpm setup:telegram`.
- Instagram: нужен активный тестовый Instagram Graph connection с `status=active`, `ig_account_id`, рабочим access token и входящим Direct-сообщением, которое уже создало Telegram-карточку через поток T-020.

## Проверки
1. Гигиена кода: выполните `pnpm lint && pnpm typecheck`. Ожидаемо: обе команды завершаются без ошибок.
2. Контрактный demo без реальных Telegram/Instagram: выполните `pnpm tsx scripts/manual/T-021-demo.ts`. Ожидаемо: вывод содержит `T-021 demo OK`, а сценарии показывают порядок `answer:Отправляю… -> claim`, отсутствие повторной отправки для stale/double-click, `skipped_manual`, error+retry и успешный retry.
3. Happy path в реальном чате: в Telegram нажмите «✅ Отправить» на свежей карточке черновика. Ожидаемо: callback-спиннер быстро закрывается текстом «Отправляю…», сообщение появляется в Instagram Direct, карточка в Telegram редактируется на месте в шапку «✅ отправлено в HH:MM @получатель» без inline-кнопок (без строки категории, черновик в цитате), в `drafts.status` стоит `sent`, в `message_log` добавлена запись `direction=out`, в `usage_stats.drafts_sent` увеличился счётчик.
4. Защита от двойного клика: создайте новый черновик и быстро нажмите «✅ Отправить» дважды или попросите второго администратора нажать ту же кнопку. Ожидаемо: в Instagram уходит только один ответ; вторая попытка не вызывает повторную отправку и карточка показывает «⌛️ Карточка устарела».
5. Ручной ответ выигрывает: создайте свежий черновик, затем до нажатия кнопки ответьте клиенту вручную из Instagram. После этого нажмите «✅ Отправить» в Telegram. Ожидаемо: новый автоответ не отправляется, `drafts.status` становится `skipped_manual`, карточка показывает «⚠️ Отменено: вы уже ответили вручную» и остаётся без кнопок.
6. Retry после ошибки: временно испортите access token тестового подключения или отключите сеть до Instagram, нажмите «✅ Отправить». Ожидаемо: `drafts.status=error`, карточка показывает «❌ Ошибка отправки: …» и одну кнопку «🔁 Повторить». Верните рабочий token/сеть и нажмите «🔁 Повторить». Ожидаемо: выполняется повторная отправка, карточка становится «✅ отправлено в HH:MM @получатель», повторная кнопка исчезает.

## Негативные проверки
1. Нажмите старую кнопку «🔁 Повторить» на карточке, которая уже была успешно отправлена или отменена. Ожидаемо: Instagram-сообщение не отправляется повторно, карточка редактируется в «⌛️ Карточка устарела».
2. Нажмите callback с неизвестным `callback_data` через тестовый Telegram update или старую кнопку другого типа. Ожидаемо: бот молча отвечает `answerCallbackQuery`, ошибок в логах Next.js нет.

## Покрытие критериев приёмки
- Дубль-отправка невозможна: проверки 2 и 4 демонстрируют атомарный claim и отсутствие второго `ig-send`.
- Ручной ответ владельца выигрывает у кнопки: проверка 5 демонстрирует re-check Direct после `trigger_ts` и статус `skipped_manual`.
- Из `error` есть путь retry, карточка всегда отражает финальный статус: проверки 3, 5, 6 и негативная проверка 1 покрывают финальные статусы и retry-клавиатуру.

## Как понять, что тикет НЕ работает
- Callback-спиннер крутится дольше нескольких секунд: смотрите логи `pnpm dev` и Telegram webhook, вероятно `answerCallbackQuery` не вызвался первым.
- В Instagram появились два одинаковых сообщения после двойного клика: проверьте условный update `drafts pending -> sending` и записи `drafts.status`.
- После ручного ответа всё равно ушёл автоответ: проверьте `ig_account_id`, timestamps `trigger_ts`/`created_time` и результат `getConversation` в логах/Meta Graph API Explorer.
- Retry не появляется или не исчезает: проверьте `drafts.error`, `drafts.status`, callback data `retry:{draftId}` и логи редактирования Telegram-сообщения.
