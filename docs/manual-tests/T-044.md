# Ручной тест T-044 · Визард подключения Instagram

## Что сделано

Страница подключения Instagram стала визардом: принятие приглашения тестировщика, OAuth и результат находятся в одном маршруте. До подтверждения заявки OAuth недоступен.

В Telegram на всех экранах визарда вместо крестика отображается системная стрелка «Назад»; после выхода со страницы снова доступен обычный крестик.

## Подготовка

- ветка: `git fetch && git checkout epic/E11 && pnpm install`
- env: заполните переменные из `.env.example` для Mini App, Supabase и Telegram-бота; для настоящего OAuth также укажите Instagram App ID и секрет из Meta Developers, а в `ADMIN_TELEGRAM_IDS` — Telegram ID тестового администратора. Добавьте предоставленный скриншот в `public/images/ig-tester-invite.png`.
- сервисы: примените миграции к dev-проекту Supabase командой `pnpm db:push`, запустите `pnpm dev`, настройте HTTPS-туннель для callback URL и откройте Mini App через Telegram. Для полного OAuth нужен одобренный Instagram username и аккаунт, добавленный в testers приложения Meta.

## Проверки

1. Для пользователя с одобренной, но ещё не подключённой Instagram-заявкой откройте подключение с кнопки на дашборде. → Открывается первый экран визарда: заголовок, скриншот приглашения, локализованные инструкции и кнопки «Открыть настройки Instagram» и «Далее».
2. Нажмите «Открыть настройки Instagram». → Telegram открывает `https://www.instagram.com/accounts/manage_access/`. Примите приглашение тестировщика для приложения `neuroposts-IG`, вернитесь в Mini App и нажмите «Далее». → Сразу открывается страница логина Instagram (отдельного промежуточного OAuth-экрана нет); «Далее» работает независимо от того, было ли приглашение принято ранее.
3. Завершите авторизацию в Instagram и вернитесь в Mini App. → Автоматически появляется экран «Подключение выполнено» с анимированной нарисованной галочкой и кнопкой «К дашборду».
4. Нажмите «К дашборду». → Открывается `/app`; повторное открытие `/app/connect-instagram` для уже активного подключения сразу показывает экран успеха.
5. На каждом экране визарда нажимайте Telegram BackButton. → На ошибке он возвращает к приглашению; на приглашении, успехе и ожидании возвращает к исходному экрану (дашборду или финальному экрану онбординга). Откройте затем любой другой раздел приложения. → Стрелка исчезает, Telegram снова показывает обычный крестик закрытия.
6. Пройдите онбординг до финального экрана, дождитесь статуса `ready` и нажмите «Подключить Instagram». → Открывается тот же визард; BackButton с первого экрана возвращает к финальному экрану онбординга.
7. Повторите проверки 1–6 с немецким языком Telegram. → Все новые заголовки, пояснения, кнопки и сообщения результата отображаются на немецком.

## Негативные проверки

1. Во время OAuth отмените выдачу доступа в Instagram. → Открывается экран «Не удалось подключить Instagram» с причиной отмены и кнопкой «Попробовать снова». Нажмите её. → Снова открывается страница логина Instagram; новый успешный проход приводит к экрану успеха.
2. Создайте новую или pending-заявку, не подтверждая её администратором, и откройте `/app/connect-instagram`. → Визард и кнопка OAuth недоступны; виден экран с часами «Ожидание подтверждения администратором» и пояснением о сообщении от бота.
3. После неуспешного OAuth убедитесь, что `GET /api/miniapp/ig/status` возвращает `connect: "error"`, затем снова откройте визард с дашборда. → Открывается шаг 1 с приглашением тестировщика, а не экран ошибки.

## Как понять, что тикет НЕ работает

- Нет скриншота или ссылка открывает не настройки доступа Instagram: проверьте наличие `public/images/ig-tester-invite.png` и вызов `openLink` в `app/(miniapp)/app/connect-instagram/page.tsx`.
- После OAuth остаётся прежняя панель или не появляется результат: во вкладке Network проверьте redirect callback на `/app/connect-instagram?ig=success|denied|error` и ответ `GET /api/miniapp/ig/status` (поле `connect`).
- OAuth доступен до одобрения либо вместо ожидания виден визард: проверьте `GET /api/miniapp/ig/status`, статус записи `ig_accounts` и ответ `GET /api/miniapp/ig/login-url` (для pending должен быть `403` с `not_approved`).
- Стрелка BackButton не скрывается после выхода: проверьте cleanup эффекта BackButton на странице визарда и повторите проверку внутри реального клиента Telegram, а не в обычном браузере.
