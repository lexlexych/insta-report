# Ручной тест T-004 · Слой репозиториев

## Что сделано

Добавлен серверный слой `src/lib/db`: lazy Supabase service-role клиент, типизированные репозитории для всех таблиц и общие ошибки `DbError`, `PendingExistsError`, `ForbiddenLabelError`.
Секреты Instagram шифруются через `encrypt()` перед записью и расшифровываются через `decrypt()` при чтении.
Автотесты по OVERRIDES не добавлялись; ключевые edge cases проверяются ручным demo-скриптом.

## Подготовка

- ветка: `git fetch && git checkout ticket/T-004 && pnpm install`
- env: для статической проверки реальные секреты не нужны; для запуска приложения/реальных репозиториев заполните `.env` по `.env.example`, особенно:
  - `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY` — из Supabase Project Settings → API;
  - `ENCRYPTION_KEY` — сгенерировать командой `pnpm gen:key`;
  - остальные переменные можно взять из `.env.example` и заполнить тестовыми значениями, если запускаете только lint/typecheck/demo.
- сервисы: для demo-скрипта Supabase поднимать не нужно; для проверки на реальной БД человек применяет миграции из T-002 командой `pnpm db:push` на своём dev-проекте Supabase.

## Проверки (Действие → Ожидаемо)

1. `pnpm lint` → команда завершается без ошибок; ESLint содержит правило, запрещающее импорт `@supabase/supabase-js` вне DB-клиента.
2. `pnpm typecheck` → TypeScript strict проходит без ошибок.
3. `pnpm tsx scripts/manual/T-004-demo.ts` → все строки начинаются с `OK`, в конце печатается `T-004 demo completed`.
4. Откройте `src/lib/db/client.ts` → файл начинается с `import 'server-only'`, клиент создаётся через `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY`, `persistSession: false`.
5. Откройте `src/lib/db/igConnections.ts` → `upsertForTenant` кладёт в БД поля `access_token_enc`/`app_secret_enc` только через `encrypt(...)`, а `getForTenant` возвращает `accessToken`/`appSecret` через `decrypt(...)`.
6. Откройте `src/lib/db/drafts.ts` → `claimPendingToSending(id)` делает `update({ status: 'sending' })` с условиями `id = ...` и `status = 'pending'`, затем `maybeSingle()`, поэтому уже не-pending черновик вернёт `null` без исключения.
7. Откройте `src/lib/db/processedEvents.ts` → `tryInsert` возвращает `false` для ошибки Supabase/Postgres `23505`, а остальные ошибки оборачивает в `DbError`.
8. Откройте `src/lib/db/labels.ts` → обновление имени на `Без категории` и удаление существующей метки с этим именем бросают `ForbiddenLabelError`.

## Негативные проверки

1. Временно добавьте в любой файл вне `src/lib/db/client.ts` строку `import { createClient } from '@supabase/supabase-js';` и запустите `pnpm lint` → ESLint должен завершиться ошибкой `no-restricted-imports`. После проверки откатите строку.
2. Временно замените в `scripts/manual/T-004-demo.ts` проверку `processedEvents.includes('return false')` на заведомо неверную строку и запустите `pnpm tsx scripts/manual/T-004-demo.ts` → скрипт должен напечатать `FAIL` и завершиться ошибкой. После проверки откатите изменение.

## Как понять, что тикет НЕ работает

- `pnpm lint` пропускает импорт `@supabase/supabase-js` вне `src/lib/db/client.ts` — правило restricted imports сломано или удалено.
- `pnpm tsx scripts/manual/T-004-demo.ts` печатает `FAIL` — один из обязательных edge cases не отражён в коде.
- В `ig_connections` при реальной проверке попадает plaintext access token/app secret — писать в БД нужно только через `igConnections.upsertForTenant`, а не напрямую.
- `claimPendingToSending` бросает ошибку для уже отправленного/отменённого черновика — должен возвращаться `null`, чтобы кнопки Telegram были идемпотентны.
