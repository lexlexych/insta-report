# Ручной тест T-050 · Zernio: отправка ответа кнопкой «Отправить» и гашение собственного echo

## Что сделано

Карточки черновиков, полученные через Zernio, теперь отправляют текст в ту же переписку через
Zernio Inbox API. Перед отправкой система проверяет, не ответил ли владелец вручную, а после
успеха регистрирует ID исходящего сообщения, поэтому собственный `message.sent` не создаёт новый
черновик. При закрытом 24-часовом окне однажды используется разрешённый тег `HUMAN_AGENT`.

## Подготовка

- ветка: `git fetch origin && git checkout zernio && pnpm install`
- env: заполните все обязательные переменные из `.env.example`; для этого теста особенно нужны
  `ZERNIO_API_KEY` и `ZERNIO_WEBHOOK_SECRET` (создаются в кабинете Zernio), а также
  `TELEGRAM_BOT_TOKEN`, `TELEGRAM_WEBHOOK_SECRET`, `SUPABASE_URL`, `SUPABASE_SECRET_KEY` и
  `APP_BASE_URL`. `ZERNIO_API_BASE` можно оставить значением по умолчанию.
- сервисы: примените ранее добавленные миграции к отдельному dev-проекту Supabase командой
  `pnpm db:push`; в кабинете Zernio укажите публичный HTTPS URL вебхука приложения и подключите
  Instagram-аккаунт. Запустите приложение: `pnpm dev`. Откройте Mini App через бота и завершите
  подключение Zernio так, чтобы аккаунт имел статус active.
- нужны два Instagram-аккаунта: бизнес-аккаунт, подключённый через Zernio, и отдельный аккаунт
  клиента, с которого можно отправлять DM.

## Проверки

1. С клиентского Instagram-аккаунта отправьте бизнес-аккаунту короткое DM. Дождитесь карточки
   черновика в Telegram и нажмите «Отправить». → Ответ появляется в этой переписке в Instagram;
   Telegram-карточка меняется на «✅ отправлено» без кнопок.
2. В Supabase Studio откройте `message_log` и `usage_stats` тенанта из шага 1. → В `message_log`
   есть новая строка с направлением `out` и текстом черновика, а `drafts_sent` увеличился ровно
   на 1.
3. После шага 1 подождите доставку Zernio-события `message.sent`. → В Telegram не появляется
   вторая карточка для только что отправленного текста. Если в журнале вебхуков Zernio виден
   `platformMessageId`, убедитесь, что он совпадает с зарегистрированным ID события; если ID
   ответа Zernio и `platformMessageId` различаются, зафиксируйте это в отчёте о пилоте: исходящий
   echo всё равно не должен создать карточку благодаря обработке outgoing-события.
4. Отправьте ещё одно DM от клиента и дождитесь карточки. До клика «Отправить» ответьте на это
   DM вручную из Instagram бизнес-аккаунта, затем нажмите кнопку в Telegram. → Нового ответа не
   уходит; карточка показывает «⚠️ Отменено: вы уже ответили вручную».
5. Запустите проверочный сценарий с моками:
   `pnpm exec tsx scripts/manual/T-050-demo.ts`. Если в вашей оболочке не находится `tsx`,
   используйте `node node_modules/tsx/dist/cli.mjs scripts/manual/T-050-demo.ts`. → В конце
   выводится `T-050 demo OK`; сценарии покрывают успешную Zernio-отправку, ручной ответ,
   повтор с `HUMAN_AGENT`, частично доставленный длинный текст и прежний Meta-путь.

## Негативные проверки

1. В кабинете Zernio отключите подключённый Instagram-аккаунт. На уже созданной карточке
   нажмите «Отправить». → Карточка показывает ошибку с предложением переподключить Instagram и
   кнопку «🔁 Повторить»; сообщение клиенту не уходит. После повторного подключения нажмите
   «🔁 Повторить». → Черновик повторно проходит проверку и может быть отправлен.
2. Временно уберите обе переменные `ZERNIO_API_KEY` и `ZERNIO_WEBHOOK_SECRET`, перезапустите
   приложение и нажмите «Отправить» на Zernio-карточке. → Карточка остаётся в статусе ошибки
   `zernio_disabled` с кнопкой повтора, а процесс не завершается исключением. Верните переменные
   после проверки.

## Как понять, что тикет НЕ работает

- Кнопка отправляет через Meta Graph API или требует Meta access token для Zernio-черновика —
  проверьте логи приложения и поле `drafts.provider`: для этой проверки оно должно быть `zernio`.
- После отправки появляется дубликат карточки — проверьте `processed_events` тенанта и журналы
  вебхуков Zernio: нужны ID исходящего сообщения и `message.sent`/`platformMessageId`.
- Ручной ответ не отменяет карточку — проверьте, что Inbox API возвращает исходящее сообщение
  позднее `trigger_ts`, и что Zernio-аккаунт остаётся active.
- Ошибка без кнопки «Повторить» или без понятного текста — проверьте статус и `error` в `drafts`,
  затем логи Zernio API (в частности 400 `PLATFORM_LIMITATION` и 429 rate limit).
