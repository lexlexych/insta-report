# Ручной тест T-041 · Онбординг 2.1 — маршрут без IG-шагов, кнопка «Назад», полноэкранное поле базы знаний

## Что сделано

Онбординг теперь проходит по маршруту `sphere → business → knowledge → finish → done`: заявка на Instagram всё так же создаётся на шаге бизнеса, но не задерживает пользователя на промежуточных экранах. На экранах бизнеса и базы знаний доступна кнопка «Назад».

Поле базы знаний занимает свободную высоту экрана, а длинный текст прокручивается только внутри него. Старые состояния `ig_wait` и `ig_connect` при открытии Mini App продолжаются с шага базы знаний.

## Подготовка

- ветка: `git fetch && git checkout epic/E11 && pnpm install`
- env: заполните переменные из `.env.example`, необходимые для запуска Mini App, Supabase и Telegram-бота; для проверки заявки укажите ID тестового администратора в `ADMIN_TELEGRAM_IDS`. Токен бота получите у BotFather, параметры Supabase — в настройках dev-проекта Supabase.
- сервисы: примените схему к dev-проекту командой `pnpm db:push`, запустите приложение через `pnpm dev`, затем откройте Mini App из Telegram тестовым пользователем. У администратора должен быть открыт чат с ботом для получения уведомления.

## Проверки

1. Для нового пользователя откройте онбординг, выберите сферу и нажмите «Далее» → открывается экран данных бизнеса; кнопки «Назад» на экране выбора сферы нет.
2. На экране бизнеса введите название и свободный валидный Instagram username, нажмите «Далее» → в Telegram каждому ID из `ADMIN_TELEGRAM_IDS` приходит уведомление о pending-заявке, а сразу в Mini App открывается «Опишите ваш бизнес»; экранов «Заявка подана» и «Подключите Instagram» нет.
3. Повторите шаг 2 с username, заранее одобренным и ещё не привязанным к этому тенанту → сразу открывается «Опишите ваш бизнес», без промежуточных экранов; аккаунт привязан к текущему тенанту.
4. На экране бизнеса заполните оба поля, нажмите «Назад» → открывается выбор сферы, а затем при повторном переходе на бизнес ранее введённые название и username сохранены. В Network виден PATCH `/api/miniapp/tenant` с `onboardingStep: "sphere"`.
5. На экране базы знаний измените текст и нажмите «Назад» → открывается экран бизнеса; снова нажмите «Далее» с тем же username → открывается база знаний с внесёнными изменениями. В Network виден PATCH с `onboardingStep: "business"`; повторная отправка не создаёт вторую заявку.
6. Вставьте в поле базы знаний текст длиннее высоты экрана, включая много строк, и прокрутите его → страница и нижняя панель кнопок не прокручиваются, прокручивается только содержимое textarea. Откройте экранную клавиатуру Telegram → кнопка «Завершить» остаётся доступна.
7. Завершите заполнение базы знаний и нажмите «Завершить» → маршрут переходит к `finish` (финальный экран «Готово! Что дальше?» — его содержимое проверяется тестом T-042; выход в разделы через пункты нижней навигации). Кнопки «Назад» нет на экранах сферы и финала.
8. Для тестового тенанта вручную установите в dev-БД `onboarding_step` в `ig_wait`, затем отдельно в `ig_connect`, перезапускайте Mini App после каждого изменения → в обоих случаях открывается экран базы знаний; после первого перехода назад или вперёд в Network отправляется только допустимое новое значение шага.

## Негативные проверки

1. На экране бизнеса укажите username, закреплённый за другим тенантом, и нажмите «Далее» → под полем появляется локализованная ошибка о занятом Instagram-аккаунте, экран бизнеса остаётся открыт, перехода к базе знаний и нового уведомления администраторам нет.
2. Отправьте PATCH `/api/miniapp/tenant` с `{"onboardingStep":"ig_wait"}` или `{"onboardingStep":"ig_connect"}` → API отвечает `400` с ошибкой malformed, новое legacy-состояние не сохраняется.

## Как понять, что тикет НЕ работает

- После экрана бизнеса отображается экран ожидания заявки или подключения Instagram, либо pending-заявка не создаёт уведомления администраторам: проверьте Network для `POST /api/miniapp/onboarding/ig-account` и логи приложения/бота.
- «Назад» отсутствует на business/knowledge, не меняет шаг в PATCH или очищает введённый текст: проверьте запросы `PATCH /api/miniapp/tenant` и состояние формы после возврата.
- Страница прокручивается вместе с нижней панелью или кнопка скрыта клавиатурой: проверьте computed styles `height: 100dvh`, `overflow` контейнера и textarea в DevTools.
- Legacy-тенант не открывает базу знаний: проверьте ответ `POST /api/miniapp/auth` и поле `onboardingStep` в Network.
