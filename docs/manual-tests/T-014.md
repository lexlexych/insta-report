# Ручной тест T-014 · Диагностика подключения Instagram

## Что сделано
Добавлен endpoint `/api/miniapp/ig/status`, который проверяет сохранённый Instagram Access Token живым вызовом Graph API, обновляет username/account id и переводит подключение в `active` или `error`.
В визарде подключения появился блок диагностики «токен · handshake · событие» с ручным обновлением и polling каждые 5 секунд до полного успеха. На дашборде добавлен компактный бейдж статуса подключения.

## Подготовка
- ветка: `git fetch && git checkout ticket/T-014 && pnpm install`
- env: заполните `.env.local` как для T-012/T-013: `APP_BASE_URL` с публичным HTTPS URL приложения, Supabase URL/keys, `CRYPTO_KEY_BASE64`, Telegram Mini App auth-переменные; для живой проверки нужен действующий Instagram Access Token из Meta App Dashboard и App Secret этого приложения.
- сервисы: примените миграции к dev-проекту Supabase (`pnpm db:push`), запустите приложение `pnpm dev`; для webhook handshake используйте публичный туннель на порт 3000 (например, `npx localtunnel --port 3000` или cloudflared) и выставьте его в `APP_BASE_URL`.

## Проверки
1. Откройте Telegram Mini App и перейдите в `/app/connect-instagram`.
   → До сохранения токена и секрета блок диагностики не показывается; после сохранения появляется карточка «Диагностика подключения» с кнопкой «Обновить».
2. Сохраните действующие `Access Token` и `App Secret`.
   → В строке «Токен доступа» после обновления отображается ✅; в Supabase в `ig_connections` заполнены `ig_account_id`, `ig_username`, а `status` становится `active`.
3. Не выполняя webhook verify в Meta Dashboard, нажмите «Обновить».
   → Строка «Вебхук подтверждён (handshake)» показывает ❌ и hint: проверить Callback URL/Verify Token без пробелов и нажать Verify and Save.
4. В Meta Dashboard вставьте Callback URL и Verify Token из шага Webhooks, нажмите Verify and Save.
   → После автообновления или кнопки «Обновить» строка handshake меняется на ✅.
5. Отправьте сообщение Instagram-аккаунту с другого профиля Instagram и нажмите «Обновить».
   → Если `webhook_last_seen_at` стал позже `handshake_at`, строка «Событие получено» показывает ✅; до этого она показывает ⏳ и hint отправить сообщение с другого профиля.
6. Когда все три строки ✅.
   → Появляется зелёный блок «Подключение работает! @{username}» и кнопка «На дашборд».
7. Перейдите на `/app`.
   → На дашборде виден компактный бейдж «Instagram подключён: @{username}» и ссылка обратно к настройке Instagram.
8. Оставьте вкладку диагностики открытой до полного успеха, затем наблюдайте Network 10–15 секунд.
   → После состояния «все ok» повторные запросы `/api/miniapp/ig/status` прекращаются; при уходе со страницы polling также прекращается.

## Негативные проверки
1. Замените Access Token на заведомо неверный или отозванный токен и нажмите «Обновить».
   → Строка «Токен доступа» показывает ❌ с инструкцией «Токен недействителен или отозван — сгенерируйте новый в Meta App Dashboard», а `ig_connections.status` становится `error`.
2. Отключите интернет/Graph API или временно подставьте недоступный endpoint через devtools/network blocking и нажмите «Обновить».
   → UI показывает «Не удалось обновить диагностику. Попробуйте ещё раз.» без утечки токена или секрета в экран/логи браузера.

## Как понять, что тикет НЕ работает
- Нет карточки диагностики после сохранения токена и App Secret: проверьте Network-запрос `/api/miniapp/ig/status` и auth Mini App.
- Токен валиден в Meta, но строка токена ❌: смотрите server logs для ошибки Graph API без вывода самого токена; проверьте, что токен именно Instagram API with Instagram Login.
- Handshake не становится ✅: проверьте, что `APP_BASE_URL` совпадает с публичным туннелем, Callback URL скопирован полностью, Verify Token без пробелов.
- Event остаётся ⏳: проверьте webhook subscription на поле `messages`, server logs `/api/wh/ig/[tenantId]` и поле `webhook_last_seen_at` в `ig_connections`.
