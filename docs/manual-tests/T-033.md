# Ручной тест T-033 · Business Login for Instagram

## Что сделано
- Добавлен OAuth-флоу Instagram API with Instagram Login: Mini App выдаёт signed state, callback меняет code на long-lived token, получает профиль, подписывает аккаунт на messages и сохраняет соединение как `platform_app`.
- Добавлен глобальный Instagram webhook `/api/wh/ig`, который проверяет платформенную подпись, маппит `entry[].id` на tenant по `ig_account_id` и передаёт событие в существующий pipeline.
- При выключенных `INSTAGRAM_*` переменных приложение остаётся в прежнем ручном `own_app` режиме.

## Подготовка
- Ветка: `git checkout ticket/T-033 && pnpm install`.
- БД: используйте dev-проект Supabase и примените миграцию `pnpm db:push`; должна примениться `supabase/migrations/0009_business_login.sql`.
- Запуск: `pnpm dev`; для реального OAuth/webhook нужен публичный HTTPS URL (`APP_BASE_URL`) через Vercel preview, cloudflared или другой туннель.
- Meta App:
  1. Создайте/откройте один платформенный Meta App и добавьте продукт Instagram.
  2. В Business Login settings добавьте redirect URI `{APP_BASE_URL}/api/ig/callback`.
  3. В Webhooks настройте callback `{APP_BASE_URL}/api/wh/ig`, verify token из `IG_WEBHOOK_VERIFY_TOKEN`, подпишитесь на поле `messages`.
  4. Добавьте рабочий профессиональный Instagram-аккаунт Instagram-тестировщиком и примите приглашение в Instagram.
  5. В Instagram включите доступ connected tools / Allow access to messages.
- Env: заполните обычные переменные проекта и дополнительно все три переменные `INSTAGRAM_APP_ID`, `INSTAGRAM_APP_SECRET`, `IG_WEBHOOK_VERIFY_TOKEN`; опционально `TELEGRAM_BOT_USERNAME` без `@` для кнопки возврата.

## Проверки
1. Откройте Mini App → «Подключение Instagram». Ожидаемо: сверху виден блок «Подключить через Instagram», а ручной wizard скрыт в аккордеоне «Продвинутый способ (вручную)».
2. Нажмите «Подключить Instagram», подтвердите consent в Instagram и дождитесь callback. Ожидаемо: HTML-страница «Instagram подключён» без отображения токенов/code/secret и с возвратом в Telegram.
3. Вернитесь в Mini App и нажмите «Проверить». Ожидаемо: статус показывает подключённый `@username`, кнопка становится «Переподключить».
4. В Supabase проверьте `ig_connections` для tenant. Ожидаемо: одна запись, `connection_mode = platform_app`, `ig_account_id` заполнен, `ig_username` заполнен, `status = active`, access token хранится только в `access_token_enc`.
5. Повторите подключение тем же tenant. Ожидаемо: запись обновляется идемпотентно, новые строки для tenant не плодятся; смена IG-аккаунта перезаписывает `ig_account_id`/`ig_username`.
6. Отправьте DM на подключённый IG-аккаунт с аккаунта-тестировщика в Development Mode. Ожидаемо: Meta вызывает `/api/wh/ig`, `webhook_last_seen_at` обновляется, в Telegram появляется карточка черновика.
7. Пилотная проверка Dev/Live из research §9: отправьте DM с постороннего аккаунта без роли в Development Mode. Ожидаемо: событие не приходит. Затем переведите Meta App в Live Mode и повторите. Ожидаемо для жизнеспособности схемы: событие приходит, карточка появляется в Telegram; если не приходит — зафиксируйте это как блокер пилота и проверьте роли, professional account, `subscribed_apps` и Allow access to messages.
8. Откройте ручной аккордеон и проверьте прежний `own_app` путь: сохраните test access token + app secret, настройте per-tenant webhook `/api/wh/ig/{tenantId}`. Ожидаемо: ручной flow и диагностика работают как раньше.
9. Уберите `INSTAGRAM_APP_ID`, `INSTAGRAM_APP_SECRET`, `IG_WEBHOOK_VERIFY_TOKEN` из env и перезапустите приложение. Ожидаемо: страница выглядит как прежний ручной wizard, `/api/miniapp/ig/login-url` возвращает 404 `{ ok: false, error: "disabled" }`.
10. Запустите cron `GET /api/cron/refresh-tokens` с корректным `CRON_SECRET` на окружении со старым `token_refreshed_at`. Ожидаемо: `platform_app` соединения попадают в ту же выборку refresh, что и `own_app`.

## Негативные проверки
1. Откройте callback с испорченным `state`: `{APP_BASE_URL}/api/ig/callback?state=bad&code=bad`. Ожидаемо: HTTP 403, вызовов к Meta нет.
2. Отмените consent в Instagram. Ожидаемо: страница «Доступ не выдан», вызовов обмена токена нет.
3. Отправьте POST на `/api/wh/ig` с неверным `X-Hub-Signature-256`. Ожидаемо: HTTP 401, событие не обрабатывается, секреты не печатаются в логах.
4. Отправьте webhook с неизвестным `entry[].id`, но корректной подписью. Ожидаемо: HTTP 200, debug-log о неизвестном аккаунте, Meta не ретраит.

## Как понять, что тикет НЕ работает
- OAuth возвращает ошибку redirect URI: проверьте точное совпадение `{APP_BASE_URL}/api/ig/callback` в Meta Dashboard и `APP_BASE_URL` в env.
- Callback пишет «Не получилось»: смотрите server logs по префиксу `[ig/callback]`; там не должно быть access token, code или app secret.
- Webhook verification получает 403: сравните `IG_WEBHOOK_VERIFY_TOKEN` в env и Meta Dashboard.
- Webhook POST получает 401: проверьте, что Meta App Secret в `INSTAGRAM_APP_SECRET` относится именно к этому Instagram App.
- DM не создаёт карточку: проверьте `ig_connections.ig_account_id`, `webhook_last_seen_at`, режим Dev/Live, принятую роль тестировщика и включённый Allow access to messages.
