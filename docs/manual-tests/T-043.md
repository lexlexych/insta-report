# Ручной тест T-043 · Предодобрение Instagram-аккаунтов и статус подключения

## Что сделано

Администратор может заранее одобрить Instagram username командой `/ig_add`. Пользователь с одобренным именем может начать OAuth, а до одобрения видит ожидание администратора. Дашборд показывает единый статус подключения и не предлагает OAuth до одобрения.

## Подготовка

- ветка: `git fetch && git checkout ticket/T-043 && pnpm install`
- env: заполните переменные из `.env.example`, необходимые для запуска приложения: URL и ключ Supabase dev-проекта, токен и username Telegram-бота (из BotFather), `ADMIN_TELEGRAM_IDS` с Telegram ID проверяющего администратора, а также Instagram App ID и секрет из Meta Developers.
- сервисы: примените миграции к отдельному dev-проекту Supabase командой `pnpm db:push`, затем запустите приложение командой `pnpm dev`. В BotFather укажите URL Mini App на запущенное приложение; для локального URL используйте HTTPS-туннель.

## Проверки

1. От имени Telegram ID из `ADMIN_TELEGRAM_IDS` отправьте боту `/ig_add preapproved_account` до создания пользователя. → Бот в том же чате отвечает `✅ Подтверждено: @preapproved_account`.
2. Откройте ботом Mini App нового пользователя, пройдите онбординг и на шаге 2 введите `preapproved_account`. → Шаг сохраняется без ожидания администратора; в `ig_accounts` та же запись получает tenant пользователя, а `GET /api/miniapp/ig/status` возвращает `connect: "ready"`.
3. Откройте дашборд этого пользователя. → Показана кнопка «Подключить Instagram», она открывает `/app/connect-instagram`; запрос `GET /api/miniapp/ig/login-url` возвращает URL OAuth.
4. Для другого нового пользователя пройдите шаг 2 с уникальным username. → Администраторы получают заявку с кнопкой «Подтвердить», а на дашборде пользователя показаны `⏳ Ожидание подтверждения администратором` и нет ссылок или кнопок для подключения.
5. Одобрите эту заявку командой `/ig_add <тот же username>`. → Бот подтверждает имя в чате администратора; владельцу приходит уведомление со ссылкой `startapp=connect`; после обновления дашборда `connect` равен `ready`.
6. Повторите сценарий из предыдущего пункта, но используйте кнопку «Подтвердить» в сообщении заявки. → Владелец получает такое же уведомление, а его `connect` становится `ready`.
7. Завершите OAuth для одобренного аккаунта, затем проверьте `GET /api/miniapp/ig/status` и дашборд. → API возвращает `connect: "active"`; дашборд сохраняет отображение подключённого аккаунта. При ошибке подключения API и дашборд показывают `error` и ссылку на экран подключения.
8. Откройте дашборд пользователя без строки в `ig_accounts`. → Показано нейтральное «Instagram не подключён» без кнопки или ссылки подключения; `connect` равен `none`.

## Негативные проверки

1. Отправьте `/ig_add bad username!` от администратора. → Бот отвечает «Некорректный username», запись не создаётся.
2. Отправьте `/ig_add preapproved_account` повторно. → Бот отвечает «Уже подтверждено», данные записи не меняются.
3. Отправьте `/ig_add any_valid_name` от Telegram ID, отсутствующего в `ADMIN_TELEGRAM_IDS`. → Бот не выполняет команду и не создаёт запись.
4. Для пользователя с pending-заявкой вызовите `GET /api/miniapp/ig/login-url`. → HTTP 403 с телом `{ "code": "not_approved" }`; OAuth не открывается.

## Как понять, что тикет НЕ работает

- У пользователя с pending-заявкой появляется кнопка подключения или `login-url` возвращает OAuth URL: проверьте Network для `ig/status` и `ig/login-url`, а также статус строки `ig_accounts` в Supabase.
- Команда `/ig_add` доступна не-администратору либо повторный вызов меняет запись: проверьте `ADMIN_TELEGRAM_IDS`, Telegram-логи и поля `status`, `approved_at`, `approved_by_tg_id` в `ig_accounts`.
- После одобрения не приходит уведомление: проверьте `tg_chat_id` tenant, `TELEGRAM_BOT_USERNAME` и логи приложения.
