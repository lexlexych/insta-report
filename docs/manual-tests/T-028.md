# Ручной тест T-028 · Настройки + GDPR-действия

## Что сделано
Добавлена полноценная страница `/app/settings`: редактирование базы знаний и системного промпта, переключение языка RU/DE, отключение Instagram и ссылки на правовые страницы. Добавлены GDPR-действия: удаление аккаунта с Telegram-прощанием, очисткой cookie и явным каскадным удалением tenant-данных.

## Подготовка
- ветка: `git fetch && git checkout ticket/T-028 && pnpm install`
- env: заполнить стандартные переменные Mini App/БД/Telegram из `.env.example`: `APP_BASE_URL`, `MINIAPP_JWT_SECRET`, Supabase URL/service key, `TELEGRAM_BOT_TOKEN`.
- сервисы: применить миграции в dev Supabase через `pnpm db:push`, затем запустить `pnpm dev`; Mini App открыть из Telegram-бота.
- тестовые данные: пройти `/app/onboarding`, сохранить базу знаний, при желании заполнить Instagram-токен на странице подключения.

## Проверки
1. Откройте `/app/settings`. Ожидаемо: видны секции «База знаний», «Язык интерфейса», «Instagram», «Правовая информация», «Опасная зона».
2. В секции «База знаний» нажмите «Развернуть», затем «Редактировать». Измените текст, убедитесь, что счётчик показывает максимум `/20000`, нажмите «Сохранить». Ожидаемо: запрос `PUT /api/miniapp/knowledge` успешен, превью обновлено, в таблице `tenants.knowledge_base` новое значение.
3. Раскройте «Системный промпт» и сохраните небольшое изменение. Ожидаемо: `tenants.system_prompt` обновлён, пользовательские секреты в ответ API не попадают.
4. Переключите язык на Deutsch, перезагрузите Mini App. Ожидаемо: интерфейс остаётся немецким, в `tenants.ui_locale = 'de'`; переключение обратно на русский сохраняет `ui_locale = 'ru'`.
5. Если Instagram был подключён, нажмите «Отключить» и подтвердите. Ожидаемо: `POST /api/miniapp/ig/disconnect` успешен; в `ig_connections` для tenant `access_token_enc = null`, `app_secret_enc = null`, `status = 'pending'`, `handshake_at = null`; карточка показывает состояние «не подключено/требует настройки».
6. Проверьте ссылки `/legal/privacy` и `/legal/terms`. Ожидаемо: Mini App открывает URL, даже если полноценный контент будет добавлен в T-032.
7. Для удаления введите слово подтверждения по текущей локали: `УДАЛИТЬ` для ru или `LÖSCHEN` для de, нажмите удаление и подтвердите браузерный диалог. Ожидаемо: бот сначала присылает сообщение «Все данные удалены. /start — начать заново», затем Mini App показывает экран «Данные удалены», cookie `session` очищена.
8. В Supabase выполните выборки по tenant id в таблицах `ig_connections`, `labels`, `drafts`, `message_log`, `usage_stats`, `processed_events`, `tenants`. Ожидаемо: строк этого tenant нет ни в одной таблице.
9. Отправьте `/start` боту и снова откройте Mini App. Ожидаемо: создаётся чистый аккаунт без старой базы знаний, labels, drafts, логов и Instagram-секретов.

## Негативные проверки
1. В «Опасной зоне» введите любое слово кроме точного `УДАЛИТЬ`/`LÖSCHEN`. Ожидаемо: кнопка удаления disabled и запрос `DELETE /api/miniapp/tenant` не отправляется.
2. Попробуйте сохранить пустую базу знаний или системный промпт. Ожидаемо: API возвращает ошибку валидации, старые значения в БД остаются.
3. После отключения Instagram повторно откройте `/app/connect-instagram`. Ожидаемо: секреты не отображаются и не восстанавливаются; требуется ввести новые значения.

## Как понять, что тикет НЕ работает
- После удаления остались строки tenant в любой tenant-scoped таблице — смотреть `src/lib/db/tenants.ts`, список явного удаления в `deleteCascade`.
- Telegram-прощание не приходит или приходит после удаления — смотреть логи `DELETE /api/miniapp/tenant` и `src/lib/tg/api.ts`.
- Язык сбрасывается после перезагрузки — проверить миграцию `0007_tenant_ui_locale.sql`, ответ `/api/miniapp/auth` и PATCH `/api/miniapp/tenant`.
- Отключение Instagram оставляет `access_token_enc`/`app_secret_enc` ненулевыми — смотреть `src/lib/db/igConnections.ts` и `/api/miniapp/ig/disconnect`.
