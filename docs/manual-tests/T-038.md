# Ручной тест T-038 · Онбординг 2.0 — полноэкранный визард

## Что сделано

Онбординг заменён полноэкранным визардом: сфера, данные бизнеса и Instagram, ветка заявки или OAuth, база знаний и финальная справка. База знаний начинается с локализованного шаблона выбранной сферы, а финал сохраняет её вместе со статическим системным промптом и стандартной меткой.

## Подготовка

- Ветка: `git fetch && git checkout ticket/T-038 && pnpm install`.
- env: заполните значения из `.env.example`, необходимые для запуска Mini App, Supabase, Telegram-бота и Instagram Business Login. Токен бота создаётся в BotFather; учётные данные Instagram приложения берутся в Meta for Developers.
- Сервисы: примените миграции в dev-проект Supabase командой `pnpm db:push`, затем запустите `pnpm dev`. Откройте Mini App из Telegram. Для OAuth задайте доступный извне URL приложения и callback в Meta.

## Проверки

1. Откройте Mini App новым пользователем → открывается только полноэкранный онбординг без нижней навигации; внизу видна одна неактивная кнопка «Далее».
2. Выберите одну сферу → карточка выделяется, «Далее» становится активной; после нажатия открываются данные бизнеса, а повторное открытие Mini App возвращает на этот шаг.
3. Введите имя из двух и более символов и свободный корректный `@username` → «Далее» активна; после нажатия отображается экран «Заявка подана», затем активная кнопка ведёт к шаблону базы знаний.
4. На экране базы знаний → текст уже заполнен шаблоном выбранной сферы на языке Telegram; замените данные, нажмите «Завершить» → показан финальный экран со справкой о тест-чате, входящих, категориях, настройках и дашборде.
5. Нажмите «Начать работу» → открывается дашборд и нижняя навигация снова видна. В базе данных у tenant сохранены `knowledge_base`, статичный `system_prompt`, `onboarding_step = done`, а также создана метка «Без категории».
6. Для заранее подтверждённого, но ещё не привязанного username пройдите первые два шага → открывается экран подключения Instagram. Завершите реальный Instagram Business Login, вернитесь в Mini App и нажмите «Проверить» → после статуса `active` кнопка «Далее» активна и ведёт к базе знаний.
7. Закройте и снова откройте Mini App на каждом из шагов, включая «Заявка подана» и подключение Instagram → открывается сохранённый незавершённый шаг.
8. Откройте кнопку подключения из сообщения администратора → Mini App открывает экран подключения Instagram.
9. Повторите начало онбординга с Telegram на русском, затем на немецком (или другом языке) → все экраны и шаблон базы знаний соответственно русские или немецкие; после первого экрана `tenants.ui_locale` совпадает с языком Telegram.

## Негативные проверки

1. На каждом экране оставьте обязательные данные пустыми → нижняя кнопка остаётся disabled и визуально приглушённой.
2. На шаге бизнеса введите имя из одного символа, недопустимый username или username, занятый другим tenant → переход недоступен либо показана ошибка под полем; шаг не меняется.
3. На шаге базы знаний удалите весь текст или вставьте более 20 000 символов → «Завершить» неактивна.
4. Отмените OAuth или вернитесь без подключения → «Далее» на экране подключения остаётся неактивной.

## Как понять, что тикет НЕ работает

Если видна нижняя навигация до завершения, потерян текущий шаг после переоткрытия, шаблон не соответствует сфере/языку или можно пройти без обязательных данных — проверьте запросы Network к `/api/miniapp/tenant`, `/api/miniapp/onboarding/ig-account` и `/api/miniapp/onboarding/complete`. При проблеме OAuth проверьте callback и URL из `/api/miniapp/ig/login-url`; при проблеме финализации — записи tenant и метку в Supabase.
