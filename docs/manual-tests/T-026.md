# Ручной тест T-026 · UI тест-чата

## Что сделано
Страница `/app/simulator` заменена с заглушки на интерфейс мессенджера: пузырьки клиента и ассистента, бейдж категории, typing-индикатор, примеры первого вопроса и поле ввода с Enter/Shift+Enter. История сохраняется в `sessionStorage`, закрываемая UX-подсказка — в `localStorage`, а запрос отправляет весь диалог в `/api/miniapp/simulator/message`.

## Подготовка
- ветка: `git fetch && git checkout ticket/T-026 && pnpm install`
- env: заполните стандартные переменные Mini App/Supabase/LLM из `.env.example`; для проверки лимита можно временно поставить `SIMULATOR_DAILY_LIMIT=3`.
- данные: в Mini App должен быть пользователь с завершённым онбордингом и сохранённой базой знаний. Для негативной проверки onboarding можно открыть пользователя без `knowledge_base` или очистить базу знаний в dev-проекте Supabase.
- сервисы: примените миграции в dev-проект Supabase через `pnpm db:push`, затем запустите `pnpm dev` и откройте Mini App в Telegram или локально с валидной miniapp-сессией.

## Проверки
1. Откройте `/app/simulator` с пустой историей. Ожидаемо: виден заголовок «Тест-чат», закрываемый баннер «Вы пишете как клиент…», кнопка «⟲ Начать заново», три чипа-примера и пустое поле ввода.
2. Нажмите чип «Сколько стоит…?». Ожидаемо: текст подставился в textarea, сообщение ещё не отправлено.
3. Нажмите Enter или кнопку отправки. Ожидаемо: справа появляется пузырь клиента, кнопка и textarea временно disabled, слева показывается typing-индикатор, затем появляется пузырь ассистента с бейджем `🏷 <категория>` и текстом черновика.
4. Отправьте ещё минимум 4 сообщения подряд. Ожидаемо: в DevTools → Network → запрос `/api/miniapp/simulator/message` содержит JSON `messages` со всей историей клиент/ассистент, а диалог визуально автоскроллится к последнему сообщению.
5. Введите многострочный текст: `строка 1`, затем Shift+Enter, затем `строка 2`, затем Enter. Ожидаемо: Shift+Enter добавляет перенос, Enter отправляет сообщение, textarea растёт не выше 4 строк.
6. Обновите страницу после нескольких сообщений. Ожидаемо: история восстановилась из `sessionStorage`.
7. Закройте подсказку крестиком и обновите страницу. Ожидаемо: подсказка не появляется повторно, потому что флаг сохранён в `localStorage`.
8. Нажмите «⟲ Начать заново» и подтвердите очистку. Ожидаемо: история и поле ввода очищены, снова видны чипы первого входа.

## Негативные проверки
1. Установите `SIMULATOR_DAILY_LIMIT=3`, перезапустите `pnpm dev` и отправьте сообщения сверх лимита. Ожидаемо: вместо падения интерфейса появляется системный пузырь «Дневной лимит тест-чата исчерпан, продолжите завтра».
2. Откройте `/app/simulator` пользователем без базы знаний. Ожидаемо: после отправки первого сообщения показывается экран-заглушка «Сначала пройдите онбординг» с кнопкой «Пройти онбординг», кнопка ведёт на `/app/onboarding`.
3. В DevTools временно заблокируйте запрос `/api/miniapp/simulator/message` или выключите dev-сервер после загрузки страницы и отправьте сообщение. Ожидаемо: появляется системный пузырь «Ошибка, попробуйте ещё раз» и кнопка «Повторить последнее сообщение»; интерфейс остаётся работоспособным.

## Соответствие критериям приёмки
- «Диалог 5+ сообщений сохраняет контекст» — покрыто проверкой 4: в Network видно, что весь массив `messages` уходит в запрос.
- «Все ошибки бэкенда обработаны нефатально» — покрыто негативными проверками 1–3: лимит, onboarding_required и неизвестная ошибка не ломают экран.
- «UX-подсказка объясняет связь c реальным потоком» — покрыто проверками 1 и 7.

## Как понять, что тикет НЕ работает
- Нет пузыря ассистента или бейджа категории после успешного ответа: проверьте Network payload/response для `/api/miniapp/simulator/message` и логи `pnpm dev`.
- После обновления пропадает история: проверьте Application → Session Storage → ключ `instareply:simulator:messages`.
- Подсказка появляется после закрытия: проверьте Application → Local Storage → ключ `instareply:simulator:hint-dismissed`.
- Enter не отправляет или Shift+Enter не переносит строку: проверьте фокус в textarea и ошибки React в консоли браузера.
