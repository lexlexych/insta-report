# Ручной тест T-018 · Классификация обращения по меткам

## Что сделано
- Добавлен промпт классификатора: LLM получает список категорий, историю диалога и новое обращение и должен вернуть JSON `{"label":"..."}`.
- Добавлен модуль `src/lib/pipeline/classify.ts`, который выбирает метку по ответу LLM без учёта регистра и пробелов.
- Если ответ пустой/неизвестный или LLM вернул ошибку, пайплайн безопасно выбирает «Без категории»; если у тенанта только «Без категории», LLM не вызывается.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-018 && pnpm install`.
- Env: для демо-скрипта реальные ключи не нужны; скрипт сам выставляет dummy-значения и не делает сетевых LLM-вызовов.
- Сервисы: внешние сервисы и Supabase не нужны.

## Проверки
1. Выполнить `pnpm lint` → ожидаемо: ESLint завершается без ошибок.
2. Выполнить `pnpm typecheck` → ожидаемо: TypeScript завершается без ошибок.
3. Выполнить `pnpm tsx scripts/manual/T-018-demo.ts` → ожидаемо: все строки начинаются с `OK`, в конце напечатано `OK T-018 demo completed`.
4. В выводе демо найти строку `OK точное имя выбирает метку` → ожидаемо: точный ответ LLM `Доставка` выбирает метку `Доставка`.
5. В выводе демо найти строку `OK регистр и пробелы нормализуются` → ожидаемо: ответ LLM с другим регистром и пробелами выбирает существующую метку `Возврат`.
6. В выводе демо найти строку `OK единственная Без категории возвращается без LLM` и `OK LLM вызван только для первого сценария с calls` → ожидаемо: при единственной метке «Без категории» LLM не вызывается.

## Негативные проверки
1. В выводе демо найти строки `OK выдуманная метка падает в Без категории` и `OK пустая строка падает в Без категории` → ожидаемо: неизвестный или пустой ответ LLM выбирает «Без категории».
2. В выводе демо найти строку `OK LlmError не пробрасывается наружу` → ожидаемо: ошибка LLM не роняет классификацию, возвращается «Без категории».

## Покрытие критериев приёмки
- «Классификация никогда не бросает исключение наружу» → проверяется негативным сценарием `LlmError не пробрасывается наружу`.
- «Экономия вызова при единственной метке реализована» → проверяется строками про единственную «Без категории» и счётчик вызовов LLM.

## Как понять, что тикет НЕ работает
- `pnpm tsx scripts/manual/T-018-demo.ts` печатает хотя бы одну строку `FAIL` или завершается с ошибкой.
- Неизвестный/пустой ответ LLM выбирает не «Без категории».
- При одной метке «Без категории» демо падает с ошибкой `LLM must not be called for the only default label`.
- При ошибке LLM скрипт падает вместо строки `OK LlmError не пробрасывается наружу`.
