# Ручной тест T-010 · Генерация базы знаний и system prompt

## Что сделано

Добавлена серверная генерация базы знаний и системного промпта через LLM для онбординга Mini App. Результат сохраняется в `tenants`, повторная генерация без подтверждения возвращает 409, а при успешной генерации вызывается сид метки «Без категории».

## Подготовка

- Ветка: `git checkout work && pnpm install` (в этом окружении удалённый `origin` недоступен; в GitHub используйте ветку PR для T-010).
- Env: заполните переменные из `.env.example`, необходимые для Mini App, Supabase service role и LLM (`LLM_API_KEY`, `LLM_BASE_URL`, `LLM_MODEL_DRAFT`, `LLM_MODEL_CLASSIFY`). Ключ LLM возьмите у выбранного OpenAI-compatible провайдера.
- БД: примените миграции в dev-проект Supabase командой `pnpm db:push`.
- Сервисы: запустите приложение `pnpm dev` и откройте Mini App как Telegram-пользователь, для которого проходит `POST /api/miniapp/auth`.

## Проверки

1. Откройте `/app/onboarding`, заполните название организации и подробное описание бизнеса длиной больше 80 символов, затем запустите генерацию.
   → Ожидаемо: запрос `POST /api/miniapp/onboarding/generate` возвращает 200 с JSON `{ knowledgeBase, systemPrompt }`; в `tenants` заполнены `org_name`, `org_description`, `knowledge_base`, `system_prompt`, `onboarding_step = review_kb`.

2. Проверьте структуру `knowledge_base` в ответе или в Supabase.
   → Ожидаемо: markdown содержит разделы «О компании», «Услуги и цены», «Практическая информация», «Тон общения», «Частые вопросы»; отсутствующие адрес/часы/контакты/цены помечены как «уточнить у владельца», а не выдуманы.

3. Проверьте `system_prompt` в ответе или в Supabase.
   → Ожидаемо: prompt инструктирует ассистента представляться от имени организации, отвечать кратко и дружелюбно на языке клиента, опираться только на базу знаний и не выдумывать цены/обещания.

4. Проверьте сид метки по умолчанию SQL-запросом в Supabase SQL Editor:
   ```sql
   select name, description, instruction, sort
   from labels
   where tenant_id = '<tenant_id>'
   order by sort;
   ```
   → Ожидаемо: среди меток есть ровно одна «Без категории» с `sort = 999`; повторная успешная генерация не создаёт дубль.

5. Выполните `GET /api/miniapp/knowledge` из авторизованной Mini App-сессии.
   → Ожидаемо: возвращаются текущие `knowledgeBase` и `systemPrompt`.

6. Отредактируйте базу знаний и/или системный prompt через UI ревью или запрос `PUT /api/miniapp/knowledge` с телом `{"knowledgeBase":"...","systemPrompt":"..."}`.
   → Ожидаемо: API возвращает `{ "ok": true }`, в `tenants.knowledge_base` и `tenants.system_prompt` сохранены новые значения; можно отправить только одно из двух полей.

7. Обновите шаг онбординга запросом `PATCH /api/miniapp/tenant` с телом `{"onboardingStep":"done"}`.
   → Ожидаемо: API возвращает `{ "ok": true, "tenant": ... }`, в БД `onboarding_step = done`.

## Негативные проверки

1. Повторите генерацию для тенанта, у которого уже заполнен `knowledge_base`, без поля `overwrite` или с `"overwrite": false`.
   → Ожидаемо: `POST /api/miniapp/onboarding/generate` возвращает HTTP 409 и тело `{ "code": "exists" }`; сохранённые `knowledge_base` и `system_prompt` не изменяются.

2. Повторите генерацию с теми же данными и `"overwrite": true`.
   → Ожидаемо: API возвращает 200, `knowledge_base` и `system_prompt` перезаписаны, метка «Без категории» не дублируется.

3. Отправьте некорректное тело генерации: пустой `orgName`, `orgDescription` короче 80 символов или не-JSON.
   → Ожидаемо: HTTP 400 `{ "ok": false, "error": "malformed" }`, LLM не вызывается и БД не обновляется.

4. Отправьте `PUT /api/miniapp/knowledge` с `knowledgeBase` или `systemPrompt` длиннее 20 000 символов либо с пустым телом `{}`.
   → Ожидаемо: HTTP 400 `{ "ok": false, "error": "malformed" }`.

## Как понять, что тикет НЕ работает

- `POST /api/miniapp/onboarding/generate` возвращает 500: проверьте логи `pnpm dev`, значения `LLM_*` в `.env` и доступность OpenAI-compatible провайдера.
- В БД нет `system_prompt`: проверьте Network-ответ генерации и колонку `tenants.system_prompt`.
- Повторная генерация молча перезаписывает данные без `overwrite: true`: проверьте условие 409 в route генерации.
- Метка «Без категории» отсутствует или дублируется: проверьте миграцию `seed_default_labels` и вызов RPC после успешного сохранения генерации.
- Редактирование knowledge/system prompt не сохраняется: проверьте запрос `PUT /api/miniapp/knowledge` во вкладке Network и строки `tenants` в Supabase.
