# Ручной тест T-042 · Финальный экран «Готово! Что дальше?»

## Что сделано

После завершения онбординга открывается финальный экран с настоящей нижней навигацией и кратким описанием каждого раздела. Онбординг остаётся на этом экране после перезапуска Mini App, пока пользователь не выберет раздел или не начнёт подключение Instagram. Блок над навигацией зависит от статуса заявки Instagram.

## Подготовка

- ветка: `git fetch && git checkout ticket/T-042 && pnpm install`
- env: заполните переменные для локального запуска Mini App и авторизации Telegram из `.env.example`; для проверки статусов заявки нужен тестовый пользователь Telegram и запись его Instagram-заявки в dev-проекте Supabase.
- сервисы: примените миграции к dev-проекту Supabase командой `pnpm db:push`, затем запустите `pnpm dev` и откройте Mini App через тестового Telegram-бота.

## Проверки (каждая: Действие → Ожидаемо)

1. Пройдите онбординг до шага редактирования базы знаний и нажмите «Завершить». → Открывается экран «Готово! Что дальше?»; кнопки «Начать работу» нет, внизу видна навигация из четырёх пунктов.
2. На финальном экране сравните список с нижней навигацией. → Для Дашборда, Тест-чата, Категорий и Настроек показаны те же иконки, названия и понятные описания; сверху есть пояснение, что черновик входящего Instagram Direct придёт в Telegram и отправляется одним нажатием.
3. Закройте и снова откройте Mini App, не нажимая пункты навигации. → Снова открывается финальный экран, а не дашборд; нижняя навигация остаётся видимой.
4. Для нового тестового тенанта без подтверждённой заявки или с pending-заявкой откройте финальный экран. → Над навигацией есть строка с иконкой часов «Ожидание подтверждения администратором», кнопки подключения нет.
5. Подтвердите Instagram-ник тестового тенанта администратором, но не выполняйте OAuth-подключение; перезапустите Mini App и откройте finish. → Видна только кнопка «Подключить Instagram». Нажмите её. → Открывается `/app/connect-instagram`, а после следующей загрузки Mini App онбординг больше не показывается.
6. Для тенанта с активным подключением Instagram откройте finish. → Вместо кнопки показан бейдж «Instagram подключён».
7. На отдельном новом тестовом тенанте повторите шаг 1, затем по очереди нажмите каждый из четырёх пунктов нижней навигации. → Каждый пункт открывает свой раздел; после перезапуска Mini App открывается обычный раздел приложения, а не finish, поскольку шаг стал `done`.
8. Повторите проверки 1–7 с немецким языком Telegram. → Все новые строки финального экрана, статуса и кнопки отображаются на немецком.

## Негативные проверки (минимум одна: неверный ввод/подпись/токен → корректная ошибка)

1. До подтверждения Instagram-заявки попробуйте найти действие подключения на finish. → Кнопка «Подключить Instagram» отсутствует; пользователь не может перейти к OAuth до статуса `ready`.

## Как понять, что тикет НЕ работает (типовые симптомы и куда смотреть: логи, вкладка Network, БД)

Если после «Завершить» видна старая кнопка или нет нижней навигации, проверьте страницу `/app/onboarding` в DevTools. Если не меняется статус блока, во вкладке Network проверьте ответ `GET /api/miniapp/ig/status` и поле `connect`. Если после выбора раздела finish открывается снова, проверьте успешность `PATCH /api/miniapp/tenant` с `{ "onboardingStep": "done" }` и поле `tenants.onboarding_step` в dev-проекте Supabase.
