# Ручной тест T-040 · BUG: язык мини-аппа не наследуется из Telegram

## Что сделано

Исправлено чтение launch-параметров Telegram: язык интерфейса и deep-link брались из
несуществующих полей SDK (`initDataUnsafe`/`startParam`), из-за чего все новые пользователи
получали немецкий интерфейс, а deep-link `startapp=connect` не срабатывал. Теперь язык до
завершения онбординга приходит сервер-авторитетно (`tgLocale` из валидированных init-данных),
клиент читает реальные поля SDK (`tgWebAppData`, `tgWebAppStartParam`), а после онбординга язык
по-прежнему берётся из `tenants.ui_locale`.

## Подготовка

- Ветка: `git fetch && git checkout epic/E10 && pnpm install`.
- env: заполните значения из `.env.example`, необходимые для запуска Mini App, Supabase и
  Telegram-бота. Токен бота создаётся в BotFather.
- Сервисы: примените миграции в dev-проект Supabase (`pnpm db:push`), запустите `pnpm dev`,
  откройте Mini App из Telegram.
- Понадобятся два Telegram-аккаунта (или смена языка Telegram): один с языком интерфейса
  **русский**, другой — с **немецким** (или любым не-русским, например English).

## Проверки

1. Новым пользователем откройте Mini App из Telegram-аккаунта с **русским** языком интерфейса →
   онбординг и вся навигация на русском (кнопки «Далее», заголовки шагов). До этого баг давал
   немецкий. (Критерий: RU Telegram → RU интерфейс до онбординга.)
2. Новым пользователем откройте Mini App из аккаунта с **не-русским** языком (немецкий/English) →
   интерфейс немецкий. (Критерий: не-RU → `de`, фолбэк не изменился.)
3. В Network найдите ответ `POST /api/miniapp/auth` → в JSON присутствует поле верхнего уровня
   `tgLocale` (`ru` для п.1, `de` для п.2), рядом с `tenant`, а не внутри него.
4. Завершите онбординг у русского пользователя, в настройках смените язык интерфейса на немецкий
   и сохраните → переоткройте Mini App: интерфейс немецкий (после онбординга язык берётся из
   `tenants.ui_locale`, а не из языка Telegram). Затем верните русский — язык снова русский.
   (Критерий: после онбординга язык из `ui_locale`, регресса нет.)
5. Deep-link: пройдите сценарий подтверждения IG-аккаунта админом (T-037) и откройте Mini App по
   кнопке «Подключить Instagram» из сообщения бота (ссылка `startapp=connect`) → приложение
   открывает экран `/app/connect-instagram`. (Критерий: deep-link `connect` работает.)

## Негативные проверки

1. `grep -rn "initDataUnsafe\|\.startParam" src app` → пусто (нет обращений к несуществующим
   полям SDK).
2. У пользователя, уже завершившего онбординг с сохранённым `ui_locale`, откройте Mini App из
   Telegram-аккаунта с **другим** языком (например ui_locale=ru, Telegram=de) → интерфейс
   остаётся на сохранённом `ui_locale` (русском), язык Telegram НЕ перебивает выбор.
3. Обычный запуск без `startapp` (открытие Mini App из меню бота) → нового редиректа на
   `/app/connect-instagram` не происходит; работает штатная маршрутизация онбординг/дашборд.

## Как понять, что тикет НЕ работает

Если новый русский пользователь всё ещё видит немецкий интерфейс — проверьте, что в ответе
`POST /api/miniapp/auth` есть `tgLocale: "ru"` и что `TenantProvider` применяет его до онбординга;
проверьте, что клиент читает `launchParams.tgWebAppData?.user?.language_code`, а не `initDataUnsafe`.
Если deep-link `connect` не открывает экран подключения — проверьте, что читается
`launchParams.tgWebAppStartParam` (camelCase-режим `useLaunchParams(true)`), а не `startParam`.
Если после онбординга язык «прыгает» на язык Telegram — проверьте ветку `onboardingStep === 'done'`
в `TenantProvider`.
