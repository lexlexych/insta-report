# Ручной тест T-013 · Клиент Instagram Graph API

## Что сделано
- Добавлен типизированный клиент Instagram Graph API в `src/lib/ig/client.ts` с единственной точкой вызова `https://graph.instagram.com/v25.0`.
- Добавлено UTF-8 byte-safe разбиение длинных сообщений в `src/lib/ig/split.ts` и последовательная отправка всех частей.
- Для библиотечного модуля без UI добавлен demo-скрипт `scripts/manual/T-013-demo.ts`, который покрывает ключевые edge cases из тикета.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-013 && pnpm install`.
- Env: для demo-скрипта реальные переменные окружения и Meta-аккаунт не нужны — `fetch` мокается внутри скрипта.
- Сервисы: ничего поднимать не нужно.

## Проверки
1. Выполнить `pnpm lint` → ожидаемо команда завершается без ошибок ESLint.
2. Выполнить `pnpm typecheck` → ожидаемо TypeScript завершается без ошибок.
3. Выполнить `pnpm tsx scripts/manual/T-013-demo.ts` → ожидаемо печатается `T-013 demo passed`.
4. Проверить в выводе/коде demo-сценария, что длинный текст с кириллицей и emoji режется на 3+ частей, каждая часть не превышает 900 UTF-8 байт, а `parts.join('')` равен исходному тексту → покрывает критерий «Разбиение по байтам UTF-8 корректно для кириллицы и emoji».
5. Проверить, что demo-сценарий мокает `sendMessage()` и количество POST-запросов равно количеству частей; конкатенация отправленных body-текстов равна исходному тексту → подтверждает последовательную отправку всех частей.
6. Выполнить `rg "graph\.instagram\.com" src` → ожидаемо единственное совпадение находится в `src/lib/ig/client.ts`; другие модули не вызывают Graph API напрямую.

## Негативные проверки
1. Demo-скрипт мокает ответ `{ error: { type: 'OAuthException', code: 190 } }` → ожидаемо ошибка классифицируется как `IgAuthError`, а строка стека не содержит тестовый токен `secret-token`.
2. Demo-скрипт мокает HTTP 429 / code 613 → ожидаемо ошибка классифицируется как `IgRateLimitError`.
3. При ручном изменении `splitMessage(text, 900)` на слишком маленький лимит для длинного текста команда `pnpm tsx scripts/manual/T-013-demo.ts` должна упасть на проверке размера/склейки — это показывает, что demo реально проверяет инварианты разбиения.

## Как понять, что тикет НЕ работает
- `pnpm tsx scripts/manual/T-013-demo.ts` падает с ошибкой `split parts must reconstruct the source text` — разбиение потеряло или изменило часть исходного текста.
- Demo падает с ошибкой `each part must be <= 900 bytes` — лимит Meta по размеру части нарушен; смотреть `src/lib/ig/split.ts`.
- Demo падает на `OAuthException must become IgAuthError` или `429/code 613 must become IgRateLimitError` — сломана классификация ошибок в `src/lib/ig/client.ts`.
- `rg "graph\.instagram\.com" src` показывает совпадения вне `src/lib/ig/client.ts` — нарушена архитектурная граница тикета.
