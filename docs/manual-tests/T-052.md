# Ручной тест T-052 · Zernio: включение на чистом окружении и сквозной прогон

## Что сделано

Добавлены инструкция включения Zernio на новом окружении, эксплуатационные заметки о тарифах,
Inbox addon, лимите профилей и восстановлении вебхука. Описаны два способа подключения
Instagram и единый сквозной прогон: от настройки до ответа клиенту и безопасного выключения
интеграции.

## Подготовка

- ветка: `git fetch origin && git checkout zernio && pnpm install`.
- аккаунты: нужны сервисный аккаунт на zernio.com, Business-аккаунт Instagram для подключения и
  второй Instagram-аккаунт клиента для отправки DM. Для Mini App также нужен тестовый Telegram
  пользователь и бот проекта.
- env: заполните обязательные переменные приложения из `.env.example`. В Zernio Dashboard
  создайте API key и задайте его как `ZERNIO_API_KEY`. Выполните
  `pnpm tsx scripts/gen-key.ts`, скопируйте случайную base64-строку из первой строки вывода в
  `ZERNIO_WEBHOOK_SECRET`; `ZERNIO_API_BASE` оставьте `https://zernio.com/api`. Не вставляйте
  ключ или секрет в отчёт, скриншоты и историю терминала. Укажите публичный HTTPS-адрес
  развёртывания или туннеля в `APP_BASE_URL`.
- сервисы: используйте отдельный dev-проект Supabase, выполните `pnpm db:push`, затем запустите
  приложение (`pnpm dev`) и сделайте его доступным по `APP_BASE_URL`. После этого из того же
  настроенного окружения выполните `pnpm tsx scripts/manual/zernio-setup-webhook.ts`.

## Проверки

1. После setup-скрипта откройте Zernio Dashboard → Webhooks. → Есть ровно одна активная
   настройка `insta-reply` с URL `<APP_BASE_URL>/api/wh/zernio` и событиями
   `message.received`, `message.sent`, `account.connected`, `account.disconnected`. Повторный
   запуск скрипта не создаёт дубликат и не выводит секрет.
2. Откройте `/app` в Telegram как тестовый тенант. → Видна область «Подключить Instagram через
   Zernio» рядом с отдельной неизменённой областью прямого Meta-подключения. Нажмите кнопку,
   завершите OAuth для Business-аккаунта и вернитесь в Mini App. → Область показывает
   подключённый `@username` и статус «активно».
3. С клиентского Instagram-аккаунта отправьте короткое DM подключённому Business-аккаунту. → В
   Telegram появляется карточка с черновиком, а в Zernio Dashboard/`GET /v1/webhooks/logs`
   видна успешная доставка `message.received`.
4. В Telegram нажмите «Отправить» на этой карточке. → Карточка меняется на «✅ отправлено»,
   ответ появляется в той же переписке Instagram, а последующий `message.sent` не создаёт
   вторую карточку. В Supabase Studio у тенанта появляется исходящая запись в `message_log` и
   счётчик `drafts_sent` увеличивается на 1.
5. В Mini App нажмите «Отключить» и подтвердите действие. → Область предлагает
   переподключение, а аккаунт исчезает из профиля Zernio. Прямая Meta-область, если была
   подключена, не изменяет свой статус.
6. В Zernio Dashboard проверьте доступные ограничения плана. → Зафиксируйте в отчёте, потребовал
   ли Inbox API addon, какой лимит профилей показывает тариф и принимает ли сервис
   Creator-аккаунт. Если addon нужен, включите его и повторите шаги 3–4.

## Негативные проверки

1. На отдельном dev-окружении удалите **обе** переменные `ZERNIO_API_KEY` и
   `ZERNIO_WEBHOOK_SECRET`, перезапустите приложение и обновите `/app`. → Области Zernio нет,
   остальные функции и прямое Meta-подключение работают как до эпика.
2. В открытой Mini App с действующей сессией выполните `GET /api/miniapp/zernio/status`, а затем
   `POST /api/miniapp/zernio/connect` и `/api/miniapp/zernio/disconnect` через вкладку Network.
   → Каждый маршрут отвечает `404` с `error: "disabled"`.
3. Выполните запрос без подписи: `curl -i -X POST <APP_BASE_URL>/api/wh/zernio`. → Ответ
   `404`; проверка подписи не требуется, потому что интеграция выключена. Верните обе переменные,
   перезапустите приложение и повторно выполните setup-скрипт.

## Как понять, что тикет НЕ работает

- setup-скрипт создаёт дубликаты или webhook не становится активным — проверьте публичность
  `APP_BASE_URL`, журнал `GET /v1/webhooks/logs` и запустите скрипт повторно после исправления;
  после 10 последовательных ошибок Zernio выключает webhook.
- Входящее DM не создаёт карточку — проверьте журнал вебхуков Zernio, `message.received`, статус
  аккаунта в Mini App и `zernio_accounts`; при 403 Inbox включите соответствующий addon.
- Ответ не появляется в Instagram или появляется вторая карточка — проверьте `message.sent`,
  `processed_events`, `message_log` и 24-часовое окно Meta. Для ограничений платформы ожидается
  понятная ошибка карточки, а не повторная отправка.
- Секция остаётся видимой после удаления обеих переменных либо маршруты не дают 404 — убедитесь,
  что удалены именно обе переменные из работающего окружения и сервер действительно
  перезапущен.
