# Ручной тест T-027 · Дашборд

## Что сделано
- Добавлен `GET /api/miniapp/dashboard?days=7|30`, который собирает usage-метрики, статусы черновиков, краткий статус Instagram и последние 10 событий `message_log`.
- Главная страница `/app` заменена на дашборд с карточкой подключения, переключателем 7/30 дней, сеткой метрик, строкой LLM и блоком последней активности.
- Дашборд обновляется при видимой вкладке раз в 30 секунд и сразу после возвращения вкладки в видимое состояние.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-027 && pnpm install`.
- Env: заполнить стандартные переменные из `.env.example` для Mini App и Supabase (`TELEGRAM_BOT_TOKEN`, `MINIAPP_JWT_SECRET`, `SUPABASE_URL`, `SUPABASE_SECRET_KEY`, `APP_BASE_URL`, `ENCRYPTION_KEY`). Секреты Instagram нужны только если проверяется активное подключение.
- Сервисы: применить актуальные миграции к dev-проекту Supabase (`pnpm db:push`), затем запустить приложение `pnpm dev` и открыть Mini App через Telegram-бота.
- Данные: нужен tenant с завершённым онбордингом (`onboarding_step = 'done'`). Для проверки цифр можно вручную добавить в Supabase строки `usage_stats`, `drafts`, `message_log` за текущую дату и за дату старше 7 дней.

## Проверки
1. Откройте `/app` в Mini App. → Ожидаемо: видна карточка Instagram-статуса, переключатель «7 дней / 30 дней», четыре метрики «Диалогов / Черновиков / Отправлено / Вручную», строка `LLM: ...` и блок «Последняя активность».
2. В Supabase добавьте за текущий день `usage_stats`: `llm_calls=3`, `tokens_in=100`, `tokens_out=50`, `drafts_created=4`, `drafts_sent=2`. Добавьте за текущий период 4 черновика: `sent`, `pending`, `cancelled`, `skipped_manual`. Обновите `/app?days=7`. → Ожидаемо: «Черновиков» = 4, «Отправлено» = 2, «Вручную» = 2, `LLM: 3 вызовов · 150 токенов`; «Диалогов» соответствует числу черновиков периода.
3. Добавьте аналогичные строки старше 7 дней, но не старше 30 дней, затем переключите период с 7 на 30 дней. → Ожидаемо: метрики увеличиваются с учётом старых строк; при возврате на 7 дней старые строки не учитываются.
4. Добавьте в `message_log` 3 записи направлений `in`, `out`, `manual`, одну запись сделайте длиннее 80 символов. → Ожидаемо: в «Последней активности» видны иконки 📩 / ✅ / ✋, относительное время, а длинный текст обрезан многоточием.
5. Нажмите карточку статуса подключения. → Ожидаемо: открывается `/app/connect-instagram`.
6. Нажмите каждую карточку метрик. → Ожидаемо: открывается релевантный раздел Mini App (тест-чат, категории или экран подключения Instagram), а приложение не падает.
7. Очистите `message_log` для tenant и обновите `/app`. → Ожидаемо: вместо списка активности показан понятный пустой блок с CTA: при активном подключении — «Попробовать тест-чат», иначе — «Подключить Instagram».
8. Откройте DevTools → Network, оставьте вкладку видимой примерно на 35 секунд. → Ожидаемо: появляется повторный запрос `GET /api/miniapp/dashboard?days=...`. Сверните вкладку и верните её. → Ожидаемо: после `visibilitychange` дашборд обновляется ещё раз.

## Негативные проверки
1. В Network вручную вызовите `/api/miniapp/dashboard?days=14` с валидной session-cookie. → Ожидаемо: HTTP 400 и JSON с ошибкой `malformed`.
2. Удалите/испортите session-cookie и вызовите `/api/miniapp/dashboard?days=7`. → Ожидаемо: HTTP 401, данные чужого tenant не возвращаются.

## Как понять, что тикет НЕ работает
- Метрики не совпадают с Supabase: проверьте даты `usage_stats.day`, `drafts.created_at` и выбранный период 7/30 дней; API берёт период от сегодняшнего UTC-дня включительно.
- В UI всегда ошибка загрузки: смотрите Network для `/api/miniapp/dashboard` и серверные логи Next.js; типичные причины — невалидная session-cookie или Supabase env.
- Последняя активность не появляется: проверьте, что строки `message_log` относятся к тому же `tenant_id`; API отдаёт только 10 последних записей.
- Пустое состояние показывает неправильный CTA: проверьте строку `ig_connections.status`, наличие сохранённых токена/app secret и `handshake_at`.

## Покрытие критериев приёмки
- «Цифры дашборда сходятся c тестовыми данными» — шаги 2–3.
- «Каждая карточка ведёт на релевантный экран» — шаги 5–6.
- «Пустое состояние осмысленно для нового пользователя» — шаг 7.
