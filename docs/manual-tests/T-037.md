# Ручной тест T-037 · Заявки на подключение Instagram

## Что сделано

Пользователь может подать заявку на Instagram-аккаунт, которого ещё нет в Meta App. Каждый администратор получает кнопку подтверждения в Telegram. После подтверждения владелец заявки получает персональную ссылку на экран подключения Instagram в Mini App.

## Подготовка

- ветка: `git fetch && git checkout ticket/T-037 && pnpm install`
- env: заполните переменные из `.env.example`, необходимые для запуска приложения; для этого сценария особенно нужны `SUPABASE_URL`, `SUPABASE_SECRET_KEY`, `TELEGRAM_BOT_TOKEN` (токен из BotFather), `TELEGRAM_WEBHOOK_SECRET`, `APP_BASE_URL`, `TELEGRAM_BOT_USERNAME` (username бота без `@`) и `ADMIN_TELEGRAM_IDS` (CSV из Telegram user ID администраторов).
- сервисы: примените миграции к dev-проекту Supabase командой `pnpm db:push`; запустите `pnpm dev`. Для Telegram webhook поднимите публичный HTTPS-туннель на порт 3000 и настройте webhook бота на `<публичный URL>/api/telegram` с `TELEGRAM_WEBHOOK_SECRET`.
- в Meta App подготовьте тестовый Instagram username, которого ещё нет в реестре `ig_accounts`, и тестовый аккаунт администратора из `ADMIN_TELEGRAM_IDS`.

## Проверки (каждая: Действие → Ожидаемо)

1. Войдите в Mini App под тестовым пользователем и отправьте `POST /api/miniapp/onboarding/ig-account` с телом `{ "igUsername": "@FooBar" }` из вкладки Network или HTTP-клиента с сессионной cookie → ответ `200 { "status": "pending" }`; в `ig_accounts` появляется ровно одна строка `foobar` со статусом `pending` и tenant текущего пользователя.
2. Проверьте Telegram каждого ID из `ADMIN_TELEGRAM_IDS` → каждому приходит ровно одно сообщение о заявке с нормализованным username, названием бизнеса (если оно задано), Telegram ID заявителя и кнопкой «✅ Подтвердить».
3. Повторите тот же POST как тот же пользователь, используя `foobar` без `@` → снова `200 { "status": "pending" }`; новая строка и новые сообщения администраторам не появляются.
4. Нажмите «✅ Подтвердить» от имени администратора → карточка этого администратора меняется на «✅ Подтверждено: @foobar» без активной кнопки; строка становится `approved`; пользователь получает сообщение с кнопкой «Подключить Instagram».
5. Нажмите кнопку из сообщения пользователю → Mini App открывается сразу на `/app/connect-instagram`.
6. Повторно нажмите кнопку подтверждения в сообщении другого администратора → Telegram показывает «Уже подтверждено», клавиатура убирается; повторное пользовательское уведомление не приходит.
7. Отправьте `GET /api/miniapp/onboarding/ig-account` от имени заявителя → ответ `200` содержит `{ "status": "approved", "igUsername": "foobar" }`; для tenant без заявки ответ содержит `{ "status": "none" }`.

## Негативные проверки

1. Войдите вторым пользователем и отправьте username `@FooBar` → ответ `409 { "code": "taken" }`, заявка и Telegram-уведомления не создаются.
2. Отправьте `{ "igUsername": "@@bad name" }` → ответ `400 { "code": "invalid_username" }` без записи в БД и без сообщений.
3. Нажмите «✅ Подтвердить» с Telegram-аккаунта, которого нет в `ADMIN_TELEGRAM_IDS` → Telegram показывает «Недостаточно прав», а статус заявки остаётся `pending`.

## Как понять, что тикет НЕ работает

Если POST возвращает 500, проверьте вкладку Network, настройки сессионной cookie и доступ service role к Supabase. Если заявка есть, но Telegram-сообщения нет, проверьте webhook, `TELEGRAM_BOT_TOKEN`, `ADMIN_TELEGRAM_IDS` и серверные логи доставки. Если подтверждение не меняет статус или не приходит сообщение владельцу, проверьте запись `ig_accounts`, поля tenant `tg_chat_id` и `TELEGRAM_BOT_USERNAME`; токены и другие секреты в логах отсутствовать не должны.
