# Ручной тест T-034 · Вывод own_app-подключения Instagram из эксплуатации

## Что сделано
- Ручной Instagram-флоу с access token/App Secret удалён: подключение доступно только через Instagram Business Login (OAuth).
- Per-tenant webhook `/api/wh/ig/<tenantId>` удалён; рабочим остаётся только глобальный webhook `/api/wh/ig`.
- Миграция удаляет старые колонки `ig_connections.connection_mode`, `app_secret_enc`, `verify_token`, `handshake_at` и `tenants.history_thread_id`.

## Подготовка
- Ветка: `git fetch && git checkout ticket/T-034 && pnpm install`.
- Важно ДО миграции: `pnpm db:push` удалит все строки `ig_connections` с ручным режимом `own_app`. Проверьте в dev-базе, что владельцы этих подключений готовы переподключиться через OAuth.
- Env в `.env.local`:
  - `INSTAGRAM_APP_ID` — ID продукта Instagram из Meta Dashboard.
  - `INSTAGRAM_APP_SECRET` — App Secret платформенного Meta/Instagram App.
  - `IG_WEBHOOK_VERIFY_TOKEN` — verify token глобального webhook `/api/wh/ig`.
  - `TELEGRAM_BOT_USERNAME` — username бота без `@` для возврата после OAuth (желательно заполнить).
  - Остальные обязательные переменные проекта — как в `.env.example`.
- Сервисы:
  - Dev-проект Supabase, затем `pnpm db:push`.
  - Публичный HTTPS URL приложения (`APP_BASE_URL`): Vercel preview или туннель к `pnpm dev`.
  - В Meta Dashboard настройте Instagram Webhooks: callback `{APP_BASE_URL}/api/wh/ig`, verify token из `IG_WEBHOOK_VERIFY_TOKEN`, поле `messages`.

## Проверки
1. Запустите `pnpm db:push` → миграция применяется без ошибок. В Supabase Table Editor / SQL Editor проверьте: в `ig_connections` нет колонок `connection_mode`, `app_secret_enc`, `verify_token`, `handshake_at`; в `tenants` нет `history_thread_id`; индекс `ig_connections_platform_ig_account_id_key` уникален по `ig_account_id` с условием `ig_account_id is not null`.
2. Уберите из `.env.local` любую одну переменную из `INSTAGRAM_APP_ID`, `INSTAGRAM_APP_SECRET`, `IG_WEBHOOK_VERIFY_TOKEN` и запустите `pnpm dev` → приложение не стартует при первом обращении к env и показывает понятную ошибку валидации. Верните переменную обратно.
3. Откройте Mini App `/app/connect-instagram` → видна только карточка Business Login с кнопкой «Подключить через Instagram» / «Переподключить»; аккордеона «Продвинутый способ (вручную)», callback URL, verify token и полей Access Token/App Secret нет.
4. Нажмите «Подключить через Instagram», пройдите consent Instagram и вернитесь в Mini App → страница показывает подключённый `@username`, а повторное нажатие «Переподключить» обновляет то же подключение без создания дублей.
5. Отправьте GET на старый URL `curl -i "${APP_BASE_URL}/api/wh/ig/<tenantId>?hub.mode=subscribe&hub.verify_token=x&hub.challenge=1"` → ответ 404.
6. В Meta Dashboard выполните Verify and Save для глобального webhook `/api/wh/ig` → проверка проходит с `IG_WEBHOOK_VERIFY_TOKEN`.
7. Отправьте DM подключённому Instagram-аккаунту с другого профиля → глобальный webhook принимает событие, маппит его по `ig_account_id`, в Telegram приходит карточка черновика.
8. Откройте диагностику подключения → есть только проверки `token` и `event`; проверки `handshake` нет ни в API-ответе `/api/miniapp/ig/status`, ни в UI.
9. Откройте `/app` и `/app/settings` → дашборд показывает статус Instagram, а «Отключить Instagram» переводит подключение в pending без ошибок. После этого повторное подключение через OAuth работает.

## Негативные проверки
1. Отправьте POST на `/api/miniapp/ig/connect` с любым JSON → ожидается 405/404 от Next.js, ручной API сохранения токена недоступен.
2. Отправьте POST на `/api/wh/ig` с неверным `X-Hub-Signature-256` → ответ 401, событие не обрабатывается, секреты не появляются в логах.
3. Откройте `/api/miniapp/ig/login-url` с валидной Mini App-сессией → всегда возвращается OAuth URL; ответа `{ error: "disabled" }` больше нет.

## Как понять, что тикет НЕ работает
- На странице подключения есть ручной аккордеон, поля Access Token/App Secret или callback URL — UI старого флоу не удалён.
- `/api/miniapp/ig/connect` принимает POST и возвращает `ok: true` — ручной API всё ещё жив.
- Старый `/api/wh/ig/<tenantId>` отвечает не 404 — per-tenant webhook не удалён.
- В `/api/miniapp/ig/status` или UI диагностики есть `handshake` — диагностика всё ещё зависит от старого webhook handshake.
- После `pnpm db:push` в схеме остались удаляемые колонки или нет unique-индекса по `ig_account_id` — миграция не соответствует тикету.
