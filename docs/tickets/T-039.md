# T-039 · Демонтаж LLM-генерации онбординга

> **Перед началом:** прочитай `docs/plan.md` §5 (ревизия v1.2), `docs/tickets/OVERRIDES.md`
> и `docs/tickets/T-038.md` (что пришло на замену). Зависимости: T-038 (влит).
> Онбординг 2.0 работает на предустановленных шаблонах — генерация базы знаний через LLM
> из свободного описания бизнеса удаляется целиком: роут, промпт, поле БД, остатки UI и i18n.

## Цель

Удалить мёртвый код и данные старого онбординга (T-009/T-010): эндпоинт генерации, промпт,
`tenants.org_description`, осиротевшие компоненты и ключи локализации.

## Инвентаризация

- Роут `app/api/miniapp/onboarding/generate/route.ts` (T-010) — удалить.
- `src/lib/llm/prompts.ts`: удалить `kbGenerationPrompt` (и его Zod-схему ответа
  `{knowledge_base, system_prompt}`, если объявлена рядом). `defaultSystemPrompt` (T-038),
  `classifyPrompt`, `draftPrompt` — остаются.
- Компоненты/шаги старого визарда (welcome-тезисы, org_form c textarea свободного описания,
  generating-скелетон, review_kb) — всё, что не переиспользовано в T-038.
- i18n: осиротевшие ключи старого визарда (ru и de симметрично); критерий — grep по ключу
  не находит использований.
- БД: `tenants.org_description` больше никем не читается и не пишется.

## Шаги выполнения

1. Удалить роут generate; проверить, что фронт больше никуда на него не ходит
   (`grep -rn "onboarding/generate" src app`).
2. Почистить `prompts.ts`; убедиться, что simulator/classify/draft-пайплайн не задет.
3. Удалить осиротевшие компоненты старого визарда и i18n-ключи.
4. Миграция `supabase/migrations/0012_drop_org_description.sql`:
   `alter table tenants drop column org_description;` + правка `src/lib/db/types.gen.ts`
   и репозитория `tenants` (типы patch). Применение — шаг человека в manual-test
   (`pnpm db:push`, предупреждение: свободные описания бизнеса безвозвратно удаляются —
   существующие базы знаний НЕ затрагиваются).
5. Проверить смежные потребители: `/app/settings` (редактирование КБ/промпта), симулятор
   (ошибка 409 `onboarding_required`), dashboard — работают без изменений поведения.
6. Финальный grep: `grep -rn "org_description\|kbGenerationPrompt\|onboarding/generate" src app scripts`
   — вхождений нет.
7. Обновить `README.md` и другие затронутые доки, если в них остались упоминания
   LLM-генерации онбординга.
8. Manual-test `docs/manual-tests/T-039.md`: онбординг нового тенанта проходит целиком
   (никаких обращений к generate в network-логе), настройки/симулятор работают, миграция
   применена, колонка отсутствует.
