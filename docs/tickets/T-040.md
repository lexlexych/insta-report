# T-040 · BUG: язык мини-аппа не наследуется из Telegram (всегда немецкий)

> **Тип:** bugfix. **Приоритет:** высокий (первое впечатление нового пользователя).
> **Затронутый код:** эпик E10 (T-037 deep-link `connect`, T-038 синхронизация `ui_locale`).
> **Перед началом:** прочитай `docs/orchestration/workflow.md`, `docs/tickets/OVERRIDES.md`,
> этот тикет и файлы из раздела «Затронутые файлы».

## Симптом

Новый пользователь с русским интерфейсом Telegram открывает мини-апп первый раз — и весь
интерфейс мини-аппа на **немецком**. Ожидание: до завершения онбординга язык берётся из языка
Telegram-аккаунта пользователя (см. комментарий в `TmaProvider.tsx:154`), т.е. для русского
Telegram — русский интерфейс.

По факту немецкий получают **все** новые пользователи независимо от языка Telegram: язык из
Telegram не читается вообще, а `de` — это жёсткий фолбэк.

## Корень проблемы

Язык вычисляется из launch-параметров Telegram SDK в
[`src/components/miniapp/TmaProvider.tsx:76`](src/components/miniapp/TmaProvider.tsx#L76):

```ts
resolveLocale(
  launchParams.initDataUnsafe?.user?.languageCode ??
    launchParams.initDataUnsafe?.user?.language_code,
)
```

Проблема: **у `@telegram-apps/sdk-react` v3 (`useLaunchParams()`) нет поля `initDataUnsafe`.**
`initDataUnsafe` — это форма legacy-API `window.Telegram.WebApp`, а не этого SDK. В v3 разобранные
init-данные лежат в `tgWebAppData` (тип `LaunchParams` из `@telegram-apps/types`):

```
LaunchParams { tgWebAppData?: InitData; tgWebAppStartParam?: string; ... }
InitData      { user?: { id; language_code?; ... }; ... }
```

(проверено в `node_modules/@telegram-apps/types/.../launch-params.d.ts` и `init-data.d.ts`).

Почему это не поймал компилятор: в репозитории есть **самодельная заглушка типов**
[`src/types/telegram-apps-sdk-react.d.ts`](src/types/telegram-apps-sdk-react.d.ts#L38), которая
через `declare module '@telegram-apps/sdk-react'` полностью подменяет типы модуля и объявляет
несуществующие поля `initDataUnsafe` и `startParam`. TypeScript валидирует обращение к ним,
`pnpm typecheck` зелёный — но в рантайме `launchParams.initDataUnsafe` всегда `undefined`,
поэтому `resolveLocale(undefined)` → `de` (см. `resolveLocale` в
[`src/lib/i18n/shared.ts:393`](src/lib/i18n/shared.ts#L393): всё, что не начинается с `ru`, → `de`).

### Сопутствующий баг из того же корня (обязательно исправить заодно)

Тем же битым стабом сломан deep-link онбординга из T-037: в
[`TmaProvider.tsx:185`](src/components/miniapp/TmaProvider.tsx#L185) читается
`launchParams.startParam`, тогда как реальное поле — `tgWebAppStartParam`. Значит `startParam`
всегда `undefined`, и переход по кнопке «Подключить Instagram» из уведомления админа
(`startapp=connect`) на `/app/connect-instagram` **не срабатывает**. Это тот же класс ошибки
(вымышленная форма launch-параметров), чинится в одном месте.

## Решение

Двухуровневое: (1) сервер-авторитетный язык как надёжный источник до онбординга,
(2) починка клиентского стаба типов, чтобы устранить сам корень и заодно `tgWebAppStartParam`.

### 1. Сервер-авторитетный язык (основное, устойчиво к форме SDK)

Init-данные и так валидируются на сервере, где `user.language_code` гарантированно доступен
([`app/api/miniapp/auth/route.ts:17`](app/api/miniapp/auth/route.ts#L17),
[`src/lib/tg/initData.ts:12`](src/lib/tg/initData.ts#L12)). Отдаём вычисленный язык клиенту:

- В `route.ts` импортировать `resolveLocale` из `@/lib/i18n` и добавить в JSON-ответ поле
  `tgLocale: resolveLocale(user.language_code)` (рядом с `tenant`, не внутри — это язык из
  Telegram, а не сохранённый выбор тенанта).
- В `TmaProvider.tsx` (`TenantProvider`) применять язык так:
  - онбординг **завершён** (`onboardingStep === 'done'`) и есть `tenant.uiLocale` →
    `setLocale(uiLocale)` (текущее поведение сохраняется);
  - иначе (новичок) → `setLocale(payload.tgLocale)`.
  Это точно реализует задекларированную логику «до онбординга язык из Telegram» и не зависит
  от формы клиентского SDK.

### 2. Починка клиентского чтения launch-параметров (корень + регресс startParam)

- В [`src/types/telegram-apps-sdk-react.d.ts`](src/types/telegram-apps-sdk-react.d.ts#L38)
  привести тип `LaunchParams` в соответствие реальному SDK:
  - заменить `initDataUnsafe?: { user?: {...} }` на `tgWebAppData?: { user?: { language_code?: string; languageCode?: string } }`;
  - в camelCase-перегрузке `useLaunchParams(true)` заменить `startParam?: string`
    на `tgWebAppStartParam?: string`.
- В `LaunchParamsLocale` (`TmaProvider.tsx:76`) читать язык из
  `launchParams.tgWebAppData?.user?.language_code ?? launchParams.tgWebAppData?.user?.languageCode`
  (клиентский фолбэк на случай, если ответ auth ещё не пришёл; итоговый язык всё равно
  подтвердит сервер из п.1).
- В `TmaProvider.tsx:185`/`:197` читать `launchParams.tgWebAppStartParam` вместо
  `launchParams.startParam` (чинит deep-link `connect`); поправить массив зависимостей `useEffect`.

> Примечание: заглушку типов не удалять целиком — она также описывает `init`, `miniApp`,
> `viewport`, `themeParams` и т.д., используемые в `TmaProvider`. Правим только форму
> launch-параметров.

## Затронутые файлы

- `src/types/telegram-apps-sdk-react.d.ts` — форма `LaunchParams` (`tgWebAppData`, `tgWebAppStartParam`).
- `src/components/miniapp/TmaProvider.tsx` — чтение языка и `startParam`, применение `tgLocale`.
- `app/api/miniapp/auth/route.ts` — вернуть `tgLocale` из валидированных init-данных.
- (без изменений логики) `src/lib/i18n/shared.ts` — `resolveLocale` переиспользуется как есть.

## Критерии приёмки

- [ ] Новый пользователь с русским Telegram при первом входе видит **русский** интерфейс мини-аппа
- [ ] Новый пользователь с не-русским Telegram видит немецкий (фолбэк не изменился)
- [ ] После онбординга язык берётся из `tenant.ui_locale` (регресса нет)
- [ ] Deep-link `startapp=connect` снова уводит на `/app/connect-instagram` (регресс T-037 закрыт)
- [ ] В коде мини-аппа нет обращений к несуществующим полям SDK
      (`grep -rn "initDataUnsafe\|\.startParam" src app` — пусто)
- [ ] `pnpm lint && pnpm typecheck` зелёные
- [ ] Manual-test `docs/manual-tests/T-040.md`: вход из русского и не-русского Telegram-аккаунта,
      проверка языка на онбординге и после него, проверка deep-link `connect`
