**Ревизия v1.4 (Эпик 12, «Instagram через Zernio»):** параллельный (второй) способ подключения
Instagram — через агрегатор Zernio (https://zernio.com): OAuth идёт через прошедший App Review
Meta App самого Zernio, поэтому пользователю не нужны ни одобрение админом, ни приглашение
тестировщика — один клик и экран авторизации. Прямая Meta-интеграция НЕ удаляется и не меняется;
провайдер фиксируется на уровне черновика (`drafts.provider`), двойная доставка гасится дедупом
по mid. Обязательное чтение для тикетов эпика — `docs/research/zernio.md`. **Ветка эпика — `zernio`**
(санкционированное человеком отступление от имени `epic/E12`). Другие каналы Zernio (WhatsApp и
т.д.) — вне объёма, архитектура заложена (`platform`-колонка, общий вебхук).

**Ревизия v1.3 (Эпик 11):** онбординг упрощён — IG-шаги (экраны 3а/3б из T-038) вынесены из
визарда; подключение Instagram запускается с финального экрана онбординга или дашборда и идёт
через отдельный пошаговый визард (приглашение тестировщика Meta App → OAuth → результат).
Подключение возможно только после подтверждения ника администратором (`/ig_add` или кнопка
«Подтвердить» из T-037).

**Ревизия v1.2 (Эпик 10):** онбординг переработан — вместо LLM-генерации базы знаний (T-009/T-010) используются предустановленные шаблоны сфер деятельности (`src/lib/kb-templates/`); T-009/T-010 упразднены и оставлены как историческая справка.

**Критический путь MVP:** T-001 → T-002/T-003 → T-004 → T-005 → T-011 → T-013 → T-016 → T-017 → T-018 → T-019 → T-020 → T-021 → T-022.
Параллельные дорожки после T-007: онбординг (T-008…T-010), UI (T-006, T-009, T-012), симулятор (T-025, T-026).

**Правило для оркестрации:** после каждого тикета — код-ревью diff'а вторым агентом по чек-листу: критерии приёмки тикета + соглашения plan.md §5 + отсутствие изменений вне заявленных файлов.
