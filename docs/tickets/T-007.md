# T-007 · Аутентификация Mini App (initData → JWT)

> **Перед началом:** прочитай `docs/plan.md` §2, §5. Зависимости: T-005, T-006.

## Цель
Проверка подлинности `initData` Telegram, сессия в httpOnly cookie, хелпер `requireTenant` для API.

## Шаги выполнения
1. `src/lib/tg/initData.ts` — `validateInitData(raw: string, botToken: string, maxAgeSec = 3600)`:
   - распарсь query-string; извлеки и удали `hash`;
   - data_check_string = отсортированные по ключу пары `key=value`, соединённые `\n`;
   - secretKey = HMAC_SHA256(key: строка `WebAppData`, message: botToken);
   - ожидаемый hash = hex(HMAC_SHA256(key: secretKey, message: data_check_string)); сравнение `timingSafeEqual`;
   - проверь `auth_date` не старше maxAgeSec;
   - верни `{ user: { id, first_name, language_code, ... } }` (поле `user` — JSON внутри initData) или бросай `InitDataError('bad_hash' | 'expired' | 'malformed')`.
2. Установи `jose`. `src/lib/auth/session.ts`: `signSession({tenantId}): Promise<string>` (HS256, `env.MINIAPP_JWT_SECRET`, exp 12h), `verifySession(token)`.
3. `app/api/miniapp/auth/route.ts` POST: body `{ initData: string }` (Zod) → validateInitData → `tenants.upsertByTelegramUserId(user.id, { tg_chat_id: user.id })` → signSession → `Set-Cookie: session=...; HttpOnly; Secure; SameSite=None; Path=/; Max-Age=43200` (SameSite=None обязателен: WebView Telegram — третья сторона) → `{ ok: true, tenant: { id, onboardingStep, orgName } }`. Ошибки initData → 401 c кодом причины.
4. `src/lib/auth/requireTenant.ts`: `requireTenant(req): Promise<Tenant>` — читает cookie, verifySession, `tenants.getById`; отсутствие/невалидность → бросает `HttpError(401)`. Добавь общий обработчик: хелпер `apiHandler(fn)` оборачивающий роуты — ловит HttpError → Response с кодом, прочее → 500 + лог.
5. Фронт: в TmaProvider после init() выполняй POST /api/miniapp/auth c `retrieveRawInitData()`; состояние `{ status: 'loading'|'ready'|'error', tenant }` в context `useTenant()`. При `error` — экран с кнопкой «Повторить».
6. Тесты `tests/initData.test.ts`: сгенерируй фикстуру — сам подпиши initData тестовым botToken по алгоритму из п.1 и проверь: валидная проходит; изменение одного символа user → bad_hash; auth_date = now-7200 → expired; отсутствие hash → malformed. Тест `requireTenant`: без cookie → 401.

## Критерии приёмки
- [ ] Подпись проверяется по алгоритму Telegram, сравнение timing-safe
- [ ] Все роуты `/api/miniapp/*` (кроме /auth) используют requireTenant через apiHandler
- [ ] Фронт автоматически аутентифицируется при старте
