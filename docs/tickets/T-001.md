# T-001 · Скаффолд проекта

> **Перед началом:** прочитай `docs/plan.md`, разделы 2 (архитектура) и 5 (соглашения). Ключевые правила: доступ к env только через `src/lib/env.ts`; Supabase только через `src/lib/db`; секреты — только через `src/lib/crypto.ts`. Завершение тикета = `pnpm lint && pnpm typecheck && pnpm test` зелёные.

## Цель
Создать каркас репозитория: Next.js 15 + TypeScript strict + Tailwind, структура каталогов, линтеры, Vitest, типобезопасный env-модуль, CI.

## Шаги выполнения
1. Инициализируй проект: `pnpm create next-app@latest . --ts --tailwind --eslint --app --src-dir=false --import-alias "@/*"`. App Router обязателен, каталог `/app` в корне.
2. Включи strict-режимы в `tsconfig.json`: `"strict": true`, `"noUncheckedIndexedAccess": true`.
3. Создай структуру каталогов (пустые `.gitkeep` там, где нет файлов):
   - `/src/lib/db`, `/src/lib/ig`, `/src/lib/tg`, `/src/lib/llm`, `/src/lib/pipeline`
   - `/supabase/migrations`, `/scripts`, `/tests/fixtures`, `/docs`
4. Скопируй файл плана проекта в `docs/plan.md` (он передан вместе с тикетом).
5. Создай `src/lib/env.ts`:
   - Zod-схема `envSchema` со ВСЕМИ переменными из docs/plan.md §2.5. Все — `z.string().min(1)`, кроме: `ADMIN_TELEGRAM_IDS` — `z.string().default('')`; `ENCRYPTION_KEY` — строка с refine: base64-декод даёт ровно 32 байта.
   - Экспорт `export const env = envSchema.parse(process.env)` c ленивой инициализацией (getter или memo-функция `getEnv()`), чтобы импорт модуля в тестах не падал без переменных. Экспортируй и `envSchema` для тестов.
6. Создай `.env.example` — все переменные с комментарием-описанием и примером значения (для секретов — `changeme`).
7. Настрой Prettier (`.prettierrc`: singleQuote, semi, printWidth 100) и добавь prettier-плагин в ESLint конфиг. Скрипты в package.json: `lint`, `typecheck` (`tsc --noEmit`), `test` (`vitest run`), `format`.
8. Настрой Vitest: `vitest.config.ts` c alias `@/* -> ./src/*` и `@app/* -> ./app/*`, environment `node`.
9. Напиши первый тест `tests/env.test.ts`: (a) валидный набор env проходит схему; (b) отсутствие `TELEGRAM_BOT_TOKEN` — ошибка; (c) `ENCRYPTION_KEY` неверной длины — ошибка.
10. Создай страницу-заглушку `app/(miniapp)/app/page.tsx` («InstaReply — загрузка…»).
11. Создай `.github/workflows/ci.yml`: триггер push/PR в main; jobs: checkout → pnpm install (с кэшем) → `pnpm lint` → `pnpm typecheck` → `pnpm test`.
12. Заполни `README.md`: назначение проекта (2 абзаца), требования (Node 20+, pnpm), быстрый старт, описание структуры каталогов, ссылки на docs/plan.md.

## Критерии приёмки
- [ ] `pnpm dev` поднимает `/app` без ошибок в консоли
- [ ] `pnpm lint && pnpm typecheck && pnpm test` зелёные локально и в CI
- [ ] В `.env.example` есть все переменные из docs/plan.md §2.5
- [ ] Импорт `env` из произвольного модуля типобезопасен (автодополнение полей)
