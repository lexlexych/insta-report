# T-019 · Генерация черновика ответа

> **Перед началом:** прочитай `docs/plan.md` §2.2, §5. Зависимости: T-008, T-018.

## Цель
`src/lib/pipeline/draft.ts`: текст черновика на основе базы знаний, инструкции метки и контекста.

## Шаги выполнения
1. Заполни `draftPrompt(...)` в prompts.ts:
   - system = `tenant.system_prompt` + блок «=== База знаний ===\n{knowledge_base}» + правила: «Отвечай на языке, на котором пишет клиент. Не выдумывай факты, цены и обещания — если данных нет в базе знаний, вежливо скажи, что уточнишь. Пиши как живой сотрудник, без подписи и без markdown. Верни ТОЛЬКО текст ответа.»;
   - user: «История диалога:\n{history}\n\nНеотвеченные сообщения клиента:\n{pendingText}\n\nИнструкция для этой категории обращений: {label.instruction}».
2. `generateDraft(tenant, label, ctx, ev): Promise<{ draftText: string; usage }>`:
   - случай «только вложение» (`ev.hasAttachments && !ctx.pendingText.trim()` или pendingText === fallback-пустышка): не зови LLM — верни шаблонный текст-уточнение из i18n-подобного набора (по reply_language тенанта; 'auto' → русский+ немецкий вариант выбери по языку истории простым эвристическим правилом: наличие кириллицы) вида «Здравствуйте! Вижу, вы отправили вложение — подскажите, пожалуйста, что вас интересует?»;
   - обычный случай: `complete(MODEL_DRAFT, temperature 0.7, maxTokens 500, tenantId)`;
   - постобработка: trim; срежь обрамляющие кавычки, если LLM обернул ответ; пустой результат → LlmError('empty_draft').
3. Собери оркестратор `src/lib/pipeline/handleIncoming.ts` (частично, доставка — T-020):
   ```
   conn = igConnections.getForTenant; status !== 'active' → выход c логом
   ctx = buildContext(conn, ev)
   labels = labels.listByTenant
   { label } = classify(...)
   { draftText } = generateDraft(...)
   await deliverDraft({ tenant, conn, ev, ctx, label, draftText })   // T-020, пока заглушка
   messageLog.add(tenantId, key, 'in', ctx.pendingText)
   usageStats.increment(tenantId, { drafts_created: 1 })
   ```
   `conversationKey = `${ev.accountId}:${ev.contactId}``  — вынеси в `src/lib/pipeline/key.ts`.
4. Тесты `tests/pipeline.draft.test.ts` (мок llm): промпт содержит kb, инструкцию метки, history и pendingText (проверь аргументы мока); вложение без текста → LLM не вызван, шаблон вернулся; пустой ответ LLM → LlmError; handleIncoming: неактивное подключение → ни одного вызова дальше.

## Критерии приёмки
- [ ] Все элементы контекста попадают в промпт (тест по аргументам)
- [ ] Кейс «вложение без текста» не тратит LLM-вызов
- [ ] usage/статистика инкрементятся
