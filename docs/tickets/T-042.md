# T-042 · Финальный экран «Готово! Что дальше?» — меню разделов и кнопка подключения

> **Перед началом:** прочитай `docs/plan.md` §1, §5 и `docs/tickets/OVERRIDES.md`.
> Зависимости: T-041 (маршрут онбординга 2.1), T-043 (connect-статус).
> Тикет перерабатывает экран `finish` визарда онбординга: вместо кнопки «Начать работу» —
> реальная нижняя навигация с описанием её пунктов и кнопка подключения Instagram (если ник
> подтверждён администратором).

## Цель

Финальный экран онбординга показывает стандартную нижнюю навигацию (4 пункта), описывает
назначение каждого пункта (иконка + описание) и предлагает подключить Instagram по готовности.
Кнопка «Начать работу» удаляется.

## Реализация

1. Шаг `finish` сохраняется: `POST /api/miniapp/onboarding/complete` по-прежнему ставит
   `onboarding_step='finish'`; выход и повторный вход в мини-апп возвращают на этот экран,
   пока пользователь не ушёл в один из разделов.
2. `BottomNav`: показывать также при `onboarding_step='finish'` (сейчас — только при `'done'`).
3. Контент экрана: заголовок «Готово! Что дальше?»; краткая вводная строка про входящие из
   Instagram (когда клиент напишет в Direct, в Telegram придёт карточка с черновиком — отправка
   одним нажатием; перенести из шпаргалки T-038); затем список из 4 пунктов, СООТВЕТСТВУЮЩИХ
   нижней навигации — те же иконки, что в `BottomNav`, + 1–2 строки описания:
   - **Дашборд** — статистика диалогов и статус подключения Instagram;
   - **Тест-чат** — проверьте базу знаний: напишите боту как клиент и посмотрите черновики
     ответов;
   - **Категории** — категории обращений и правила ответа для каждой;
   - **Настройки** — база знаний и стиль ответов редактируются в любой момент.
4. Кнопку «Начать работу» удалить. Выход из онбординга — тапом по пункту нижней навигации:
   клик по пункту `BottomNav` при шаге `finish` выполняет `PATCH tenant
   { onboardingStep: 'done' }` (fire-and-forget) и переходит в раздел; после этого онбординг
   закрыт навсегда (существующие гарды `done` не трогать).
5. Внизу контента, НАД нижней навигацией — блок по `connect`-статусу
   (`GET /api/miniapp/ig/status`, T-043):
   - `ready` → кнопка «Подключить Instagram»: `PATCH { onboardingStep: 'done' }` + переход на
     `/app/connect-instagram`;
   - `awaiting_admin` (и `none`) → статус-строка «Ожидание подтверждения администратором»
     (иконка-часы), кнопки нет;
   - `active` → бейдж «Instagram подключён», кнопки нет.
6. i18n ru/de всех строк экрана.
7. Manual-test `docs/manual-tests/T-042.md`: после «Завершить» открывается новый финальный
   экран с видимой навигацией; перезапуск мини-аппа возвращает на finish; клик по каждому из
   4 пунктов закрывает онбординг (шаг `done`) и открывает раздел; кнопка «Подключить Instagram»
   видна только при `ready` и ведёт на страницу подключения; при `pending`-заявке — статус
   ожидания; при уже активном подключении — бейдж.

## Критерии приёмки

- [ ] На `finish` видна реальная нижняя навигация; на странице — описания её 4 пунктов с теми
      же иконками
- [ ] Кнопка «Начать работу» удалена; выход из онбординга — через пункты меню, шаг становится
      `done`
- [ ] Блок над навигацией: кнопка «Подключить Instagram» только при `ready`; при
      `awaiting_admin`/`none` — «Ожидание подтверждения администратором»; при `active` — бейдж
- [ ] Повторный вход до выбора раздела возвращает на `finish`
- [ ] `pnpm lint && pnpm typecheck` зелёные; manual-test написан
