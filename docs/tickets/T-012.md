# T-012 · Визард подключения Instagram (UI)

> **Перед началом:** прочитай `docs/plan.md` §1 (переходная схема own_app), §5. Зависимости: T-007, T-011.

## Цель
Страница `/app/connect-instagram`: инструкция по созданию Meta App + форма секретов + отображение webhook-реквизитов.

## Шаги выполнения
1. Бэкенд: `app/api/miniapp/ig/connect/route.ts`:
   - GET (requireTenant): вернуть текущее состояние `{ status, igUsername, webhookUrl, verifyToken, hasToken, hasSecret, webhookLastSeenAt, handshakeAt }`. Если записи нет — создать: `verify_token = randomBytes(16).toString('hex')`, status 'pending'. `webhookUrl = env.APP_BASE_URL + '/api/wh/ig/' + tenantId`. Секреты НЕ возвращать никогда — только булевы hasToken/hasSecret.
   - POST: Zod `{ accessToken?: string, appSecret?: string }` (min length 10) → `igConnections.upsertForTenant` (шифрование внутри репозитория) → 200.
2. UI-структура страницы — вертикальные шаги-аккордеоны (компонент `GuideStep` c номером, заголовком, состоянием open/done):
   1. «Создайте приложение Meta» — developers.facebook.com → Create App → тип Business; место под скриншот `/public/guide/step1.png` (создай каталог c README «положить скриншоты», рендерь через `next/image` c fallback-плейсхолдером при отсутствии).
   2. «Добавьте продукт Instagram» → «API setup with Instagram business login».
   3. «Привяжите ваш профессиональный аккаунт Instagram» (аккаунт должен быть Business/Creator).
   4. «Сгенерируйте токен доступа» — в разделе API setup, кнопка Generate token; предупреждение: токен показывается один раз, скопируйте.
   5. «Настройте Webhooks» — поля Callback URL и Verify Token; рядом наши значения c кнопками копирования (`CopyField`: моноширинный текст + кнопка; используй `navigator.clipboard`, фидбек «Скопировано ✓»); подписка на поле **messages**.
   6. «Введите данные ниже» — форма: App Secret (App settings → Basic) и Access Token; `type=password` c глазком; submit → POST; поля после сохранения показываются как `••••` c возможностью «Заменить».
3. После сохранения — авто-переход к блоку диагностики (T-014; до его готовности — плейсхолдер «Проверка появится здесь»).
4. Ссылку «Подключить Instagram» добавь на дашборд-заглушку и в шаг done онбординга (проверь, что маршрут совпадает).
5. Тесты: компонентный — CopyField копирует значение (мок clipboard); форма не отправляется c пустыми полями; API-тест — GET создаёт connection c verify_token при первом обращении; POST не логирует секреты (спай на console).

## Критерии приёмки
- [ ] Секреты никогда не возвращаются c бэкенда и не пишутся в логи
- [ ] webhookUrl/verifyToken корректны и копируются в один тап
- [ ] Повторный вход показывает статус и маскированные поля
