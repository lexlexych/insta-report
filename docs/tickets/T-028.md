# T-028 · Настройки + GDPR-действия

> **Перед началом:** прочитай `docs/plan.md` §1, §4 (рынок ЕС), §5. Зависимости: T-010.

## Цель
Страница настроек: правка базы знаний/промпта, язык, отключение Instagram, полное удаление данных.

## Шаги выполнения
1. UI `/app/settings`, секции:
   - **База знаний**: превью (первые 6 строк + «Развернуть») → полноэкранный редактор (textarea, счётчик/20000, кнопки Сохранить/Отмена) → PUT /knowledge; отдельно «Системный промпт» (collapsed, пометка «для продвинутых»);
   - **Язык интерфейса**: ru/de → сохранение в tenants (`ui_locale`, миграция 0007) + context i18n;
   - **Instagram**: статус + кнопка «Отключить» (confirm) → `POST /api/miniapp/ig/disconnect`: обнулить token/secret (записать null), status='pending', handshake_at=null; карточка возвращается в состояние «не подключено»;
   - **Правовая информация**: ссылки на /legal/* (T-032);
   - **Опасная зона**: «Удалить аккаунт и все данные».
2. Удаление: `DELETE /api/miniapp/tenant` (requireTenant): двойное подтверждение на фронте (confirm + ввод слова «УДАЛИТЬ»/«LÖSCHEN» по локали); бэкенд: отправить прощальное сообщение в TG («Все данные удалены. /start — начать заново») ДО удаления; `tenants.deleteCascade(id)`; очистить cookie сессии; фронт → экран «Данные удалены».
3. Проверь каскады: миграции T-002 должны сносить ig_connections, labels, drafts, message_log, usage_stats, processed_events, rate_limits (если есть FK; для таблиц c tenant_id без FK — удали явным delete в репозитории deleteCascade). Напиши это в deleteCascade явно (последовательные delete + tenants последним) — надёжнее, чем полагаться только на FK.
4. Тесты: deleteCascade вызывает delete для КАЖДОЙ таблицы c tenant_id (шерсти список таблиц из types.gen.ts, чтобы тест падал при добавлении новой таблицы без обновления deleteCascade); disconnect затирает секреты (в мок уходит null); прощальное сообщение до удаления (порядок моков).

## Критерии приёмки
- [ ] После удаления в БД нет ни одной строки тенанта; повторный /start создаёт чистый аккаунт
- [ ] Тест-страховка от «забытой таблицы» при удалении
- [ ] Отключение IG уничтожает секреты, а не только меняет статус
