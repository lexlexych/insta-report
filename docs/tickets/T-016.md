# T-016 · Парсер событий Instagram и dedup

> **Перед началом:** прочитай `docs/plan.md` §2.2 (поток A), §5. Зависимости: T-011. Референс логики — n8n Code-нода «Parse & Route» из прежней реализации (описание ниже).

## Цель
Превратить сырое тело вебхука в типизированное событие, отсечь дубликаты, диспатчить echo/входящие.

## Шаги выполнения
1. `src/lib/pipeline/types.ts`: 
   `type IgEvent = { kind: 'incoming'|'echo'; accountId: string; contactId: string; text: string; hasAttachments: boolean; mid: string; ts: number }`.
2. `src/lib/pipeline/parse.ts` — Zod-схема тела Meta (поля, которые используем): `{ object: 'instagram', entry: [{ id, time, messaging?: [{ sender:{id}, recipient:{id}, timestamp, message?: { mid, text?, is_echo?, attachments?: [...] }, read?: {...} }] }] }`. Схема — `passthrough()` (Meta шлёт больше полей).
   `parseIgEvent(body: unknown): IgEvent | null`:
   - невалидная схема / нет `entry[0].messaging[0]` / нет `message` (например, read-событие) → null;
   - `const m = entry[0].messaging[0]`; `isEcho = m.message.is_echo === true`;
   - **правило сторон (порт из n8n):** echo → accountId = sender.id, contactId = recipient.id; иначе accountId = recipient.id, contactId = sender.id;
   - text = `m.message.text ?? ''`; hasAttachments = массив attachments непуст; mid, ts = timestamp.
3. `src/lib/pipeline/handleIgEvent.ts` (замени заглушку T-011):
   ```
   const ev = parseIgEvent(body); if (!ev) return;
   if (!(await processedEvents.tryInsert(tenantId, ev.mid))) return;   // дубликат/ретрай Meta
   if (ev.kind === 'echo') return handleEcho(tenantId, ev);            // T-022 (пока заглушка)
   if (!ev.text && !ev.hasAttachments) return;
   return handleIncoming(tenantId, ev);                                 // T-017..T-020 (пока заглушка)
   ```
   Создай файлы-заглушки `handleEcho.ts`, `handleIncoming.ts` c сигнатурами.
4. Фикстуры `tests/fixtures/ig/`: `incoming_text.json`, `echo.json`, `attachment_only.json`, `read_event.json` — собери реалистичные payload по схеме выше (object=instagram, вложенность entry/messaging).
5. Тесты `tests/pipeline.parse.test.ts`:
   - каждая фикстура → корректный IgEvent | null; в echo стороны развёрнуты (account=sender);
   - attachment_only → text='' , hasAttachments=true;
   - handleIgEvent: tryInsert=false → ни один обработчик не вызван; incoming → handleIncoming вызван c событием; мусорное тело → мягкий return без исключения.
