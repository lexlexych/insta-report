# T-034 · Вывод из эксплуатации ручного подключения Instagram (own_app) + чистка остатков архивного топика

> **Перед началом:** прочитай `docs/tickets/T-033.md` (что такое Business Login) и `docs/plan.md` §5.
> Зависимости: T-033 (влит). OAuth-флоу «Instagram API with Instagram Login» — теперь ЕДИНСТВЕННЫЙ
> способ подключения Instagram. Ручной режим `own_app` (пользователь копирует access token,
> app secret и настраивает webhook в Meta Dashboard) удаляется целиком: логика, UI, роуты, поля БД.
> Заодно удаляются мёртвые поля БД, оставшиеся после отказа от архивного топика
> (см. комментарий в `src/lib/pipeline/send.ts` — «Архивного топика больше нет»).

## Цель
1. Business Login становится обязательным и безальтернативным: убрать фича-флаг, ручной визард,
   per-tenant вебхук и все поля БД, обслуживавшие режим `own_app`.
2. Удалить из БД мёртвое поле `tenants.history_thread_id` (топик «Архив»/«История» из T-024
   упразднён, код его больше не читает и не пишет).

## Инвентаризация (что именно относится к старому флоу)

### Env — `src/lib/env.ts`, `.env.example`
- `INSTAGRAM_APP_ID`, `INSTAGRAM_APP_SECRET`, `IG_WEBHOOK_VERIFY_TOKEN` — сейчас опциональная
  группа с refine «все вместе или никак». Сделать их **обязательными** (как `TELEGRAM_BOT_TOKEN`),
  refine удалить.
- Удалить `isBusinessLoginEnabled()` и все его вызовы (feature-flag больше не существует;
  ветки «фича выключена» удаляются вместе с ним).
- Обновить комментарии в `.env.example`.

### Роуты
- **Удалить** `app/api/wh/ig/[tenantId]/route.ts` — per-tenant вебхук (GET-handshake по
  `verify_token` тенанта, POST с подписью per-tenant `app_secret`). Остаётся только глобальный
  `app/api/wh/ig/route.ts`.
- `app/api/miniapp/ig/connect/route.ts`:
  - **POST удалить** (приём accessToken/appSecret руками);
  - GET упростить: убрать `webhookUrl`, `verifyToken`, `hasToken`, `hasSecret`, `handshakeAt`,
    `businessLoginEnabled`, `connectionMode`; убрать создание записи со свежим `verify_token`
    при первом обращении (запись создаёт callback OAuth). Оставить: `status`, `igUsername`,
    `webhookLastSeenAt`.
- `app/api/miniapp/ig/status/route.ts` (диагностика T-014): чек `handshake` больше не имеет
  смысла (handshake был атрибутом per-tenant вебхука). Оставить чеки `token` и `event`;
  `event` считать по `webhook_last_seen_at` (без сравнения с `handshake_at`).
- `app/api/ig/callback/route.ts`: убрать передачу `connection_mode: 'platform_app'` в upsert
  (колонка удаляется, см. миграцию).
- `app/api/miniapp/ig/login-url/route.ts`: убрать ветку 404 `disabled` (фича всегда включена).

### Библиотеки — `src/lib`
- `db/igConnections.ts`: удалить всё про `appSecret`/`app_secret_enc` (типы `IgConnectionPatch`,
  `DecryptedIgConnection`, `decryptConnection`, `toDbPatch`, `disconnectTenant`), удалить
  `markHandshake`, дефолт `connection_mode: 'own_app'` в `upsertForTenant`, фильтр
  `.eq('connection_mode', 'platform_app')` в `getByIgAccountId`.
- `ig/status.ts` (сводка для дашборда T-027): убрать `handshakeAt` и производную от него логику;
  сверить с потребителем `app/api/miniapp/dashboard/route.ts`.
- `ig/webhook.ts`: после удаления per-tenant роута grep-ом проверить, какие хелперы остались
  без потребителей (например `timingSafeEqualStrings`, если глобальный роут его не использует),
  и удалить мёртвые.
- `db/types.gen.ts`: убрать удалённые колонки (Row/Insert/Update): `ig_connections.connection_mode`,
  `app_secret_enc`, `verify_token`, `handshake_at`; `tenants.history_thread_id`.

### UI — Mini App
- `app/(miniapp)/app/connect-instagram/page.tsx`: удалить ручной визард целиком — аккордеон
  «Продвинутый способ (вручную)», шаги с CopyField (webhook URL / verify token), PasswordField
  для token/secret, `TOKEN_MIN_LENGTH`-валидацию, сабмит на POST connect. Кнопка «Подключить
  через Instagram» — единственный и безусловный сценарий страницы.
- `src/components/miniapp/ConnectionDiagnostics.tsx`: убрать строку чека `handshake`.
- `src/lib/i18n/shared.ts`: удалить осиротевшие ключи (ru и de симметрично) — визард
  (`igStep*`, `igField*`, `igManual*` и т.п.) и `igCheck_handshake*`. Критерий — grep по ключу
  не находит использований; не удалять ключи, используемые OAuth-сценарием.
- `scripts/manual/T-021-demo.ts`: убрать `history_thread_id` из фикстуры tenant (и
  `connection_mode`/удалённые поля, если фикстуры их содержат).

### Миграция — `supabase/migrations/0010_remove_own_app.sql`
- `delete from ig_connections where connection_mode = 'own_app';` — ручные подключения не могут
  работать в новой схеме; их владельцы переподключаются через OAuth (отразить в manual-test
  как явное предупреждение человеку ДО `pnpm db:push`).
- `ig_connections`: drop колонок `connection_mode` (вместе с check-constraint
  `ig_connections_connection_mode_check`), `app_secret_enc`, `verify_token`, `handshake_at`.
- Индекс `ig_connections_platform_ig_account_id_key` (partial по `connection_mode`) пересоздать
  как `unique index ... on ig_connections (ig_account_id) where ig_account_id is not null`.
- `tenants`: drop колонки `history_thread_id`.
- Применение миграции — шаг человека в manual-test (`pnpm db:push`), Supabase MCP не используется
  (OVERRIDES §3).

## Шаги выполнения
1. Env: обязательные `INSTAGRAM_*`-переменные, удаление `isBusinessLoginEnabled` и веток
   «фича выключена» по всем потребителям (login-url, connect GET, страница подключения).
2. Миграция `0010_remove_own_app.sql` по инвентаризации выше + правка `types.gen.ts`.
3. Удаление per-tenant вебхука и чистка `src/lib/ig/webhook.ts` от мёртвых хелперов.
4. Чистка `db/igConnections.ts`, `ig/status.ts`, роутов connect/status/callback/disconnect
   (у `disconnectTenant` убрать обнуление удалённых колонок).
5. UI: страница подключения без ручного визарда, диагностика без handshake, i18n без сирот.
6. Финальная проверка мёртвого кода:
   `grep -rn "own_app\|app_secret\|verify_token\|handshake\|history_thread_id" src app scripts`
   — вхождений быть не должно (кроме `IG_WEBHOOK_VERIFY_TOKEN` — это платформенный env глобального
   вебхука, он остаётся).
7. `pnpm lint && pnpm typecheck` зелёные.
8. Manual-test `docs/manual-tests/T-034.md` по шаблону `docs/orchestration/manual-testing.md`.

## Manual-test (минимальный состав проверок)
- Предупреждение: миграция удаляет все `own_app`-подключения — перед `pnpm db:push` убедиться,
  что это ожидаемо.
- `pnpm db:push` → в схеме нет `connection_mode`, `app_secret_enc`, `verify_token`, `handshake_at`,
  `tenants.history_thread_id`; unique-индекс по `ig_account_id` на месте.
- Приложение не стартует без `INSTAGRAM_APP_ID`/`INSTAGRAM_APP_SECRET`/`IG_WEBHOOK_VERIFY_TOKEN`
  (понятная ошибка env-валидации).
- Страница `/app/connect-instagram`: только OAuth-кнопка, аккордеона «вручную» нет; полный
  OAuth-проход работает; «Переподключить» работает (идемпотентный upsert).
- Старый URL `/api/wh/ig/<tenantId>` отвечает 404; глобальный `/api/wh/ig` принимает DM →
  карточка в Telegram приходит.
- Диагностика: чеки `token` и `event` работают, чека `handshake` нет ни в API, ни в UI.
- Дашборд и «Отключить Instagram» (settings) работают.

## Вне объёма
- Несколько Meta App одновременно (`app_key`, per-app webhook пути) — см. T-033 «Вне объёма».
- App Review / Advanced Access.
