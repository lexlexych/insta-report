# T-015 · Cron: рефреш долгоживущих токенов

> **Перед началом:** прочитай `docs/plan.md` §2.3, §5. Зависимости: T-013.

## Цель
Ежесуточный рефреш IG-токенов (живут 60 дней) c алертами при провале.

## Шаги выполнения
1. Добавь в `src/lib/ig/client.ts`: `refreshToken(token)` → GET `${GRAPH}/refresh_access_token?grant_type=ig_refresh_token&access_token={token}` → `{ access_token, expires_in }`; ошибки классифицируй как в T-013.
2. `app/api/cron/refresh-tokens/route.ts` GET:
   - защита: `req.headers.get('authorization') === 'Bearer ' + env.CRON_SECRET`, иначе 401;
   - `export const maxDuration = 60;`
   - выбери `igConnections.listActiveForRefresh(7)` (token_refreshed_at старше 7 дней ИЛИ null, status='active');
   - последовательно (не Promise.all — щадим rate limit) для каждой: refreshToken → `markTokenRefreshed(tenantId, encrypt(newToken))`;
   - ошибка одного тенанта НЕ прерывает цикл: `setStatus(tenantId,'error', msg)`; отправь владельцу в TG (через `src/lib/tg/api.ts`) сообщение «⚠️ Подключение Instagram требует внимания: токен не удалось обновить. Откройте панель → Подключение» c кнопкой web_app; отправь алерт админам (`env.ADMIN_TELEGRAM_IDS`, split(','));
   - ответ: `{ processed, refreshed, failed: [tenantId...] }` + лог.
3. `vercel.json` в корне:
   ```json
   { "crons": [ { "path": "/api/cron/refresh-tokens", "schedule": "0 4 * * *" } ] }
   ```
   Примечание в docs: Vercel Cron вызывает без заголовков — используй query `?key=` ИЛИ настрой `CRON_SECRET` через встроенный механизм Vercel (заголовок `authorization` проставляется автоматически, см. доку Vercel Cron + CRON_SECRET). Реализуй проверку, совместимую c обоими (заголовок ИЛИ query key).
4. Тесты `tests/cron.refresh.test.ts` (моки ig, db, tg): 3 подключения, второе падает IgAuthError → первое и третье обновлены; уведомление владельцу и админам отправлено один раз; ответ содержит failed=[t2]; неверный секрет → 401 и нулевые вызовы.

## Критерии приёмки
- [ ] Провал одного тенанта не влияет на остальных
- [ ] Новый токен шифруется перед записью
- [ ] vercel.json c расписанием закоммичен; README/docs дополнены про CRON_SECRET
