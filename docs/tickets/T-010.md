# T-010 · Генерация базы знаний и system prompt (бэкенд)

> **Перед началом:** прочитай `docs/plan.md` §1, §2, §5. Зависимости: T-004, T-008.

## Цель
Эндпоинты генерации/редактирования базы знаний; шаблоны промптов проекта.

## Шаги выполнения
1. Создай `src/lib/llm/prompts.ts` — все промпты проекта живут здесь как функции-шаблоны:
   - `kbGenerationPrompt(orgName, orgDescription)`: попроси LLM вернуть JSON `{ knowledge_base: string, system_prompt: string }`. Требования в промпте: knowledge_base — markdown c разделами «О компании», «Услуги и цены», «Практическая информация» (адрес/часы/контакты, только из описания, ничего не выдумывать; отсутствующее пометить как «уточнить у владельца»), «Тон общения», «Частые вопросы» (3–5 пар, выведенных из описания); system_prompt — инструкция ассистенту службы поддержки: представляться от имени {orgName}, отвечать кратко и дружелюбно, на языке клиента, опираться только на базу знаний, при неуверенности — предложить дождаться ответа сотрудника; запрет выдумывать цены и обещания.
   - Заглушки для будущих тикетов: `classifyPrompt(...)` (T-018), `draftPrompt(...)` (T-019) — объяви и пометь TODO.
2. `app/api/miniapp/onboarding/generate/route.ts` POST (requireTenant): Zod body `{ orgName, orgDescription, overwrite? }`. Если у tenant уже есть knowledge_base и `overwrite !== true` → 409 `{ code: 'exists' }` (фронт покажет подтверждение). Вызов `completeJSON` (MODEL_DRAFT, схема `{knowledge_base, system_prompt}`) → `tenants.update(...)` (+ org_name, org_description) → rpc `seed_default_labels(tenantId)` → ответ `{ knowledgeBase, systemPrompt }`.
3. `app/api/miniapp/knowledge/route.ts`: GET — вернуть kb и prompt; PUT — Zod `{ knowledgeBase?, systemPrompt? }` (max 20_000 символов каждое), update.
4. `app/api/miniapp/tenant/route.ts`: PATCH `{ onboardingStep }` (enum-валидация) — если ещё не создан в T-009.
5. Тесты `tests/onboarding.generate.test.ts` (мок llm и db): happy-path сохраняет оба поля и вызывает seed; повторный вызов без overwrite → 409; c overwrite → перезапись; невалидный body → 400.

## Критерии приёмки
- [ ] Промпты — только в `src/lib/llm/prompts.ts`
- [ ] 409-протокол перезаписи реализован и покрыт тестом
- [ ] Метка «Без категории» создаётся при генерации
