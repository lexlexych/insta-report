# T-044 · Визард подключения Instagram: приглашение тестировщика, OAuth, результат

> **Перед началом:** прочитай `docs/plan.md` §1, §5 и `docs/tickets/OVERRIDES.md`.
> Зависимости: T-033 (Business Login), T-042 (точка входа с финального экрана онбординга),
> T-043 (connect-статус и гейт).
> Скриншот для шага 1 предоставляет ЧЕЛОВЕК: файл `public/images/ig-tester-invite.png`
> (страница Instagram «Приложения и сайты → Приглашения для тестировщиков»). Человек добавляет
> файл в ветку до мержа; в коде — обычный `<img src="/images/ig-tester-invite.png">`, чтобы
> отсутствие файла не ломало сборку.

## Цель

Страница `/app/connect-instagram` становится пошаговым визардом из 3 экранов (приглашение
тестировщика → OAuth → результат); на время визарда в шапке мини-аппа — стрелка «Назад»
(Telegram BackButton) вместо дефолтного крестика закрытия.

## Гейт

1. При открытии страницы запросить `GET /api/miniapp/ig/status` (T-043):
   - `connect` = `awaiting_admin` или `none` → визард не показывать; вместо него экран-статус
     «Ожидание подтверждения администратором» (иконка-часы, краткое пояснение, что администратор
     подтверждает подключение и придёт сообщение от бота);
   - `active` → сразу шаг 3 (успех);
   - `ready` / `error` → шаг 1.

## Шаг 1 — приглашение для тестировщиков

2. Контент: заголовок; скриншот `public/images/ig-tester-invite.png`; пояснение по шагам:
   откройте настройки Instagram, во вкладке «Приглашения для тестировщиков» нажмите «Принять»
   для приложения «neuroposts-IG», затем вернитесь в мини-апп (стрелка «Назад» вверху) и
   нажмите «Далее». Приписка: «Если вы уже принимали приглашение раньше — просто нажмите
   Далее». Кнопка-ссылка «Открыть настройки Instagram» →
   `openLink('https://www.instagram.com/accounts/manage_access/')`. Кнопка «Далее» всегда
   активна → шаг 2. Шаг показывается при КАЖДОМ запуске визарда (программно проверить принятие
   приглашения невозможно).

## Шаг 2 — OAuth

3. Переиспользовать логику `InstagramConnectPanel` (получение URL из
   `/api/miniapp/ig/login-url`, открытие через `openLink`, перепроверка статуса по фокусу окна
   и кнопке «Проверить») — компонент перестроить под шаг визарда; после T-041 страница
   `/app/connect-instagram` — единственное место его использования. Статус стал `active` →
   автоматический переход на шаг 3 (успех). Возврат с `?ig=denied|error` или статус `error` →
   шаг 3 (ошибка).

## Шаг 3 — результат

4. Успех: анимированная галочка (инлайн-SVG, CSS-анимация обводки через
   `stroke-dasharray`/`stroke-dashoffset`, БЕЗ новых зависимостей) + «Подключение выполнено» +
   кнопка «К дашборду» → `/app`.
   Ошибка: восклицательный знак (аналогичное анимированное появление) + «Не удалось подключить
   Instagram» + краткая причина, если известна (`denied` и т.п.) + кнопка «Попробовать снова» →
   шаг 2.

## Стрелка «Назад» в шапке мини-аппа

5. На странице визарда показывать Telegram BackButton (`@telegram-apps/sdk-react`) — Telegram
   заменяет крестик закрытия на стрелку «Назад». Обработка нажатия:
   - шаг 2 → шаг 1; шаг 3 (ошибка) → шаг 2;
   - шаг 1, шаг 3 (успех) и экран-статус ожидания → уход со страницы (на дашборд либо назад
     в историю — откуда пришли: дашборд или финальный экран онбординга).
   При размонтировании страницы BackButton скрывать и снимать обработчик — остальной мини-апп
   ведёт себя как раньше (крестик).
6. i18n ru/de всех строк; без новых зависимостей.
7. Manual-test `docs/manual-tests/T-044.md`: полный успешный проход (шаг 1 со скриншотом и
   рабочей ссылкой → OAuth → анимированная галочка → «К дашборду»); отмена/ошибка OAuth →
   экран ошибки → «Попробовать снова» → повторный OAuth; BackButton на каждом шаге ведёт себя
   по схеме из п.5 и исчезает вне визарда; гейт: при `pending`-заявке визард недоступен, виден
   статус ожидания; вход в визард с дашборда и с финального экрана онбординга (T-042); при уже
   активном подключении сразу открывается экран успеха.

## Критерии приёмки

- [ ] `/app/connect-instagram` — визард из 3 шагов; шаг 1 со скриншотом, ссылкой на
      `manage_access` и всегда активной «Далее»
- [ ] OAuth-шаг работает как прежняя панель; `active` → автоматически экран успеха с
      анимированной галочкой; отказ/ошибка → экран с «!» и кнопкой «Попробовать снова»
- [ ] BackButton (стрелка вместо крестика) показывается на всех экранах визарда и ведёт себя по
      схеме п.5; на остальных страницах мини-аппа не показывается
- [ ] При неподтверждённой заявке (`awaiting_admin`/`none`) визард недоступен — экран «Ожидание
      подтверждения администратором»
- [ ] `pnpm lint && pnpm typecheck` зелёные; manual-test написан
