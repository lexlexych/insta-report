# T-013 · Клиент Instagram Graph API

> **Перед началом:** прочитай `docs/plan.md` §2.2, §5. Зависимости: T-003, T-004. Единственная точка вызова graph.instagram.com.

## Цель
`src/lib/ig/client.ts` + `src/lib/ig/split.ts`: типизированный клиент c разбиением длинных сообщений.

## Шаги выполнения
1. Константы: `const GRAPH = 'https://graph.instagram.com'; const V = 'v25.0';`
2. Общий приватный `igFetch(token, path, init?)`:
   - `fetch(`${GRAPH}/${V}${path}`, { headers: { Authorization: `Bearer ${token}` }, signal: AbortSignal.timeout(15_000), ...init })`;
   - не-ok ответ: распарси `{ error }`; классифицируй: `error.type === 'OAuthException'` или code 190 → `IgAuthError`; http 429 или code 4/17/613 → `IgRateLimitError`; иначе `IgApiError(code, message)`. Все ошибки несут `path`, но НЕ несут токен.
3. Публичные функции:
   - `getAccount(token)` → GET `/me?fields=user_id,username` → `{ igAccountId, username }`;
   - `getUsername(token, igsid)` → GET `/{igsid}?fields=username` → string (ошибку глотать → `null`, юзер мог ограничить доступ);
   - `getConversation(token, igAccountId, contactId, limit=20)` → GET `/me/conversations?user_id={contactId}&fields=messages.limit({limit}){message,from,created_time}` → массив `{ text, fromId, createdTime }[]` (отсортируй по created_time ASC; отфильтруй элементы без message);
   - `sendMessage(token, igAccountId, igsid, text)` → разбей через `splitMessage(text)` и отправь последовательно POST `/{igAccountId}/messages` body `{ recipient: { id: igsid }, message: { text: part } }`; собери массив mid; ошибка на части N → бросай c указанием отправленных частей.
4. `src/lib/ig/split.ts` — `splitMessage(text, maxBytes = 900): string[]`:
   - если byteLength (TextEncoder) ≤ max → [text];
   - дели по абзацам (\n\n), затем по предложениям (регэксп по `. ! ? …` c учётом пробела после), затем — жёстко по словам; накапливай части, замеряя именно БАЙТЫ utf-8;
   - гарантия: ни одна часть не превышает maxBytes; слова не режутся посередине (кроме патологического слова длиннее лимита — тогда режь по байтовой границе символа, не разрывая code point).
5. Тесты `tests/ig.client.test.ts` (мок fetch):
   - getConversation сортирует и маппит поля;
   - sendMessage для текста 2500 байт кириллицы делает 3+ POST, каждый ≤900 байт, конкатенация частей === исходник;
   - split: кириллица/emoji не рвутся посередине code point (проверь декодированием каждой части);
   - OAuthException → IgAuthError; 429 → IgRateLimitError;
   - токен отсутствует в message/стеке ошибок (assert на строку).

## Критерии приёмки
- [ ] Разбиение по байтам UTF-8 корректно для кириллицы и emoji
- [ ] Классификация ошибок покрыта тестами; токен не утекает в ошибки
- [ ] Ни один модуль вне `src/lib/ig` не вызывает graph.instagram.com
