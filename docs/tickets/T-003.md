# T-003 · Модуль шифрования секретов

> **Перед началом:** прочитай `docs/plan.md` §5. Зависимости: T-001. Модуль будет единственным способом записи/чтения токенов Instagram и app_secret в БД.

## Цель
`src/lib/crypto.ts` — AES-256-GCM шифрование строк с версионированным форматом.

## Шаги выполнения
1. Реализуй на `node:crypto` (без сторонних библиотек):
   - `getKey(): Buffer` — декодирует `env.ENCRYPTION_KEY` из base64, проверяет длину 32 байта, кэширует.
   - `encrypt(plain: string): string`:
     iv = `randomBytes(12)`; `createCipheriv('aes-256-gcm', key, iv)`;
     результат: `` `v1:${iv.b64}:${authTag.b64}:${ciphertext.b64}` ``.
   - `decrypt(payload: string): string` — парсит формат, проверяет префикс `v1`, `createDecipheriv`, `setAuthTag`, возвращает utf-8. Любая ошибка формата/тега → бросай собственный класс `DecryptError` (без включения payload в message).
2. Экспортируй также `isEncrypted(value: string): boolean` (проверка префикса `v1:`) — пригодится в репозиториях для защиты от двойного шифрования.
3. Создай `scripts/gen-key.ts`: печатает `randomBytes(32).toString('base64')` и подсказку «добавьте в ENCRYPTION_KEY». Скрипт `pnpm gen:key`.
4. Тесты `tests/crypto.test.ts` (env замокать через `vi.stubEnv` / прямую передачу ключа — сделай `getKey` перегружаемым для тестов):
   - roundtrip: `decrypt(encrypt(s)) === s` для ascii, кириллицы, emoji, пустой строки;
   - два вызова `encrypt(s)` дают разные строки (разный iv);
   - порча одного символа в tag или ciphertext → `DecryptError`;
   - `decrypt('мусор')` → `DecryptError`.
