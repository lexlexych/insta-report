# T-023 · UI управления категориями

> **Перед началом:** прочитай `docs/plan.md` §1, §5. Зависимости: T-007, T-010.

## Цель
CRUD категорий (меток) в Mini App: имя, «когда применяется», «как отвечать», порядок.

## Шаги выполнения
1. Бэкенд `app/api/miniapp/labels/route.ts` (+ `[id]/route.ts`):
   - GET: `labels.listByTenant` (сортировка по sort, «Без категории» всегда последняя);
   - POST: Zod `{ name: 1..50, description: 0..300, instruction: 0..1000 }`; дубль имени (unique violation) → 409;
   - PUT [id]: те же поля + `sort`; попытка переименовать «Без категории» → 403;
   - DELETE [id]: ForbiddenLabelError → 403; в репозитории drafts.label_id имеет `on delete set null` (проверь миграцию T-002; если нет — добавь миграцию 0004);
   - PUT `/api/miniapp/labels/reorder`: `{ ids: string[] }` → массовое обновление sort по индексу.
2. UI `/app/labels`:
   - список карточек: имя, description (серым), счётчик черновиков за 30 дней (добавь в GET агрегат из drafts) — опционально, если просто;
   - «Без категории» — c замком, без edit/delete;
   - FAB «+ Категория» → модальная форма (bottom sheet): три поля c подсказками-примерами (name: «Запись на услугу»; description: «Клиент хочет записаться или спрашивает свободное время»; instruction: «Уточни желаемую дату и услугу, предложи два ближайших окна»);
   - long-press/иконка → меню Изменить/Удалить (удаление c confirm);
   - перетаскивание для порядка: используй HTML5 drag или кнопки ↑↓ (проще и надёжнее в WebView — выбери кнопки);
   - все строки через i18n.
3. Тесты: API — дубль имени 409; rename «Без категории» 403; reorder меняет sort. Компонентный: форма валидирует и создаёт (мок fetch).

## Критерии приёмки
- [ ] «Без категории» неудаляема/непереименуема на всех уровнях (UI, API, repo)
- [ ] Порядок сохраняется и влияет на выдачу GET
