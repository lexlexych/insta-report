# T-045 · Zernio: env-группа и API-клиент

> **Перед началом:** прочитай `docs/research/zernio.md` (обязательно — там все endpoints,
> payload'ы и ограничения) и `docs/plan.md` §5. Зависимости: T-001 (done).
> Эпик 12 «Instagram через Zernio», ветка эпика — `zernio`.

## Цель
Фундамент интеграции: опциональная группа env-переменных Zernio и типизированный HTTP-клиент
`src/lib/zernio/client.ts` со всеми вызовами, которые понадобятся эпику. Поведение приложения
при незаполненных переменных не меняется вовсе.

## Шаги выполнения
1. **Env** (`src/lib/env.ts` + `.env.example` с комментариями):
   - `ZERNIO_API_KEY` — опциональная (`z.string().min(1).optional()`);
   - `ZERNIO_WEBHOOK_SECRET` — опциональная; секрет подписи вебхуков (генерим сами,
     `pnpm tsx scripts/gen-key.ts` подходит);
   - `ZERNIO_API_BASE` — опциональная с дефолтом `https://zernio.com/api`;
   - `.refine`: `ZERNIO_API_KEY` и `ZERNIO_WEBHOOK_SECRET` заданы либо обе, либо ни одной
     (по образцу исторического паттерна optional-группы из T-033; сейчас в env.ts такого
     refine нет — добавь новый рядом с refine ENCRYPTION_KEY).
   - Хелпер `isZernioEnabled(): boolean` — true, когда обе заданы. Экспорт из `env.ts`.
2. **Клиент** `src/lib/zernio/client.ts` (новый каталог `src/lib/zernio/`) — по образцу
   `src/lib/ig/client.ts`: приватный `zernioFetch<T>(path, init?)` с `Authorization: Bearer
   env.ZERNIO_API_KEY`, `Content-Type: application/json`, таймаутом 15 с (AbortController)
   и разбором ошибок. Функции (сигнатуры и поля ответов — в `docs/research/zernio.md` §3–7):
   - `createProfile(name: string): Promise<{ profileId: string }>` — `POST /v1/profiles`;
   - `listProfiles(): Promise<Array<{ profileId: string; name: string }>>`;
   - `getConnectUrl(platform: 'instagram', profileId: string, redirectUrl: string):
     Promise<{ authUrl: string }>` — `GET /v1/connect/{platform}?profileId=&redirect_url=`
     (redirect_url URL-энкодить);
   - `listAccounts(filter: { profileId?: string; platform?: string; status?: 'connected'
     | 'disconnected' }): Promise<ZernioAccount[]>` — маппить `_id` → `accountId`,
     populated `profileId._id` → `profileId`;
   - `deleteAccount(accountId: string): Promise<void>`;
   - `getAccountHealth(accountId: string): Promise<{ ok: boolean; raw: unknown }>`;
   - `sendMessage(conversationId: string, accountId: string, text: string, opts?:
     { messageTag?: 'HUMAN_AGENT' }): Promise<{ messageId: string }>` —
     `POST /v1/inbox/conversations/{id}/messages`; при `opts.messageTag` добавлять
     `messagingType: 'MESSAGE_TAG'`;
   - `getConversationMessages(conversationId: string, accountId: string, opts?: { limit?:
     number; sortOrder?: 'asc' | 'desc' }): Promise<ZernioInboxMessage[]>`;
   - вебхук-настройки (нужны T-048 для setup-скрипта): `listWebhookSettings()`,
     `createWebhookSetting({ name, url, secret, events, isActive })`,
     `updateWebhookSetting({ _id, ...patch })`.
3. **Типы** `src/lib/zernio/types.ts`: `ZernioAccount { accountId, profileId, platform,
   username, displayName, isActive }`, `ZernioInboxMessage { id, conversationId, accountId,
   platform, message, senderId, senderName, direction: 'incoming' | 'outgoing', createdAt,
   attachments: Array<{ type, url }> }`, `ZernioWebhookSetting`. Не выдумывать поля —
   только зафиксированные в research-файле.
4. **Ошибки** (по образцу `IgApiError`-семейства): `ZernioApiError(status, message)`,
   `ZernioAuthError` (401), `ZernioAddonError` (403 — Inbox addon / лимит плана),
   `ZernioRateLimitError` (429). В сообщения ошибок и логи НЕ включать API-ключ,
   заголовки запроса и полный URL с ключом.
5. **Краевые случаи** (вместо автотестов — обработать в коде): не-JSON ответ; таймаут;
   402/403 с телом `{ error, requiresAddon }`; пустой `accounts` в `listAccounts`;
   вызов любого метода при `!isZernioEnabled()` → бросать `ZernioApiError` с понятным
   сообщением `zernio_disabled` (защита от неправильного использования).
6. **Demo-скрипт** `scripts/manual/T-045-demo.ts`: при заполненном env — `listProfiles`,
   `listAccounts`, `listWebhookSettings` с печатью результата (без секретов); при пустом —
   печатает `isZernioEnabled() === false` и завершается кодом 0.
7. **Manual-test** `docs/manual-tests/T-045.md` по шаблону `docs/orchestration/manual-testing.md`:
   подготовка (человек): регистрация на zernio.com, создание API-ключа в дашборде, заполнение
   env; проверки: demo-скрипт с ключом и без; негативная: битый ключ → `ZernioAuthError`,
   секрета нет в выводе/логах.

## Вне объёма
- БД, роуты, вебхуки, UI — следующие тикеты эпика (T-046…T-051).
- Каналы кроме Instagram (клиент параметризован `platform`, но включаем только `instagram`).

- [ ] При незаполненных `ZERNIO_*` приложение собирается и ведёт себя ровно как раньше;
      `isZernioEnabled()` = false

- [ ] Все методы клиента типизированы, с таймаутом и типизированными ошибками; API-ключ
      не попадает в логи, сообщения ошибок и исключения
- [ ] `.env.example` дополнен тремя переменными с комментариями
- [ ] Demo-скрипт работает в обоих режимах (с ключом / без)
- [ ] lint, typecheck зелёные; manual-test написан
