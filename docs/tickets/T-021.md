# T-021 · Обработчик кнопки «✅ Отправить»

> **Перед началом:** прочитай `docs/plan.md` §2.2 (поток B), §5. Зависимости: T-013, T-020.

## Цель
Подтверждение черновика: атомарный claim, защита от «уже ответили вручную», отправка в Instagram, обновление карточки, retry.

## Шаги выполнения
1. В `src/lib/tg/bot.ts` замени заглушку onCallback:
   - парси data: `send:{id}` → `handleSendCallback(ctx, id)`; `retry:{id}` → `handleRetryCallback`; иное → `answerCallbackQuery()` молча.
2. `src/lib/pipeline/send.ts` — `handleSendCallback(ctx, draftId)`:
   1. `answerCallback(ctx.callbackQuery.id, 'Отправляю…')` СРАЗУ (иначе у пользователя крутится спиннер);
   2. `draft = drafts.claimPendingToSending(draftId)`; null → `editMessageHTML(карточка + statusLine '⌛️ Карточка устарела')` (draft мог быть sent/cancelled) и выход;
   3. `tenant`, `conn` — загрузка; conn error → status 'error' на draft + editMessage c текстом ошибки подключения;
   4. **Re-check ручного ответа:** `msgs = getConversation(...)`; есть сообщение c fromId===igAccountId и createdTime > draft.trigger_ts → `drafts.setStatus(id,'skipped_manual')`, editMessage: карточка c statusLine '⚠️ Отменено: вы уже ответили вручную', БЕЗ клавиатуры; выход;
   5. `sendMessage(token, igAccountId, draft.contact_id, draft.draft_text)`;
   6. успех: `setStatus('sent')`, `messageLog.add(..., 'out', draft.draft_text)`, `usageStats.increment({drafts_sent:1})`, editMessage карточки на месте в variant='sent' (шапка '✅ отправлено в {HH:MM} <получатель>', время в TZ Europe/Berlin), без клавиатуры и без нижней строки статуса;
   7. ошибка IG: `setStatus('error', message)`; editMessage: карточка + '❌ Ошибка отправки: {краткое сообщение}' + клавиатура c одной кнопкой «🔁 Повторить» (`retry:{draftId}`).
3. `handleRetryCallback`: `drafts.setStatus(id,'pending')` только из статуса `error` (условный update, вернул null → «карточка устарела»); затем повтори handleSendCallback-логику (вынеси общую часть в функцию `attemptSend(draftId, cbId)`).
4. Все editMessage используют `renderDraftCard` c соответствующим statusLine — карточка сохраняет контекст (кто писал, черновик).
5. Тесты `tests/pipeline.send.test.ts` (моки):
   - happy path: claim → recheck (нет ручного) → ig.sendMessage → status sent → edit карточки на месте в «✅ отправлено в …»;
   - двойной клик: первый claim ok, второй null → второй НЕ вызывает ig.sendMessage;
   - ручной ответ (created > trigger_ts) → skipped_manual, ig.sendMessage НЕ вызван;
   - IgApiError → status error, кнопка retry в клавиатуре;
   - retry из error → повторная попытка; retry из sent → «устарела», без отправки;
   - answerCallback вызывается первым (порядок моков).

## Критерии приёмки
- [ ] Дубль-отправка невозможна (атомарный claim, тест на двойной клик)
- [ ] Ручной ответ владельца всегда выигрывает у кнопки
- [ ] Из error есть путь retry; карточка всегда отражает финальный статус
