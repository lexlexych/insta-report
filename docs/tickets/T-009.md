# T-009 · Визард онбординга (UI)

> **Перед началом:** прочитай `docs/plan.md` §1, §2, §5. Зависимости: T-006, T-007. Бэкенд генерации — T-010; здесь используй его эндпоинты (при параллельной работе — замокай fetch и оставь контракт).

## Цель
Пошаговый онбординг в Mini App c сохранением прогресса.

## Шаги выполнения
1. Страница `app/(miniapp)/app/onboarding/page.tsx`. Локальная state-machine шагов: `welcome → org_form → generating → review_kb → done`. Текущий шаг синхронизируется c `tenants.onboarding_step` (`PATCH /api/miniapp/tenant { onboardingStep }` — добавь этот простой роут через requireTenant).
2. Роутинг-гард: в `useTenant()` после auth, если `onboarding_step` не `done` и путь не `/app/onboarding` — redirect на онбординг; и наоборот, `done` → редирект c онбординга на дашборд.
3. Шаг **welcome**: 3 тезиса о продукте (иконка + строка), CTA через Telegram MainButton («Начать») — используй `mainButton` из SDK: setText/show/onClick; на остальных шагах MainButton — основная кнопка шага, скрывай при unmount.
4. Шаг **org_form**: поля «Название организации» (min 2 символа) и «Расскажите о бизнесе» (textarea, min 80 символов, счётчик; плейсхолдер-пример: услуги, цены, адрес, часы работы, тон общения). Валидация — инлайн, ошибки через i18n. Submit → шаг generating.
5. Шаг **generating**: POST `/api/miniapp/onboarding/generate` `{ orgName, orgDescription }`; скелетон-анимация + подписи этапов; ошибка → тост + кнопка «Повторить».
6. Шаг **review_kb**: показ сгенерированной базы знаний (readonly-рендер markdown простым способом: заголовки/списки/абзацы; без сторонних тяжёлых библиотек — простая функция-конвертер или `react-markdown`, если бандл < 30kb) + кнопка «Редактировать» → textarea → `PUT /api/miniapp/knowledge`. CTA «Всё верно».
7. Шаг **done**: поздравление; две кнопки: «Подключить Instagram» → `/app/connect-instagram`, «Попробовать тест-чат» → `/app/simulator`; установка onboarding_step='done'.
8. Тесты: компонентные (vitest + @testing-library/react, environment jsdom для этого файла): валидация org_form блокирует переход; ошибка generate показывает retry; happy-path проходит все шаги (fetch замокан).

## Критерии приёмки
- [ ] Закрытие и повторное открытие Mini App возвращает на незавершённый шаг
- [ ] MainButton Telegram используется как основная CTA на каждом шаге
- [ ] Happy-path тест зелёный
