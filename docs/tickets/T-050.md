# T-050 · Zernio: отправка ответа кнопкой «Отправить» и гашение собственного echo

> **Перед началом:** прочитай `docs/research/zernio.md` §6, §8.2 и `docs/plan.md` §5. Изучи
> `src/lib/pipeline/send.ts` (`attemptSend`, `handleSendCallback`, `handleRetryCallback`,
> `finalizeSentDraft`). Зависимости: T-045, T-049 (плюс done: T-021, T-022).
> Эпик 12, ветка эпика — `zernio`.

## Цель
Кнопка «Отправить» на карточке работает для черновиков с `provider='zernio'`: ответ уходит
через Zernio Inbox API, собственное echo не порождает повторной обработки, карточка
финализируется как сейчас. Черновики `provider='meta'` идут по прежнему пути без изменений.

## Шаги выполнения
1. **Ветвление в `attemptSend`** (`src/lib/pipeline/send.ts`): после
   `drafts.claimPendingToSending` смотреть `draft.provider`:
   - `'meta'` (и NULL/отсутствует — старые строки) → текущий код без изменений;
   - `'zernio'` → новая ветка (пп. 2–4). Оформи как выделенную функцию или объект-стратегию
     `{ checkConnection, antiDoubleCheck, send }` на провайдера — но не перестраивай файл
     целиком: цель — минимальный понятный diff, общие шаги (claim, markError, finalize,
     messageLog, usageStats) остаются общими.
2. **Проверки соединения (zernio-ветка)**: `zernioAccounts.getForTenant(tenantId)` —
   `status === 'active'` и есть `zernio_account_id`; `draft.zernio_conversation_id` заполнен.
   Иначе → `markError` с человекочитаемой причиной на карточке (как сейчас при отсутствии
   `accessToken`).
3. **Анти-двойник (паритет с Meta-веткой)**: `getConversationMessages(
   draft.zernio_conversation_id, zernio_account_id, { limit: 20, sortOrder: 'desc' })`;
   есть сообщение `direction === 'outgoing'` с `createdAt > draft.trigger_ts` → статус
   `skipped_manual`, карточка «уже отвечено вручную» (тот же UX, что в Meta-ветке).
   Ошибка/таймаут этого вызова НЕ блокирует отправку (как сейчас: анти-двойник —
   best-effort; посмотри текущую обработку и повтори).
4. **Отправка**: длинный текст режь существующим `splitMessage` (`src/lib/ig/split.ts` —
   лимит Instagram 1000 символов действует и через Zernio); для каждой части
   `zernio.sendMessage(conversationId, accountId, part)`;
   - каждый полученный `messageId` СРАЗУ регистрируй `processedEvents.tryInsert(tenantId,
     messageId)` — гашение собственного `message.sent` (research §8.2). ВНИМАНИЕ:
     `messageId` send-ответа может быть внутренним ID Zernio, а `mid` в вебхуке —
     `platformMessageId` Meta; если пилот покажет расхождение namespace'ов — дополнительная
     страховка уже есть: T-049 гасит outgoing-события `message.sent` нашего аккаунта через
     дедуп+echo-ветку, двойной карточки не будет. Зафиксируй фактическое поведение
     в manual-test;
   - частичный сбой на N-й части — обработай по образцу `IgPartialSendError`: зарегистрируй
     отправленные id, статус `error` с указанием «отправлено частично», кнопка `retry:`;
   - ошибка «окно 24 часа закрыто» (400 `PLATFORM_LIMITATION`) → ОДИН повтор с
     `{ messageTag: 'HUMAN_AGENT' }` (research §6; Instagram разрешает только этот тег);
     повтор тоже неудачен → `markError`.
5. **После успеха** — общий код: статус `sent`, `messageLog.add(..., 'out', ...)`,
   `usageStats.increment({ draftsSent: 1 })`, `finalizeSentDraft` — без изменений.
   `handleRetryCallback` должен работать для zernio-черновиков без правок (проверь, что
   retry снова попадает в ветвление по provider).
6. **Краевые случаи**: черновик zernio, но `isZernioEnabled()` уже false (переменные убрали)
   → `markError('zernio_disabled')`, не исключение; аккаунт отключился между карточкой и
   кликом (`status='error'` после `account.disconnected`) → `markError` с подсказкой
   переподключить; `ZernioRateLimitError` → `markError` + retry-кнопка.
7. **Demo-скрипт** `scripts/manual/T-050-demo.ts`: `attemptSend` с замоканными deps
   (по образцу `T-021-demo.ts`): happy path zernio-драфта; skipped_manual; 24h-ошибка →
   повтор с HUMAN_AGENT; частичный сбой; meta-драфт → старый путь (спай, что вызван
   `ig/client.sendMessage`, а не zernio).
8. **Manual-test** `docs/manual-tests/T-050.md`: DM → карточка → «Отправить» → ответ виден
   в Instagram, карточка «✅ отправлено», в message_log строка 'out', в usage_stats +1;
   собственный ответ не породил новой карточки (echo погашен); ручной ответ из Instagram
   до клика → «уже отвечено вручную»; негативная — отключить аккаунт в дашборде Zernio и
   нажать «Отправить» → карточка с ошибкой и retry.
