# T-051 · Zernio: область «Подключить Instagram через Zernio» на дашборде

> **Перед началом:** прочитай `docs/research/zernio.md` §4 и `docs/plan.md` §5. Изучи
> дашборд `app/(miniapp)/app/page.tsx` (существующая область подключения Instagram),
> `app/api/miniapp/dashboard/route.ts` и открытие OAuth-URL в
> `app/(miniapp)/app/connect-instagram/page.tsx` (embedded/desktop-логика T-044).
> Зависимости: T-047 (плюс done: T-027). Эпик 12, ветка эпика — `zernio`.

## Цель
На дашборде рядом с существующей областью «Подключение Instagram» появляется вторая —
«Подключить Instagram через Zernio»: одна кнопка, OAuth-экран, возврат в мини-апп, статус
и «Отключить». Никаких промежуточных шагов (заявка админу, тестировщик) и отдельной
страницы-визарда. Старая область не меняется.

## Шаги выполнения
1. **Дашборд-API**: в ответ `GET /api/miniapp/dashboard` добавь блок
   `zernio: { enabled: boolean, status: 'none' | 'pending' | 'active' | 'disconnected'
   | 'error', username: string | null }` (из `zernioAccounts.getForTenant` +
   `isZernioEnabled()`; сетевых вызовов к Zernio не делать). При `enabled: false`
   остальные поля не важны — UI блок не рендерит.
2. **Секция на дашборде** (`app/(miniapp)/app/page.tsx`), под существующей секцией
   подключения, в той же стилистике (`<section>` со статусными классами):
   - `enabled === false` → секции нет вовсе (продакшен без ключей Zernio выглядит как сейчас);
   - `status: 'none' | 'disconnected'` → заголовок, 1–2 строки описания («подключение в
     один клик, без приглашения тестировщика»), кнопка «Подключить» / «Переподключить»;
   - `status: 'pending'` → то же + ненавязчивая пометка «подключение не завершено»;
   - `status: 'active'` → `@username`, бейдж «активно», кнопка «Отключить»
     (`POST /api/miniapp/zernio/disconnect`, с `confirm`-диалогом как у существующей
     кнопки отключения IG);
   - `status: 'error'` → текст ошибки-подсказки («подключение разорвано, переподключите»)
     + кнопка «Переподключить».
3. **Клик «Подключить»**: `POST /api/miniapp/zernio/connect` c
   `{ embedded }` (embedded — тот же признак мобильного TG, что в T-044:
   `retrieveLaunchParams().tgWebAppPlatform ∈ {ios, android, android_x}`) → `{ url }`:
   - embedded → `window.location.assign(url)`;
   - desktop → `window.Telegram.WebApp.openLink(url)` (фолбэк `window.open(url, '_blank')`);
   - на время запроса кнопка disabled со спиннером; 409 `already_connected` → перезапросить
     дашборд; 404 `disabled` → скрыть секцию.
4. **Обновление после возврата**: на дашборде слушатели `window 'focus'` и
   `document 'visibilitychange'` → повторный `GET /api/miniapp/dashboard` (по образцу
   `useEffect` в connect-instagram); плюс обработка query `?zernio=success|error` (redirect
   из callback T-047): success → сразу перезапросить дашборд и показать секцию в статусе
   active (тост/бейдж «Подключено»), error → показать секцию с текстом ошибки; query-параметр
   после обработки убрать из URL (`router.replace('/app')`).
5. **i18n**: все строки — новые ключи в ОБЕИХ секциях `ru` и `de` `src/lib/i18n/shared.ts`
   (плоский camelCase с префиксом `zernio*`, напр. `zernioConnectTitle`,
   `zernioConnectDescription`, `zernioConnectButton`, `zernioDisconnectButton`,
   `zernioStatusActive`, `zernioStatusError`, `zernioStatusPending`). Пользовательские
   строки в разметку — только через существующие механизмы экранирования (React сам
   экранирует; `dangerouslySetInnerHTML` не использовать).
6. **Краевые случаи**: у тенанта одновременно активны Meta-директ и Zernio — обе секции
   показывают свои статусы независимо (дедуп сообщений уже решён в T-049; конфликтов UI
   нет); Zernio недоступен при клике (502 от connect) → тост «Попробуйте позже», кнопка
   снова активна.
7. **Manual-test** `docs/manual-tests/T-051.md`: прогон на desktop TG и мобильном TG
   (embedded-путь!): секция видна только при заполненных env; полный цикл Подключить →
   OAuth → возврат в мини-апп → статус active с @username; Отключить → статус
   disconnected; переподключение; при выключенном env секции нет и дашборд не сломан;
   негативная — обрыв OAuth (закрыть браузер) → секция в pending, повторный клик работает.

## Вне объёма
- Блок Zernio на финальном экране онбординга (T-042) — отдельным тикетом после пилота.
- Диагностика здоровья подключения (health-check) в UI.
