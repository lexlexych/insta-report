# T-017 · Сбор контекста беседы

> **Перед началом:** прочитай `docs/plan.md` §2.2, §5. Зависимости: T-013, T-016. Референс — Code-ноды WF1 (история + pendingText).

## Цель
`src/lib/pipeline/context.ts`: по событию собрать историю диалога и «неотвеченный хвост» клиента.

## Шаги выполнения
1. Тип результата: `type ConversationContext = { username: string | null; history: string; pendingText: string; lastBusinessTs: number | null; messages: { text; fromId; createdTime }[] }`.
2. `buildContext(conn: IgConnectionDecrypted, ev: IgEvent): Promise<ConversationContext>`:
   - параллельно: `getUsername(token, ev.contactId)` и `getConversation(token, conn.igAccountId, ev.contactId, 20)`;
   - `getConversation` вернул пусто (новая беседа/лаг API) → messages = [{ text: ev.text, fromId: ev.contactId, createdTime: ev.ts }];
   - **чистая функция** `deriveContext(messages, igAccountId, fallbackText)` (выдели отдельно для тестов):
     - строки истории: `Бизнес: {text}` если fromId === igAccountId, иначе `Клиент: {text}`; соединить \n; пустые тексты → `[вложение]`;
     - pendingText: пройди с конца, собери подряд идущие сообщения клиента до первого сообщения бизнеса; разверни в хронологию, соедини \n; если пусто (гонка) → fallbackText;
     - lastBusinessTs: createdTime последнего сообщения бизнеса (мс) или null.
3. Учти нормализацию `createdTime`: Graph отдаёт ISO-строку — конвертируй в epoch ms в клиенте T-013 (если ещё не сделано — добавь там и поправь тесты T-013).
4. Тесты `tests/pipeline.context.test.ts` (deriveContext — без моков сети):
   - [К, Б, К, К] → pendingText = два последних К через \n;
   - бизнес не писал вовсе → pendingText = все сообщения клиента, lastBusinessTs=null;
   - последним писал бизнес → pendingText = fallbackText;
   - история хронологична и содержит префиксы сторон; вложение → плейсхолдер.
   Плюс тест buildContext c моками: пустая беседа → single-message fallback.
