# T-011 · Per-tenant вебхук Meta: verify + приём

> **Перед началом:** прочитай `docs/plan.md` §2.2–2.3, §5. Зависимости: T-004. Обработка события (`handleIgEvent`) появится в T-016 — здесь подключи заглушку.

## Цель
Роут `app/api/wh/ig/[tenantId]/route.ts`: GET-handshake Meta и приём POST c проверкой подписи и мгновенным ответом.

## Шаги выполнения
1. `export const maxDuration = 60;` Установи `@vercel/functions` (для `waitUntil`).
2. **GET** (verify handshake): query `hub.mode=subscribe`, `hub.verify_token`, `hub.challenge`.
   - Загрузи `igConnections.getForTenant(params.tenantId)`; нет подключения → 404.
   - Сравни verify_token (timing-safe, длины выровнять до сравнения); mismatch → 403.
   - Успех: `igConnections.setStatus(tenantId,'active')` не вызывай — handshake ещё не значит «активно»; вместо этого добавь поле-факт: `touchWebhookSeen` + отдельный флаг `handshake_at` (добавь колонку миграцией `0003_handshake.sql`: `alter table ig_connections add column handshake_at timestamptz;`). Ответ: `new Response(challenge, { status: 200 })` (text/plain).
3. **POST**:
   - Прочитай тело как raw: `const raw = await req.text()` — подпись считается по СЫРЫМ байтам, парсить JSON до проверки нельзя.
   - Заголовок `x-hub-signature-256` формата `sha256=<hex>`; отсутствует → 401.
   - Вычисли `hmacSHA256(appSecret_тенанта, raw)` (appSecret из `igConnections.getForTenant`); сравни `crypto.timingSafeEqual` (буферы одинаковой длины; иначе — 401 без исключения).
   - Невалидная подпись → 401 (залогируй tenantId, без тела).
   - Валидная: `touchWebhookSeen(tenantId)`; `waitUntil(handleIgEvent(tenantId, JSON.parse(raw)).catch(err => logPipelineError(...)))`; немедленно `Response('OK', {status:200})`.
4. Создай заглушку `src/lib/pipeline/handleIgEvent.ts`: `export async function handleIgEvent(tenantId: string, body: unknown) { /* T-016 */ }` + `logPipelineError` (console.error до T-029).
5. Тесты `tests/wh.ig.route.test.ts` (вызывай экспортированные GET/POST c собранными Request):
   - GET: верный verify_token → 200 и тело === challenge; неверный → 403; нет подключения → 404;
   - POST: подпись, посчитанная тем же секретом по телу → 200, `handleIgEvent` вызван (мок) c распарсенным body;
   - подпись от другого секрета / изменённое тело → 401, handleIgEvent НЕ вызван;
   - отсутствие заголовка → 401;
   - убедись, что raw читается один раз и парсинг происходит после проверки (порядок вызовов моков).
