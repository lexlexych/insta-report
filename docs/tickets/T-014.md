# T-014 · Диагностика подключения Instagram

> **Перед началом:** прочитай `docs/plan.md` §1, §5. Зависимости: T-012, T-013.

## Цель
Самодиагностика «токен · handshake · события» в визарде подключения — главный инструмент снижения тикетов поддержки.

## Шаги выполнения
1. `app/api/miniapp/ig/status/route.ts` GET (requireTenant):
   - нет connection → `{ state: 'not_configured' }`;
   - есть токен → живой вызов `getAccount(accessToken)`: успех → сохрани `ig_account_id`, `ig_username`, статус `active` (если был pending/error), `tokenOk: true`; `IgAuthError` → статус `error` + `tokenOk: false, tokenError: '...'` (человекочитаемо: «Токен недействителен или отозван — сгенерируйте новый в Meta App Dashboard»);
   - собери ответ: `{ tokenOk, igUsername, handshakeOk: !!handshake_at, lastEventAt: webhook_last_seen_at, checks: [...] }` где checks — массив `{ id: 'token'|'handshake'|'event', status: 'ok'|'fail'|'pending', hint?: string }`. Логика event: `webhook_last_seen_at > handshake_at` → ok, иначе pending c hint «Отправьте сообщение вашему аккаунту c другого профиля Instagram и нажмите Обновить».
2. UI-компонент `ConnectionDiagnostics` на странице connect-instagram:
   - три строки-индикатора (✅ / ⏳ / ❌ + название + hint при не-ok): «Токен доступа», «Вебхук подтверждён (handshake)», «Событие получено»;
   - hint для fail handshake: «Проверьте, что Callback URL и Verify Token вставлены без пробелов, и нажмите Verify and Save в Meta Dashboard»;
   - кнопка «Обновить» (повторный GET, спиннер); авто-поллинг каждые 5 с, пока не все ok (останавливать при уходе со страницы);
   - при всех ok — зелёный блок «Подключение работает! @{username}» + кнопка «На дашборд».
3. На дашборд-заглушку выведи компактный статус-бейдж подключения (переиспользуй GET /ig/status).
4. Тесты: API — valid token → username сохранён и status active; IgAuthError → status error + понятный hint; event-check: last_seen > handshake → ok. Компонентный: поллинг останавливается, когда все ok.

## Критерии приёмки
- [ ] Каждый индикатор имеет hint, объясняющий, что чинить
- [ ] Ошибочный токен переводит подключение в error и показывает инструкцию
- [ ] Поллинг не течёт (cleanup в useEffect)
