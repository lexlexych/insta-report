# T-004 · Слой репозиториев (доступ к данным)

> **Перед началом:** прочитай `docs/plan.md` §2.4, §5. Зависимости: T-002, T-003. Всё остальное приложение обращается к БД ТОЛЬКО через этот слой.

## Цель
`src/lib/db/` — серверный Supabase-клиент + типизированные репозитории с атомарными операциями.

## Шаги выполнения
1. `src/lib/db/client.ts`: `createClient<Database>(env.SUPABASE_URL, env.SUPABASE_SERVICE_ROLE_KEY, { auth: { persistSession: false } })`, ленивый синглтон `getDb()`. Пометь файл `import 'server-only'`.
2. Общее: каждая функция возвращает типизированные данные или бросает `DbError` (обёртка над ошибкой supabase c контекстом «operation»). Никаких `any`.
3. `src/lib/db/tenants.ts`:
   - `upsertByTelegramUserId(tgUserId, patch)` → tenant (insert on conflict update tg_chat_id);
   - `getById(id)`, `getByTelegramUserId(tgUserId)`;
   - `update(id, patch)`; `deleteCascade(id)` (delete tenants — каскад по FK).
4. `src/lib/db/igConnections.ts`:
   - `upsertForTenant(tenantId, { accessToken?, appSecret?, verifyToken?, ... })` — accessToken/appSecret ПЕРЕД записью прогоняются через `encrypt()`; поля в БД: `access_token_enc`, `app_secret_enc`;
   - `getForTenant(tenantId)` → объект с РАСШИФРОВАННЫМИ `accessToken`, `appSecret` (decrypt при чтении) + остальные поля;
   - `setStatus(tenantId, status, error?)`, `touchWebhookSeen(tenantId)`, `markTokenRefreshed(tenantId, newTokenEnc)`;
   - `listActiveForRefresh(olderThanDays)` — для cron.
5. `src/lib/db/labels.ts`: `listByTenant`, `create`, `updateById`, `deleteById` (запрет для name='Без категории' — бросать `ForbiddenLabelError`), `findByNameCI(tenantId, name)`.
6. `src/lib/db/drafts.ts`:
   - `insertPending(data)` → draft; при нарушении partial-unique (23505) бросай `PendingExistsError`;
   - `getById(id)`;
   - `claimPendingToSending(id)` → draft | null: `update({status:'sending'}).eq('id',id).eq('status','pending').select().maybeSingle()`;
   - `setStatus(id, status, extra?)`;
   - `getPendingByConversation(tenantId, conversationKey)`;
   - `cancelPendingByConversation(tenantId, conversationKey)` → отменённый draft | null (update ... eq status pending returning).
7. `src/lib/db/processedEvents.ts`: `tryInsert(tenantId, mid): Promise<boolean>` — insert; код 23505 → false; иначе true.
8. `src/lib/db/usageStats.ts`: `increment(tenantId, patch)` — вызов rpc `increment_usage` c дефолтами 0; `getRange(tenantId, fromDay, toDay)`.
9. `src/lib/db/messageLog.ts`: `add(tenantId, conversationKey, direction, text)`, `recent(tenantId, limit)`.
10. `src/lib/db/index.ts` — реэкспорт всех репозиториев.
11. Тесты `tests/db.repos.test.ts`: замокай `getDb()` (vi.mock) цепочечным builder-моком. Проверь:
    - upsertForTenant шифрует (в мок попадает строка с префиксом `v1:`), getForTenant расшифровывает;
    - claimPendingToSending: мок вернул null → функция вернула null без исключения;
    - tryInsert: мок бросил 23505 → false; иная ошибка → DbError;
    - deleteById для «Без категории» → ForbiddenLabelError.
