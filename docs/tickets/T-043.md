# T-043 · Предодобрение Instagram-аккаунтов админом и статусная модель подключения

> **Перед началом:** прочитай `docs/plan.md` §1, §5 и `docs/tickets/OVERRIDES.md`.
> Зависимости: T-036 (реестр `ig_accounts`), T-037 (заявки и подтверждение админом).
> Контекст: чтобы пользователь мог подключить Instagram, администратор платформы должен внести
> его username в Meta App (тестировщики приложения) И подтвердить его в реестре `ig_accounts`.
> Тикет закрывает оба сценария появления ника в реестре:
> (а) админ внёс ник заранее, до онбординга пользователя (тенанта ещё нет);
> (б) пользователь подал заявку на шаге 2 онбординга, админ подтвердил (T-037).
> Пока ник не подтверждён — подключение Instagram для пользователя заблокировано.

## Цель

Команда бота `/ig_add` для предодобрения ника; единый connect-статус для UI; запрет старта
OAuth до подтверждения администратором; статус «Ожидание подтверждения администратором»
на дашборде.

## Команда `/ig_add`

1. В Telegram-боте команда `/ig_add <username>` — только для админов (та же проверка прав, что
   у кнопки «Подтвердить» из T-037; для остальных — отказ/игнор, как принято в боте для
   админ-действий).
2. Поведение (ответ админу в тот же чат; идемпотентно):
   - ник не проходит `normalizeIgUsername` → ответ «некорректный username»;
   - записи нет → создать в `ig_accounts` запись `status='approved'`, `tenant_id=null`,
     `approved_at=now`, `approved_by_tg_id=<админ>` (новая функция репозитория, например
     `igAccounts.createApproved`);
   - есть запись `pending` → подтвердить тем же путём, что кнопка «Подтвердить» (T-037),
     включая уведомление владельца-тенанта со `startapp=connect`, если `tenant_id` заполнен;
   - уже `approved` → ответить «уже подтверждён», ничего не менять.
3. Привязка предодобренного ника к тенанту при вводе на шаге 2 онбординга уже реализована
   (`POST onboarding/ig-account` → `bindTenant`) — код не менять, сценарий покрыть в manual-test.

## Единый connect-статус

4. Расширить `GET /api/miniapp/ig/status`: дополнительно к текущим полям возвращать
   `connect: 'none' | 'awaiting_admin' | 'ready' | 'active' | 'error'`:
   - `none` — у тенанта нет записи в `ig_accounts`;
   - `awaiting_admin` — заявка в статусе `pending`;
   - `ready` — ник `approved`, но IG-подключение (ig_connections) не `active`;
   - `active` / `error` — по статусу IG-подключения (как сейчас).
   Существующие поля ответа не ломать — `InstagramConnectPanel` продолжает работать до
   переработки в T-044.
5. `GET /api/miniapp/ig/login-url`: если у тенанта нет `approved`-записи в `ig_accounts`
   (т.е. `connect` = `none` или `awaiting_admin`) → 403 `{ code: 'not_approved' }`. OAuth
   невозможно начать до подтверждения админом.

## Дашборд

6. Блок статуса Instagram на дашборде использует `connect`:
   - `awaiting_admin` → статус-строка «Ожидание подтверждения администратором» (иконка-часы),
     кнопки/ссылки подключения нет;
   - `ready` → кнопка «Подключить Instagram» → `/app/connect-instagram`;
   - `active` → как сейчас (подключено);
   - `error` → как сейчас (ошибка подключения + переход на страницу подключения);
   - `none` → нейтральное «Instagram не подключён» без кнопки.
7. i18n ru/de всех новых строк.
8. Manual-test `docs/manual-tests/T-043.md`:
   - сценарий (а): `/ig_add` до онбординга → пользователь проходит онбординг и вводит этот ник
     на шаге 2 → тенант привязан, `connect='ready'`, кнопка подключения на дашборде доступна;
   - сценарий (б): заявка с шага 2 → уведомление админам → `/ig_add <тот же ник>` (и отдельно —
     кнопка «Подтвердить») → у пользователя `connect='ready'`, владельцу пришло уведомление;
   - `login-url` возвращает 403 при `pending`-заявке; `/ig_add` от не-админа отклоняется;
     повторный `/ig_add` отвечает «уже подтверждён».

## Критерии приёмки

- [ ] `/ig_add` доступна только админам; ветки «нет записи / pending / approved» работают
      идемпотентно; при подтверждении pending-заявки с тенантом владелец получает уведомление
- [ ] Сценарий «внесён заранее»: предодобренный ник привязывается к тенанту при вводе на шаге 2
- [ ] `GET ig/status` возвращает `connect`-статус; `login-url` отвечает 403 `not_approved`,
      пока ник не подтверждён
- [ ] Дашборд: при `awaiting_admin` — «Ожидание подтверждения администратором» без кнопки;
      кнопка «Подключить Instagram» — только при `ready`
- [ ] `pnpm lint && pnpm typecheck` зелёные; manual-test написан
