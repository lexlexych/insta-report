# T-035 · Реестр сфер деятельности и шаблоны баз знаний

> **Перед началом:** прочитай `docs/plan.md` §1, §5 (ревизия v1.2 — Онбординг 2.0) и
> `docs/tickets/OVERRIDES.md`. Зависимости: — (нужен только скаффолд T-001).
> Контекст эпика 10: LLM-генерация базы знаний при онбординге упраздняется; вместо неё —
> предустановленные Markdown-шаблоны по сферам деятельности, которые пользователь правит под себя.

## Цель

Кодовый реестр сфер деятельности (id, иконка, i18n-название) + типобезопасный доступ к
предустановленным шаблонам баз знаний из `src/lib/kb-templates/{locale}/*.md` — на каждом
языке интерфейса системы (сейчас `ru` и `de`, тип `Locale` из `src/lib/i18n`).

## Контекст

Markdown-шаблоны четырёх сфер на обоих языках уже лежат в репозитории (созданы при постановке
эпика, содержимое согласовано человеком — не переписывать, только читать). Структура:
`src/lib/kb-templates/{locale}/{sphereId}.md`, где locale ∈ {ru, de}:

- `events.md` — организация мероприятий / Eventagentur
- `photographer.md` — фотограф / Fotograf:in
- `travel.md` — туристическое агентство / Reisebüro
- `massage.md` — салон массажа / Massagesalon

Формат шаблонов: только заголовки первого уровня (`#`) и маркированные списки; конкретные
примерные величины (адрес, часы, цены), обязательный раздел «Вопросы и ответы» / «Fragen und
Antworten». Шаблон показывается пользователю на языке его интерфейса (локаль определяется из
языка Telegram при первом запуске — см. `resolveLocale` в `src/lib/i18n/shared.ts` и T-038).

## Шаги выполнения

1. `src/lib/kb-templates/index.ts`:
   - `export const BUSINESS_SPHERES = [...] as const` — элементы
     `{ id, icon, nameKey }`: `('events', '🎉')`, `('photographer', '📷')`,
     `('travel', '✈️')`, `('massage', '💆')`; `nameKey` — ключ i18n.
   - `export type BusinessSphereId = (typeof BUSINESS_SPHERES)[number]['id']`.
   - `export function getKbTemplate(id: BusinessSphereId, locale: Locale): string` и
     `export function isBusinessSphereId(v: string): v is BusinessSphereId`
     (`Locale` — существующий тип из `src/lib/i18n`; добавление нового языка в систему обязано
     ломать typecheck, пока не добавлены шаблоны).
2. Доставка содержимого .md в бандл — через кодоген (в serverless нельзя полагаться на
   `fs.readFileSync` произвольных путей): скрипт `scripts/gen-kb-templates.ts` обходит
   `src/lib/kb-templates/{locale}/*.md` и пишет `src/lib/kb-templates/templates.gen.ts`
   (объект `{ [locale]: { [sphereId]: string } }`, шапка «AUTO-GENERATED, не править руками»).
   Скрипт падает с понятной ошибкой, если для какой-либо пары сфера×локаль файла нет —
   комплект шаблонов обязан быть полным для всех языков системы.
   npm-скрипт `gen:kb-templates`. Сгенерированный файл коммитится (по образцу
   `src/lib/db/types.gen.ts`). `.md`-файлы — единственный источник истины.
3. Режим проверки `pnpm gen:kb-templates --check` — выход с ненулевым кодом, если
   `templates.gen.ts` рассинхронизирован с .md (для ревью и ручной гигиены; в CI не добавлять —
   CI остаётся lint + typecheck по OVERRIDES).
4. i18n (`src/lib/i18n`, ru + de): названия сфер — «Организация мероприятий / Eventagentur»,
   «Фотограф / Fotograf:in», «Туристическое агентство / Reisebüro»,
   «Салон массажа / Massagesalon».
5. Manual-test `docs/manual-tests/T-035.md` по шаблону `docs/orchestration/manual-testing.md`
   (+ при необходимости demo-скрипт `scripts/manual/T-035-demo.ts`, печатающий реестр и первые
   строки каждого шаблона).

## Критерии приёмки

- [ ] `getKbTemplate(id, locale)` возвращает полный текст соответствующего .md для всех
      восьми комбинаций (4 сферы × ru/de)
- [ ] `pnpm gen:kb-templates` детерминированно перегенерирует `templates.gen.ts`; `--check`
      ловит рассинхронизацию; отсутствие файла для пары сфера×локаль — ошибка кодогена
- [ ] Содержимое .md-шаблонов не изменено; названия сфер — через i18n (ru/de), иконки — в реестре
- [ ] `pnpm lint && pnpm typecheck` зелёные; manual-test написан
