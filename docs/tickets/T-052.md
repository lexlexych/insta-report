# T-052 · Zernio: деплой-документация, ops-заметки и сквозной ручной прогон

> **Перед началом:** прочитай `docs/research/zernio.md` целиком (особенно §9 — открытые
> вопросы) и `docs/plan.md` §5. Зависимости: T-047, T-048, T-050, T-051.
> Эпик 12, ветка эпика — `zernio`. Кода приложения в тикете почти нет — документация и
> финальная сверка эпика.

## Цель
Оператор сервиса может включить Zernio-канал на новом окружении по документации за
~15 минут; известные ограничения и открытые вопросы пилота зафиксированы; есть сквозной
ручной прогон всего флоу.

## Шаги выполнения
1. **`docs/deploy.md`** — новый раздел «Instagram через Zernio (опционально)»:
   - регистрация на zernio.com, создание API-ключа в дашборде;
   - env: `ZERNIO_API_KEY`, `ZERNIO_WEBHOOK_SECRET` (сгенерировать `scripts/gen-key.ts`),
     `ZERNIO_API_BASE` (менять не нужно);
   - регистрация вебхука: `pnpm tsx scripts/manual/zernio-setup-webhook.ts` (после деплоя,
     когда известен продовый `APP_BASE_URL`);
   - тарифные заметки: первые 2 подключённых аккаунта бесплатно; возможное требование
     «Inbox addon» (403) — как распознать и что включить; лимит профилей плана
     (403 на создание профиля);
   - мониторинг: Zernio выключает вебхук после 10 подряд ошибок доставки — куда смотреть
     (`GET /v1/webhooks/logs`, дашборд Zernio) и как чинить (повторный запуск
     setup-скрипта); статус сервиса status.zernio.com;
   - фича полностью выключается удалением env-переменных (поведение приложения
     возвращается к прямой Meta-интеграции).
2. **README** — раздел «Каналы подключения Instagram»: два способа (Meta-директ и Zernio),
   их различия в UX (тестировщик/одобрение админа vs один клик), ссылка на
   `docs/research/zernio.md`.
3. **`docs/research/zernio.md` §9** — актуализировать по итогам пилота (T-047…T-051 уже
   влиты и проверены человеком): какие из открытых вопросов закрыты и как (форма отказа
   OAuth, `message.sent` для ручных ответов, Inbox addon, Creator-аккаунты, латентность).
   Данные для ответов бери из manual-test-прогонов предыдущих тикетов (спроси человека
   через notes, если данных нет — пометь «не проверено в пилоте»).
4. **Сверка статусной модели**: убедись, что `docs/manual-tests/T-047…T-051.md` существуют
   и их сценарии не противоречат итоговой реализации (это сверка ссылок и шагов, не
   переписывание).
5. **Manual-test** `docs/manual-tests/T-052.md` — СКВОЗНОЙ прогон на чистом окружении по
   новому разделу deploy.md: env → db:push → setup-webhook → подключение из мини-аппа →
   DM → карточка → «Отправить» → ответ в Instagram → отключение. Плюс негативная:
   выключить env → приложение работает как до эпика (секции на дашборде нет, `/api/wh/zernio`
   и `/api/miniapp/zernio/*` отвечают 404).

## Вне объёма
- Другие каналы через Zernio (WhatsApp/Telegram/FB) — следующий эпик: фиксируй в README
  одной строкой «архитектура готова: platform-колонка, общий вебхук», без обещаний сроков.
- Автоматический мониторинг вебхука (алерты) — при необходимости отдельный тикет.
