# T-005 · Каркас Telegram-бота (grammY) и вебхук

> **Перед началом:** прочитай `docs/plan.md` §2, §5. Зависимости: T-004.

## Цель
Центральный бот принимает апдейты через `/api/telegram`, обрабатывает `/start`, есть скрипт установки вебхука и меню Mini App.

## Шаги выполнения
1. Установи `grammy`. Создай `src/lib/tg/bot.ts`:
   - ленивый синглтон `getBot(): Bot` (`new Bot(env.TELEGRAM_BOT_TOKEN)`);
   - зарегистрируй обработчики: `bot.command('start', onStart)`; `bot.on('callback_query:data', onCallback)` — пока заглушка `answerCallbackQuery()` c TODO для T-021;
   - `onStart`: `tenants.upsertByTelegramUserId(ctx.from.id, { tg_chat_id: ctx.chat.id })`; ответ приветствием (i18n, ru/de по `ctx.from.language_code`) + InlineKeyboard c кнопкой `webApp('Открыть панель', env.APP_BASE_URL + '/app')`.
2. Создай `app/api/telegram/route.ts`:
   - `export const maxDuration = 60;`
   - `POST`: проверь `req.headers.get('x-telegram-bot-api-secret-token') === env.TELEGRAM_WEBHOOK_SECRET`, иначе `401`; далее `webhookCallback(getBot(), 'std/http')(req)`.
3. `src/lib/tg/api.ts` — тонкие хелперы поверх `bot.api` для использования вне контекста апдейта: `sendMessageHTML(chatId, html, keyboard?, threadId?)`, `editMessageHTML(chatId, messageId, html, keyboard?)`, `deleteMessageSafe(chatId, messageId)` (ошибки «message to delete not found» глотать, остальные пробрасывать), `answerCallback(id, text?)`. Все с `parse_mode: 'HTML'`, `link_preview_options: { is_disabled: true }`.
4. `scripts/setup-telegram.ts`:
   - `setWebhook(APP_BASE_URL + '/api/telegram', { secret_token, allowed_updates: ['message','callback_query','my_chat_member'] })`;
   - `setMyCommands([{command:'start', description:'Запустить / открыть панель'}])` для ru и de через language_code;
   - `setChatMenuButton({ menu_button: { type:'web_app', text:'Панель', web_app:{ url: APP_BASE_URL + '/app' } } })`;
   - вывод результата getWebhookInfo. Скрипт `pnpm setup:telegram` (tsx).
5. Тесты `tests/tg.webhook.test.ts`:
   - POST без/с неверным secret header → 401, тело не читается (мок webhookCallback не вызван);
   - с верным secret → мок webhookCallback вызван;
   - `onStart` дважды с одним from.id → `upsertByTelegramUserId` вызван оба раза, моки не создают дубль (проверка аргументов).
   - `deleteMessageSafe`: мок бросает GrammyError c описанием "message to delete not found" → не пробрасывается; другая ошибка → пробрасывается.

## Критерии приёмки
- [ ] 401 при неверном secret; апдейты доходят до обработчиков при верном
- [ ] /start идемпотентен; кнопка web_app присутствует в ответе
- [ ] setup-скрипт идемпотентен (повторный запуск не ломает вебхук)
