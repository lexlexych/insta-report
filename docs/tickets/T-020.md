# T-020 · Карточка черновика в Telegram + замещение старого

> **Перед началом:** прочитай `docs/plan.md` §2.2, §5. Зависимости: T-005, T-019.

## Цель
Доставка черновика владельцу: HTML-карточка c кнопками, отзыв предыдущего pending-черновика той же беседы, запись в drafts.

## Шаги выполнения
1. `src/lib/tg/html.ts`: `escapeHTML(s)` — замена & < > " (в этом порядке). Все пользовательские строки в карточках проходят через неё.
2. `src/lib/tg/draftCard.ts` — `renderDraftCard(p: { username; pendingText; labelName; draftText; status?: 'pending'|'sent'|'skipped'|'error'; statusLine?: string }): string`:
   ```
   📩 <b>Новое сообщение</b> от <a href="https://instagram.com/{username}">@{username}</a>
   <blockquote>{pendingText}</blockquote>
   🏷 {labelName}

   <b>Черновик ответа</b> <i>(нажмите текст, чтобы скопировать)</i>:
   <pre>{draftText}</pre>
   {statusLine?}
   ```
   username может быть null → «клиента» без ссылки. Обрежь pendingText до 500 симв. и draftText до 3000 c «…» (лимит Telegram 4096 на сообщение — проверь итоговую длину, при превышении сократи draftText сильнее).
   `draftKeyboard(draftId, username)`: ряд 1 — URL-кнопка «🔗 Открыть в Instagram» (`https://ig.me/m/{username}`; нет username → кнопку не добавлять); ряд 2 — callback «✅ Отправить» c data `send:{draftId}` (uuid=36 симв., итого ≤64 байта — ок; добавь assert).
3. `src/lib/pipeline/deliver.ts` — `deliverDraft({...})`:
   1. `old = drafts.cancelPendingByConversation(tenantId, key)`; если old → `deleteMessageSafe(old.tg_chat_id, old.tg_message_id)`;
   2. `chatId = tenant.tg_chat_id` (null → лог-ошибка и выход);
   3. `msg = sendMessageHTML(chatId, renderDraftCard(...), draftKeyboard(draftId, username))` — ВАЖНО: draftId нужен до отправки → генерируй uuid на стороне приложения (`crypto.randomUUID()`), передай его в insert как id;
   4. `drafts.insertPending({ id: draftId, tenant_id, conversation_key, contact_id, contact_username, pending_text, history_snapshot: ctx.history, label_id, draft_text, tg_chat_id, tg_message_id: msg.message_id, trigger_ts: ev.ts })`;
   5. `PendingExistsError` (гонка двух вебхуков): удали только что отправленное сообщение, залогируй, выйди без исключения.
4. Подключи deliverDraft в handleIncoming (замени заглушку T-019).
5. Тесты `tests/pipeline.deliver.test.ts` (моки tg, db):
   - текст клиента `<script>alert(1)</script>` → в HTML карточки экранирован;
   - существует old pending → deleteMessageSafe вызван, cancel вызван ДО отправки новой карточки (порядок);
   - PendingExistsError → отправленное сообщение удалено, исключение не пробрасывается;
   - без username → одна кнопка; callback_data === `send:{uuid}`;
   - длинный draftText → итоговый HTML ≤ 4096.

## Критерии приёмки
- [ ] HTML-инъекции невозможны (тест)
- [ ] Один pending-черновик на беседу гарантирован (замещение + обработка гонки)
- [ ] Порядок операций исключает «две живые карточки» в чате
