# T-032 · Деплой-документация и юридические страницы

> **Перед началом:** прочитай `docs/plan.md` §4, §5. Зависимости: содержательно — все предыдущие. Финальный тикет MVP.

## Цель
Документация развёртывания «с нуля до первого тенанта» + публичные правовые страницы (шаблоны).

## Шаги выполнения
1. `docs/deploy.md` — пошагово, c командами и скриншот-плейсхолдерами:
   1. Форк/клон GitHub-репо; секреты Actions (нет — CI только lint/test);
   2. Supabase: создать проект (регион **eu-central / Frankfurt**), `supabase link`, `pnpm db:push`, `pnpm db:types`, взять URL + service_role key;
   3. Telegram: создать бота у BotFather (имя, описание, аватар), получить token; сгенерировать TELEGRAM_WEBHOOK_SECRET и ENCRYPTION_KEY (`pnpm gen:key`), MINIAPP_JWT_SECRET, CRON_SECRET (openssl rand -base64 32);
   4. Vercel: импорт репо, все env из `.env.example`, включить Fluid Compute, задеплоить; проверить crons на вкладке Cron Jobs;
   5. `APP_BASE_URL` = прод-домен → повторный деплой → `pnpm setup:telegram`;
   6. LLM: ключ OpenRouter, выбор моделей (рекомендация: недорогая для classify, качественная для draft);
   7. Чек-лист первого тенанта: /start → онбординг → тест-чат отвечает → подключение IG по визарду → диагностика зелёная → тестовое сообщение c другого аккаунта → карточка → кнопка → ответ в IG.
   8. Раздел Troubleshooting: 5 типовых проблем (handshake 403 — verify token c пробелом; нет событий — не подписано поле messages; 401 в TG — не выполнен setup:telegram; и т.д.).
2. Правовые страницы — публичные роуты вне (miniapp): `app/legal/impressum/page.tsx`, `datenschutz/page.tsx`, `avv/page.tsx`:
   - серверные компоненты, статичный контент на немецком, простая типографика;
   - Impressum: шаблон c полями-плейсхолдерами `[Name des Unternehmens]`, `[Anschrift]`, `[E-Mail]`, `[USt-IdNr.]`;
   - Datenschutzerklärung: разделы — Verantwortlicher; welche Daten (Telegram-ID, Организационные данные, содержимое переписки Instagram Direct в объёме обработки); Zwecke; Auftragsverarbeiter (Vercel, Supabase (EU-Region Frankfurt), LLM-Anbieter [Name] — c пометкой про передачу текста сообщений для генерации ответов); Speicherdauer; Rechte der Betroffenen (Auskunft, Löschung — «удаление в один клик в настройках»); Cookies (одна httpOnly-сессионная);
   - AVV (DPA, Art. 28 DSGVO): каркас-шаблон c предметом, категориями данных, TOM-приложением (заглушки);
   - на КАЖДОЙ странице заметный дисклеймер: «Vorlage — vor Nutzung rechtlich prüfen lassen / Шаблон — требует проверки юристом»;
4. Ссылки: футер настроек Mini App (T-028) и docs/deploy.md чек-пункт «заполнить плейсхолдеры legal».
5. Обнови корневой README: краткое описание, схема архитектуры (ASCII из plan.md §2.1), ссылки на docs/plan.md и docs/deploy.md, статус-бейдж CI.
6. Тест: роуты /legal/* отвечают 200 без auth (route-тест); аудит-тест T-030 знает про их публичность (исключение-список для страниц без API).
