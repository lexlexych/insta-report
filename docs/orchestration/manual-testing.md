# Шаблон docs/manual-tests/T-0XX.md

Пишется для человека, который НЕ читал код. Каждый шаг — конкретная команда или действие
и ожидаемый наблюдаемый результат. Проверяй шаги мысленно «на чистой машине».

```markdown
# Ручной тест T-0XX · <название тикета>

## Что сделано (2-4 строки, по-человечески)

## Подготовка
- ветка: `git fetch && git checkout ticket/T-0XX && pnpm install`
- env: какие переменные из .env.example должны быть заполнены ИМЕННО для этой проверки
  (и где взять значение: «токен бота из BotFather», «ключ OpenRouter» и т.п.)
- сервисы: что поднять (например `pnpm dev`; для вебхуков — туннель: `npx localtunnel --port 3000`
  или cloudflared; для БД — dev-проект Supabase и `pnpm db:push`)

## Проверки (каждая: Действие → Ожидаемо)
1. …
2. …

## Негативные проверки (минимум одна: неверный ввод/подпись/токен → корректная ошибка)
1. …

## Как понять, что тикет НЕ работает (типовые симптомы и куда смотреть: логи, вкладка Network, БД)
```

Требования к качеству:
- Проверки покрывают КАЖДЫЙ критерий приёмки тикета (в редакции OVERRIDES) — явно сопоставь.
- Если фича пока не имеет UI (библиотечный модуль, репозиторий БД) — дай проверку через
  одноразовый скрипт `pnpm tsx scripts/manual/T-0XX-demo.ts` (создай его в тикете, каталог
  scripts/manual/ разрешён) с печатью понятного результата.
- Если для проверки нужен внешний аккаунт (Meta, Telegram) — опиши минимальный путь его получения.
