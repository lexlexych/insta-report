# Deploy / operations

## Telegram threaded mode (T-024)

Для раскладки черновиков по категориям в личном чате владельца бот должен иметь включённый **Threaded Mode** в BotFather:

1. Откройте `@BotFather`.
2. Выберите бота проекта.
3. Включите режим **Threaded Mode** для личных чатов.

Если режим не включён, `has_topics_enabled=false`: приложение ожидаемо работает в плоском личном чате, без потери черновиков.

## Instagram через Zernio (опционально)

Zernio — альтернативный способ подключить Instagram. Он использует Meta App Zernio, поэтому
пользователю не требуется быть тестировщиком нашего Meta App и ждать одобрения администратора.
Интеграция не заменяет прямое Meta-подключение: оба способа могут существовать у одного тенанта.

### Включение на новом окружении

1. Зарегистрируйте сервисный аккаунт на [zernio.com](https://zernio.com) и в его Dashboard
   создайте API key. Храните ключ только в настройках окружения: ни ключ, ни его значение нельзя
   добавлять в исходный код, логи или скриншоты.
2. Сгенерируйте отдельный секрет подписи вебхука:

   ```bash
   pnpm tsx scripts/gen-key.ts
   ```

   Скопируйте случайную base64-строку из первой строки вывода в секрет хостинга как
   `ZERNIO_WEBHOOK_SECRET`. Скрипт универсальный и в подсказке называет переменную
   `ENCRYPTION_KEY`; для этого шага используйте сгенерированное значение только как секрет
   вебхука. Не сохраняйте вывод команды в историю терминала или документацию.
3. В настройках production-окружения задайте одновременно:

   ```dotenv
   ZERNIO_API_KEY=<ключ из Zernio Dashboard>
   ZERNIO_WEBHOOK_SECRET=<случайный секрет из предыдущего шага>
   ZERNIO_API_BASE=https://zernio.com/api
   ```

   `ZERNIO_API_BASE` обычно менять не нужно. Обе первые переменные обязательны вместе; после
   изменения перезапустите или передеплойте приложение. Убедитесь, что `APP_BASE_URL` уже
   содержит публичный HTTPS-адрес этого развёртывания.
4. Примените миграции к отдельному dev- или production-проекту Supabase:

   ```bash
   pnpm db:push
   ```

5. После деплоя, из окружения с этими production-переменными, создайте или обновите единый
   вебхук Zernio:

   ```bash
   pnpm tsx scripts/manual/zernio-setup-webhook.ts
   ```

   Скрипт настраивает `<APP_BASE_URL>/api/wh/zernio` для `message.received`, `message.sent`,
   `account.connected` и `account.disconnected`. Повторный запуск безопасен: он обновляет
   настройку `insta-reply`, а не создаёт ещё одну. Секрет в вывод не попадает.
6. Откройте Mini App как тестовый тенант и подключите Business-аккаунт Instagram в области
   Zernio. Затем отправьте тестовое DM с другого Instagram-аккаунта и подтвердите карточку
   черновика в Telegram. Полный сценарий — в `docs/manual-tests/T-052.md`.

### Тарифы, лимиты и эксплуатация

- Первые два подключённых аккаунта Zernio бесплатны. При создании профиля ответ `403` означает,
  что исчерпан лимит профилей текущего плана: увеличьте лимит в Zernio или освободите профиль,
  предварительно отключив его аккаунты.
- Inbox API может требовать **Inbox addon**. Его симптом — `403` от операций переписки или
  отправки с сообщением `Inbox addon required`; включите addon в тарифе Zernio и повторите
  операцию. Не маскируйте этот ответ повторными попытками.
- Zernio прекращает доставку после 10 последовательных ошибок вебхука. Проверяйте журнал
  `GET /v1/webhooks/logs` и настройки в Zernio Dashboard. Сначала исправьте доступность
  `<APP_BASE_URL>/api/wh/zernio`, статус приложения и секрет, затем повторно запустите
  `pnpm tsx scripts/manual/zernio-setup-webhook.ts`, чтобы снова включить вебхук.
- Состояние платформы смотрите на [status.zernio.com](https://status.zernio.com). При массовых
  ошибках сначала сопоставьте их с этим статусом и журналом доставок, а не отключайте аккаунты
  тенантов.

### Полное выключение

Удалите **обе** переменные `ZERNIO_API_KEY` и `ZERNIO_WEBHOOK_SECRET` из окружения и
перезапустите приложение. Не оставляйте одну из них: конфигурация намеренно требует пару.
После выключения область Zernio исчезает с дашборда, `/api/wh/zernio` и
`/api/miniapp/zernio/*` отвечают `404`, а прежний прямой Meta-путь продолжает работать без
изменений.
